# 产品管理插件

专为 [OpenClaw](https://openclaw.ai) 设计的产品管理插件，同时支持 Claude Code。覆盖完整的 PM 工作流：撰写功能规格、管理路线图、与干系人沟通、合成用户研究、分析竞品以及追踪产品指标。

## 安装

将插件目录复制到 ~/.openclaw/skills/product-management/
或使用 ClawHub: openclaw skills install product-management

## 功能概览

本插件为你提供一个 AI 驱动的产品管理搭档，可协助完成以下工作：

- **功能规格与 PRD** — 根据问题陈述或功能想法生成结构化的产品需求文档。包含用户故事、需求优先级、成功指标和范围管理。
- **路线图规划** — 创建、更新和重新排序产品路线图。支持 Now/Next/Later、季度主题和 OKR 对齐格式，含依赖关系映射。
- **干系人更新** — 生成针对不同受众（高管、工程团队、客户）定制的状态更新。从已接入工具中获取上下文，省去每周手动整理的时间。
- **用户研究合成** — 将访谈记录、问卷数据和支持工单转化为结构化洞察。识别主题、构建用户画像，并提供有据可查的机会点。
- **竞品分析** — 调研竞争对手，生成含功能对比、定位分析和战略含义的竞品简报。
- **指标复盘** — 分析产品指标、识别趋势、与目标对比，并输出可行动的洞察。
- **产品头脑风暴** — 探索问题空间、生成创意，并以犀利的对话伙伴身份压测产品思路。支持发散式构思、假设验证和策略探索，可运用 HMW、JTBD、第一性原理和机会解决方案树等框架。

## 命令

| 命令 | 功能 |
|---|---|
| `/write-spec` | 根据问题陈述撰写功能规格或 PRD |
| `/roadmap-update` | 更新、新建或重新排序路线图 |
| `/stakeholder-update` | 生成干系人更新（周报、月报、发布公告） |
| `/synthesize-research` | 合成访谈、问卷和工单中的用户研究 |
| `/competitive-brief` | 创建竞品分析简报 |
| `/metrics-review` | 复盘和分析产品指标 |
| `/brainstorm` | 与思维伙伴头脑风暴产品想法、问题空间或战略问题 |

## 技能

| 技能 | 覆盖内容 |
|---|---|
| `feature-spec` | PRD 结构、用户故事、需求分类、验收标准 |
| `roadmap-management` | 优先级框架（RICE、MoSCoW）、路线图格式、依赖关系映射 |
| `stakeholder-comms` | 按受众分类的更新模板、风险沟通、决策记录 |
| `user-research-synthesis` | 主题分析、亲和图、用户画像开发、机会规模估算 |
| `competitive-analysis` | 功能对比矩阵、定位分析、胜负分析 |
| `metrics-tracking` | 产品指标层级、目标设定（OKR）、仪表板设计、复盘节奏 |
| `product-brainstorming` | 头脑风暴模式（问题探索、解决方案构思、假设测试、战略探索）、PM 框架、会话结构、思维伙伴行为 |

## 工作流示例

### 撰写 PRD

```
你：/write-spec
AI助手：你想为哪个功能或问题写规格文档？
你：我们需要为企业客户添加 SSO 支持
AI助手：[询问目标用户、约束条件、成功指标]
AI助手：[生成完整 PRD，包含问题陈述、用户故事、需求、成功指标、待解决问题]
```

### 准备干系人更新

```
你：/stakeholder-update
AI助手：是哪类更新？（周报、月报、发布公告、临时更新）
你：给高管团队的周报
AI助手：[从项目管理工具、沟通记录和文档中提取上下文]
AI助手：[生成高管摘要，包含进展、决策、风险和下一里程碑]
```

### 合成用户研究

```
你：/synthesize-research
AI助手：你想合成哪些研究？可以粘贴访谈记录、上传文件，或由我从已接入来源中提取。
你：[粘贴 8 份访谈记录]
AI助手：[识别访谈中的主题、模式和洞察]
AI助手：[生成合成报告，包含核心发现、用户画像、机会点和证据]
```

### 头脑风暴产品想法

```
你：/brainstorm 我们是否应该为产品添加 AI 搜索功能？
AI助手：先别急着讨论 AI 搜索——用户目前在搜索上遇到了什么问题？
你：找不到东西。"找不到 X"的支持工单是我们第二大类别。
AI助手：这是发现问题，不一定是搜索技术问题。让我深入探讨一下……
AI助手：[探索问题究竟是搜索质量、信息架构，还是内容可发现性]
AI助手：[提出 5 种不同方案，从完善分类体系到 AI 搜索再到引导式导航]
AI助手：[挑战假设，建议用最低成本验证最高风险点的实验]
```

### 竞品分析

```
你：/competitive-brief
AI助手：你想分析哪些竞争对手或功能领域？
你：对比我们的新用户引导流程与[竞争对手 A]和[竞争对手 B]
AI助手：[调研竞争对手的新用户引导方式]
AI助手：[生成简报，包含功能对比、优劣势分析和战略含义]
```
