# 指标复盘

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

复盘产品指标，识别趋势，输出可行动洞察。

## 使用方式

```
/metrics-review $ARGUMENTS
```

## 工作流

### 1. 收集指标数据

如果 **~~product analytics** 已接入：
- 拉取相关时间段的核心产品指标
- 获取对比数据（上一周期、去年同期、目标值）
- 如有可能，拉取细分数据

如果没有接入分析工具，请用户提供：
- 指标及其数值（粘贴表格、截图或描述）
- 对比数据（上一周期、目标值）
- 近期变化的背景（发布、故障、季节性）

向用户询问：
- 复盘哪个时间段？（上周、上月、上季度）
- 聚焦哪些指标？还是复盘完整的产品指标体系？
- 是否有对比的目标或 OKR？
- 是否有已知事件可能解释变化（发布、故障、营销活动、季节性）？

### 2. 整理指标结构

按指标层级组织复盘：最顶层为北极星指标，其次是 L1 健康指标（获取、激活、留存、变现、满意度），L2 诊断指标用于下钻分析。详见下方**产品指标层级**的完整定义。

如果用户尚未定义指标层级，先帮助他们确定北极星指标和核心 L1 指标，再继续分析。

### 3. 分析趋势

对每个核心指标：
- **当前值**：指标现在是多少？
- **趋势**：与上一周期相比是上升、下降还是持平？时间跨度是多少？
- **vs 目标**：与目标相比如何？
- **变化速率**：趋势是在加速还是减速？
- **异常点**：是否有突发变化、峰值或下跌？

识别相关性：
- 某个指标的变化是否与另一指标相关？
- 是否有领先指标可以预测滞后指标的变化？
- 拆分数据是否揭示某个聚合趋势实际由特定群体驱动？

### 4. 生成复盘报告

#### 摘要
2-3 句话：产品整体健康度、最值得注意的变化、核心结论。

#### 指标记分卡
表格形式，方便快速扫描：

| 指标 | 当前值 | 上期值 | 变化 | 目标 | 状态 |
|--------|---------|----------|--------|--------|--------|
| [指标] | [数值] | [数值] | [+/- %] | [目标] | [达标 / 风险 / 未达标] |

#### 趋势分析
对每个值得讨论的指标：
- 发生了什么，变化有多显著
- 可能的原因（基于已知事件、相关指标、细分分析的归因）
- 这是一次性事件还是持续趋势

#### 亮点
进展顺利的方面：
- 超越目标的指标
- 需要持续保持的正向趋势
- 表现强劲的细分市场或功能

#### 关注点
需要重视的方面：
- 未达目标或呈负向趋势的指标
- 在成为大问题之前的早期预警信号
- 缺乏可见性或认知的指标

#### 建议行动
基于分析的具体后续步骤：
- 需要深入调查的内容（深挖某个令人担忧的趋势）
- 需要启动的实验（验证可能改善指标的假设）
- 需要追加投入的方向（在有效的事情上加大投入）
- 需要设置的告警（更密切地监控某个指标）

#### 背景与注意事项
- 已知的数据质量问题
- 影响可比性的事件（故障、节假日、发布）
- 应该追踪但尚未追踪的指标

### 5. 后续跟进

生成复盘后：
- 询问是否有指标需要深入调查
- 提供持续监控的仪表板规格
- 提供关注点领域的实验方案
- 提供周期性复盘的模板

## 产品指标层级

### 北极星指标
最能捕捉产品为用户带来核心价值的单一指标。它应该：

- **价值对齐**：当用户从产品获得更多价值时，它会上升
- **领先性**：预测长期业务成功（营收、留存）
- **可影响**：产品团队的工作能直接影响它
- **可理解**：公司所有人都能理解其含义和重要性

**按产品类型举例**：
- 协作工具：每周有 3 名以上成员贡献的活跃团队数
- 市场平台：每周完成的交易笔数
- SaaS 平台：每周完成核心工作流的活跃用户数
- 内容平台：每周的深度阅读/观看时长
- 开发者工具：每周使用该工具的部署次数

### L1 指标（健康指标）
共同呈现产品健康全貌的 5-7 个指标。映射到用户生命周期的关键阶段：

**获取**：新用户是否在找到产品？
- 新注册数或试用开始数（量级和趋势）
- 注册转化率（访客转化为注册用户）
- 渠道构成（新用户从哪里来）
- 每次获取成本（付费渠道）

**激活**：新用户是否到达了价值时刻？
- 激活率：完成预测留存的关键动作的新用户比例
- 激活时间：从注册到激活需要多长时间
- 引导完成率：完成入门步骤的用户比例
- 首次价值时刻：用户第一次体验核心产品价值的时机

**使用**：活跃用户是否在获取价值？
- DAU / WAU / MAU：不同时间窗口内的活跃用户
- DAU/MAU 比（粘性）：月活用户中每天回访的比例
- 核心行为频率：用户执行最重要动作的频率
- 会话深度：每次会话中用户完成的内容量
- 功能采用率：使用核心功能的用户比例

**留存**：用户是否持续回访？
- D1、D7、D30 留存：1 天、7 天、30 天后回访的用户比例
- 队列留存曲线：每个注册队列留存随时间的变化
- 流失率：每周期流失的用户或营收比例
- 召回率：流失用户中重新回来的比例

**变现**：价值是否转化为营收？
- 转化率：免费转付费（适用于 freemium）
- MRR / ARR：每月或每年经常性收入
- ARPU / ARPA：每用户或每账户平均营收
- 扩展营收：来自现有客户的营收增长
- 净营收留存率：包含扩张和收缩的营收留存

**满意度**：用户对产品的感受如何？
- NPS：净推荐值
- CSAT：客户满意度评分
- 支持工单量和解决时间
- 应用商店评分和评论情感

### L2 指标（诊断指标）
用于调查 L1 指标变化的详细指标：

- 各步骤漏斗转化率
- 功能级使用量和采用率
- 按细分维度拆分（按方案、公司规模、地域、用户角色）
- 性能指标（页面加载时间、错误率、API 延迟）
- 特定内容的参与度（哪些功能、页面或内容类型驱动参与）

## 常见产品指标

### DAU / WAU / MAU
**衡量内容**：在一天、一周或一月内执行了有效动作的独立用户数。

**关键决策**：
- 什么算"活跃"？登录？页面浏览？核心动作？谨慎定义——不同定义讲述不同的故事。
- 哪个时间窗口最重要？每日使用的产品（消息、邮件）看 DAU；每周使用的产品（项目管理）看 WAU；低频使用的产品（税务软件、旅行预订）看 MAU。

**使用方式**：
- DAU/MAU 比（粘性）：高于 0.5 表明形成了日常使用习惯；低于 0.2 说明使用频率较低。
- 趋势比绝对数更重要。活跃使用是在增长、持平还是下降？
- 按用户类型细分。高频用户和普通用户的行为差异很大。

### 留存
**衡量内容**：在 X 时期开始使用的用户中，在 Y 时期仍活跃的比例。

**常见留存时间窗口**：
- D1（次日）：首次体验是否足以让用户回来？
- D7（一周）：用户是否养成了使用习惯？
- D30（一个月）：用户是否被长期留住？
- D90（三个月）：这个用户是否真正持久？

**如何使用留存**：
- 按队列绘制留存曲线。关注：初始流失（激活问题）、持续下降（使用问题）或趋于平稳（好迹象——你有稳定的留存基础）。
- 跨时间对比队列。新队列的留存是否优于旧队列？这意味着产品改进在奏效。
- 按激活行为细分留存。完成入门的用户 vs 未完成的用户；使用了功能 X 的用户 vs 未使用的用户。

### 转化
**衡量内容**：从一个阶段进入下一阶段的用户比例。

**常见转化漏斗**：
- 访客到注册
- 注册到激活（关键价值时刻）
- 免费到付费（试用转化）
- 试用到付费订阅
- 月付到年付

**如何使用转化**：
- 绘制完整漏斗，衡量每步的转化率
- 找出最大的流失点——这些是改善杠杆最大的机会
- 按来源、方案、用户类型细分转化率。不同细分群体的转化差异很大。
- 随时间追踪转化率。迭代产品体验后，转化率是否在提升？

### 激活
**衡量内容**：新用户中首次体验到产品核心价值时刻的比例。

**定义激活**：
- 对比留存用户与流失用户。留存用户执行了而流失用户未执行的是什么动作？
- 激活事件应对长期留存有强预测力
- 应在第一次会话或最初几天内可完成
- 示例：创建了第一个项目、邀请了队友、完成了第一个工作流、接入了一个集成

**如何使用激活**：
- 追踪每个注册队列的激活率
- 衡量激活时间——越快通常越好
- 构建引导用户到达激活时刻的入门流程
- A/B 测试入门流程，衡量对留存（而非仅对激活率）的影响

## 目标设定框架

### OKR（目标与关键结果）

**目标**：定性的、鼓舞人心的目标，描述你想要实现什么。
- 振奋人心且令人印象深刻
- 有时间限制（季度或年度）
- 指向性的，而非具体的指标

**关键结果**：告诉你是否实现了目标的量化指标。
- 具体且可衡量
- 有明确目标值的时间限制
- 基于结果，而非输出
- 每个目标对应 2-4 个关键结果

**示例**：
```
目标：让产品成为日常工作流中不可或缺的一部分

关键结果：
- 将 DAU/MAU 比从 0.35 提升至 0.50
- 将新用户的 D30 留存率从 40% 提升至 55%
- 3 个核心工作流的任务完成率超过 80%
```

### OKR 最佳实践
- 设定有挑战性但可实现的 OKR。激进 OKR 的目标是完成 70%。
- 关键结果应衡量结果（用户行为、业务成果），而非输出（发布功能、完成任务）。
- OKR 不宜过多。2-3 个目标，每个对应 2-4 个关键结果已经足够。
- OKR 应让人感到不适。如果你确信能全部完成，说明目标还不够大胆。
- 在周期中期复盘 OKR。如果某些关键结果明显落后，调整资源投入。
- 在周期末诚实评分。0.0-0.3 = 未达成，0.4-0.6 = 有进展，0.7-1.0 = 达成。

### 设定指标目标
- **基准线**：当前值是多少？需要可靠的基准线才能设定目标。
- **对标值**：同类产品能做到什么？行业基准提供参考。
- **趋势**：当前趋势是什么？如果指标已经在以每月 5% 的速度改善，6% 的目标并不大胆。
- **投入力度**：你在这方面投入了多少资源？更大的赌注应对应更大胆的目标。
- **置信度**：你对完成目标有多大把握？设定"确保完成"（高置信度）和"冲刺目标"（大胆）。

## 指标复盘节奏

### 每周指标快检
**目的**：快速发现问题，监控实验，掌握产品健康动态。
**时长**：15-30 分钟。
**参与者**：产品经理，可能还有工程负责人。

**复盘内容**：
- 北极星指标：当前值，周环比变化
- 核心 L1 指标：是否有值得关注的变动
- 活跃实验：结果和统计显著性
- 异常点：是否有意外的峰值或下跌
- 告警：是否有触发监控告警的情况

**行动**：如发现异常，立即调查。否则记录下来，继续推进。

### 每月指标复盘
**目的**：深入分析趋势，评估目标进展，了解战略含义。
**时长**：30-60 分钟。
**参与者**：产品团队、核心干系人。

**复盘内容**：
- 完整 L1 指标记分卡，含月环比趋势
- 季度 OKR 目标的进展情况
- 队列分析：新队列表现是否优于老队列？
- 功能采用率：近期上线功能表现如何？
- 细分分析：不同用户群体之间是否有差异？

**行动**：确定 1-3 个需要调查或投入的领域。如果指标揭示了新信息，更新优先级。

### 季度业务回顾
**目的**：从战略高度评估产品表现，为下一季度设定目标。
**时长**：60-90 分钟。
**参与者**：产品、工程、设计、管理层。

**复盘内容**：
- 本季度 OKR 评分
- 本季度所有 L1 指标的趋势分析
- 同比对比
- 竞争背景：市场变化和竞争对手动态
- 哪些有效，哪些无效

**行动**：设定下季度 OKR。根据数据调整产品策略。

## 仪表板设计原则

### 高效的产品仪表板
一个好的仪表板应能让"产品运行得如何？"这个问题一目了然地得到回答。

**原则**：

1. **从问题出发，而非从数据出发**。这个仪表板支持什么决策？从决策倒推设计。

2. **信息层级**。最重要的指标应该视觉上最突出。北极星在顶部，L1 指标次之，L2 指标支持下钻。

3. **上下文比数字更重要**。没有上下文的数字没有意义。始终展示：当前值、对比（上期、目标、基准）、趋势方向。

4. **更少指标，更多洞察**。50 个指标的仪表板没人能用。聚焦于 5-10 个最重要的指标。其他指标放入详细报告。

5. **统一时间周期**。仪表板上的所有指标使用相同时间周期。混用日度和月度指标会造成混乱。

6. **可视化状态指示**。用颜色直观显示健康度：
   - 绿：达标或改善
   - 黄：需要关注或持平
   - 红：未达标或下降

7. **可行动性**。仪表板上的每个指标都应该是团队可以影响的。如果无法对其采取行动，它就不属于产品仪表板。

### 仪表板布局

**第一行**：北极星指标，带趋势线和目标值。

**第二行**：L1 指标记分卡——每个核心指标的当前值、变化、目标和状态。

**第三行**：核心漏斗或转化指标——每步流失的可视化漏斗图。

**第四行**：近期实验和发布——活跃 A/B 测试、近期功能发布的早期指标。

**底部/下钻**：L2 指标、细分分析、用于深入调查的详细时间序列。

### 仪表板反模式
- **虚荣指标**：总是上升但不代表健康的指标（历史总注册数、总页面浏览量）
- **指标过多**：需要滚动才能看完的仪表板。如果一屏放不下，删减指标
- **无对比**：没有上下文的原始数字（当前值，没有上期或目标）
- **陈旧仪表板**：数月未更新或未复盘的指标
- **输出仪表板**：衡量团队活动（关闭工单数、合并 PR 数）而非用户和业务成果
- **一刀切仪表板**：高管、产品经理和工程师需要不同的视角。没有放之四海而皆准的方案

### 告警
为需要立即关注的指标设置告警：

- **阈值告警**：指标低于或高于关键阈值（错误率 > 1%，转化率 < 5%）
- **趋势告警**：指标在多天/周内持续下降
- **异常告警**：指标显著偏离预期范围

**告警卫生**：
- 每条告警都应该是可行动的。如果无法采取行动，不要设告警。
- 定期审查和调整告警。误报过多会让人忽视所有告警。
- 为每条告警指定负责人。触发时谁来响应？
- 设置适当的严重级别。不是所有事情都是 P0。

## 输出格式

记分卡使用表格。使用清晰的状态指示器。摘要保持简洁——读者应能在 30 秒内获取核心信息。

## 提示

- 从"那又怎样"开始——这份指标复盘中最重要的事情是什么？以它开篇。
- 没有上下文的绝对数字没有价值。始终展示对比（vs 上期、vs 目标、vs 基准）。
- 归因时要谨慎。相关性不等于因果性。如果某个指标发生了变化，承认原因的不确定性。
- 细分分析常常揭示聚合指标掩盖的重要差异。总体持平的数字可能隐藏了一个细分市场增长、另一个萎缩的情况。
- 并非所有指标变动都值得关注。小幅波动是噪音。关注有意义的变化。
- 如果指标未达目标，不要只是报告缺口——给出建议。
- 指标复盘应推动决策。如果复盘没有导致至少一个行动，它就没有发挥价值。
