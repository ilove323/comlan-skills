# 竞品简报

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

为一个或多个竞争对手或功能领域创建竞品分析简报。

## 使用方式

```
/competitive-brief $ARGUMENTS
```

## 工作流

### 1. 明确分析范围

向用户询问：
- **竞争对手**：要分析哪些具体竞争对手？还是要对比各竞争对手在某个功能领域的表现？
- **聚焦点**：全产品对比、特定功能领域、定价/套餐、市场策略，还是定位分析？
- **决策背景**：这份分析将用于哪个决策？（产品策略、销售赋能、投资人/董事会材料、功能优先级）

### 2. 调研

**通过网络搜索**：
- 产品页面和功能列表
- 定价页面和套餐设计
- 近期产品发布、博客文章和更新日志
- 媒体报道和分析师报告
- 客户评价（G2、Capterra、TrustRadius）
- 招聘信息（战略方向的信号）
- 社交媒体和社区讨论

如果 **~~knowledge base** 已接入：
- 搜索现有竞品分析文档
- 查找胜负分析报告或销售作战卡
- 提取历史竞品研究

如果 **~~chat** 已接入：
- 在销售或产品频道中搜索竞品相关讨论
- 查找近期涉及竞争对手的交易反馈

### 3. 生成简报

#### 竞争对手概览
对每个竞争对手：
- 公司概要：成立时间、规模、融资/营收（如公开）、目标市场
- 产品定位：如何介绍自己、目标用户群
- 近期动态：产品发布、融资、合作伙伴关系、客户获取

#### 功能对比
对分析相关的核心领域逐一对比能力。评分标准和矩阵模板见下方**功能对比矩阵**部分。

#### 定位分析
分析每个竞争对手如何定位自身——目标客户、品类定义、核心差异点和价值主张。详见下方**定位分析框架**中的定位陈述模板和信息架构层次。

#### 优势与劣势
对每个竞争对手：
- **优势**：真正出色的地方；客户称赞的内容
- **劣势**：不足之处；客户抱怨的内容
- 保持客观，基于证据——不要轻视竞争对手，也不要夸大其弱点

#### 机会点
基于分析结果：
- 竞品供给存在哪些空白可以利用？
- 客户需要但没有人提供的功能是什么？
- 竞争对手做出了哪些我们不认同的赌注？
- 哪些市场变化可能对我们的方向有利？

#### 威胁
- 竞争对手正在大力投入哪些方向？
- 哪些竞争动作可能冲击我们的市场地位？
- 我们在哪些方面最脆弱？
- "噩梦情景"的竞争动作会是什么样的？

#### 战略含义
将分析结论与产品策略挂钩：
- 基于这份分析，我们应该构建、加速或降低优先级的是什么？
- 在哪些领域应该差异化，在哪些领域应该追平？
- 定位或信息传递需要如何调整？
- 后续应持续关注哪些方面？

### 4. 后续跟进

生成简报后：
- 询问用户是否需要深入某个具体章节
- 提供为高管准备的单页摘要
- 提供针对竞争交易的销售作战卡
- 提供"如何在对抗[竞争对手]时胜出"指南
- 提供竞争动态监控方案

## 竞争格局绘制

### 识别竞争集合
从多个层面定义竞争对手：

**直接竞争对手**：以相同方式为相同用户解决相同问题的产品。
- 客户在选型时会与你主动比较的产品
- 出现在你的商机、客户比价和评测网站上的产品

**间接竞争对手**：以不同方式解决相同问题的产品。
- 解决相同用户需求但路径不同（如电子表格 vs 专用项目管理工具）
- 包括"不购买"——有时竞争对手是"什么都不做"或用手工流程

**周边竞争对手**：目前不竞争但可能进入的产品。
- 拥有相似技术、客户群或分发渠道、可能扩张到你所在领域的公司
- 可能将你的功能作为特性加入自身产品的大平台
- 正在攻克某个细分市场、可能扩张至你核心市场的初创公司

**替代解决方案**：用户解决底层需求的完全不同方式。
- 雇人而不是买软件
- 用通用工具（Excel、邮件）代替专用工具
- 将流程完全外包出去

### 格局图
在有意义的维度上标注竞争对手位置：

**常用轴线**：
- 宽度 vs 深度（套件 vs 单点产品）
- 中小企业 vs 企业（市场定位）
- 自助 vs 销售驱动（市场策略）
- 简单 vs 强大（产品复杂度）
- 横向 vs 纵向（通用 vs 行业专用）

选择能揭示与你所在市场相关的战略定位差异的轴线。正确的轴线能让竞争动态一目了然。

### 持续监控格局
随时间追踪竞争动态：
- 产品发布和功能更新（更新日志、博客文章、新闻稿）
- 定价和套餐变化
- 融资轮次和收购
- 关键岗位招聘（透露战略方向）
- 客户获取与流失（尤其是你自己的胜负情况）
- 分析师和评测报道
- 合作伙伴关系公告

## 功能对比矩阵

### 构建功能对比
1. **定义能力领域**：将功能按对购买方有意义的职能类别分组（而非你的内部架构）。使用购买方在评估时使用的类别。
2. **列出具体能力**：在每个领域下，列出要对比的具体功能或能力。
3. **为每个竞争对手评分**：使用统一的评分标准。

### 评分标准选项

**简单版（大多数情况推荐）**：
- 强：市场领先能力。功能丰富，执行优秀。
- 尚可：基本功能可用。能完成任务但无差异化。
- 弱：存在但有限制。明显存在缺口或执行较差。
- 缺失：不具备此能力。

**详细版（深度对比时使用）**：
- 5：同类最佳。是其他竞品争相学习的标杆。
- 4：强。功能完善，执行优秀。
- 3：尚可。满足基本需求，无差异化。
- 2：有限。存在但有明显缺口。
- 1：最低限度。几乎无功能或处于早期测试阶段。
- 0：缺失。不可用。

### 对比矩阵模板
```
| 能力领域 | 我们的产品 | 竞争对手 A | 竞争对手 B |
|----------------|-------------|-------------|-------------|
| [领域 1]       |             |             |             |
|   [功能 1]     | 强          | 尚可        | 缺失        |
|   [功能 2]     | 尚可        | 强          | 弱          |
| [领域 2]       |             |             |             |
|   [功能 3]     | 强          | 强          | 尚可        |
```

### 功能对比提示
- 基于真实产品体验、客户反馈和评测评分，而非营销宣传
- 功能存在连续的质量谱系。"有功能 X"不如"X 做得有多好"更有价值
- 按目标客户最在意的内容加权，而非按功能总数
- 定期更新——功能对比会很快过时
- 坦诚承认竞争对手的领先之处。总是展示自己赢的对比是没有可信度的
- 为每个能力领域加注"为什么重要"。并非所有功能对购买方都同等重要

## 定位分析框架

### 定位陈述分析
对每个竞争对手，提取其定位：

**模板**：对于[目标客户]中有[需求/问题]的用户，[产品]是一个[品类]，能够[核心收益]。与[竞争对手/替代方案]不同，[产品][核心差异点]。

**定位来源**：
- 首页标题和副标题
- 应用商店或评测网站上的产品描述
- 销售演示文稿（有时由潜在客户泄露或分享）
- 分析师简报材料
- 财报电话会议用语（上市公司）

### 信息架构分析
每个竞争对手如何传递价值？

**第一层——品类**：他们定义了什么品类？（CRM、项目管理、协作平台）
**第二层——差异点**：在该品类内有何不同？（AI 驱动、一体化、开发者优先）
**第三层——价值主张**：他们承诺的成果是什么？（更快成交、更快交付产品、永不错过截止日期）
**第四层——证明点**：他们提供哪些证据？（客户 Logo、数据指标、奖项、案例研究）

### 定位空白与机会
寻找：
- **无主的定位**：竞品均未占据但对购买方有价值的主张
- **拥挤的定位**：每个竞品都在说、已失去意义的主张
- **新兴定位**：市场变化驱动的新价值主张（AI、远程办公、合规）
- **脆弱的定位**：竞品声称但实际无法完全兑现的承诺

## 胜负分析方法论

### 开展胜负分析
胜负分析揭示你真正赢单和丢单的原因，是最具可行性的竞争情报。

**数据来源**：
- 销售团队的 CRM 记录（可立即获取，但存在偏差）
- 决策后不久的客户访谈（最有价值，偏差最小）
- 流失客户问卷或离职访谈
- 丢单客户问卷

### 胜负分析访谈问题
**赢单时**：
- 你试图解决什么问题？
- 你评估了哪些替代方案？（揭示竞争集合）
- 你为何选择我们而不是其他方案？
- 什么差点让你选择别人？
- 我们需要失去什么才会让你考虑其他方案？

**丢单时**：
- 你试图解决什么问题？
- 你最终选择了什么？为什么？
- 我们的产品在哪里不足？
- 我们可以做什么不同的事？
- 你未来会重新考虑我们吗？在什么条件下？

### 分析胜负数据
- 随时间追踪胜负原因。规律在变化吗？
- 按交易类型细分：企业 vs 中小企业、新签 vs 扩张、行业垂直
- 识别赢单和丢单的前三至五大原因
- 区分产品原因（功能、质量）和非产品原因（定价、品牌、关系、时机）
- 按竞争对手计算竞争胜率：在涉及每个竞争对手的交易中，你赢了多少比例？

### 常见胜负规律
- **功能缺口**：竞品拥有你缺失的特定能力，是阻断交易的关键因素
- **集成优势**：竞品与购买方已在使用的工具有集成
- **定价结构**：不总是更便宜——有时是不同的定价模式（按座位 vs 按用量）更适合
- **现任优势**：购买方因切换成本过高而保留现有产品
- **销售执行**：更好的演示、更快的响应速度、更相关的案例研究
- **品牌/信任**：购买方选择了更安全或更知名的方案

## 市场趋势识别

### 趋势识别来源
- **行业分析师报告**：Gartner、Forrester、IDC 的市场规模和趋势研究
- **风险投资动向**：VC 在投什么？投资主题预示聪明资金看到了哪些机会
- **会议主题**：行业活动聚焦于什么？哪些话题吸引了最多观众？
- **技术变迁**：新平台、API 或能力，带动了新产品品类的出现
- **监管变化**：制造需求或机会的新法规
- **客户行为变化**：用户预期如何演变？（移动优先、AI 辅助、隐私意识）
- **人才流动**：顶尖人才去哪里了？什么技能有需求？

### 趋势分析框架
对识别出的每个趋势：

1. **正在发生什么变化**：清晰具体地描述趋势
2. **为什么是现在**：驱动变化的原因是什么？（技术、法规、行为、经济）
3. **谁受到影响**：哪些客户群或市场品类？
4. **时间线是什么**：这是正在发生、1-2 年内，还是 3-5 年后？
5. **对我们的含义是什么**：应如何影响我们的产品策略？
6. **竞争对手在做什么**：他们如何应对这一趋势？

### 区分信号与噪音
- **信号**：有行为数据、持续投资、监管行动或客户需求支撑的趋势
- **噪音**：仅靠媒体炒作、会议热点或竞品公告（无客户牵引力）支撑的趋势
- 用自己的客户数据验证趋势：你的客户是否也在提出这些需求或经历这些变化？
- 警惕"年度热点"炒作周期。很多在行业讨论中占主导地位的趋势，数年内对你的客户不会有实质影响

### 战略应对选项
对每个重要趋势：
- **引领**：提前投入，尝试定义品类或方向。高风险，高回报。
- **快速跟随**：等待客户需求的早期信号，然后快速行动。风险较低，但差异化难度更高。
- **观望**：追踪趋势但暂不投入。设置触发行动的条件。
- **忽略**：明确决定这一趋势与你的策略无关。记录原因。

正确的应对取决于：你的竞争地位、客户群、资源储备，以及趋势的演进速度。

## 输出格式

功能对比使用表格。各章节使用清晰的标题。战略含义章节保持简洁且可行动——这是读者获取价值的核心所在。

## 提示

- 坦诚对待竞品优势。轻视竞争对手会让分析失去价值。
- 聚焦客户在意的，而非产品团队在意的。客户不关心架构优雅性。
- 定价对比很难公平进行。注明注意事项（套餐不同、按座位 vs 按用量、企业定制定价）。
- 招聘信息是被低估的竞争情报。竞品招聘 ML 工程师是战略方向的信号。
- 客户评论是金矿。它们揭示了真实用户的真实喜好与不满，没有营销滤镜。
- 竞品分析最有价值的部分是"那又怎样"——战略含义。不要跳过这部分。
- 竞品分析有有效期。注明日期，并标注哪些领域变化较快。
