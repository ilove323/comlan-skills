# 技能2：风险评估与假设分析

## 概述
本技能帮助科学家通过严格的假设分析，系统性地识别、量化和管理项目风险。目标不是消除风险——无风险的项目往往只是渐进式的——而是命名它、量化它，并持续努力逐步消除它。本技能直接建立在技能1的问题构思文档之上。

## 核心原则

**"不要回避风险；与风险为友。"**

问题选择中最重要的概念是双轴评估：
- **X轴：** 成功的可能性
- **Y轴：** 成功后的影响力

本技能聚焦于X轴，帮助用户通过系统性风险分析将项目向右移动。

## 为何重要

一个高风险假设在>2年内无法验证的项目是有问题的。需要多个奇迹才能成功的项目应该避免或改进。人类的倾向是待在安全的局部空间，横向推进，推迟面对存在性风险——就像把头埋在沙子里的鸵鸟。本技能帮助用户正面迎对风险。

## 技能工作流程

### 第一阶段：提取项目假设（10-15分钟）

首先，AI助手应从技能1中收集用户项目信息：

1. **项目摘要**（来自技能1）：
   - 生物学问题
   - 技术方法
   - 新颖之处

2. **项目周期：**
   - 预计需要多长时间？（月/年）
   - 用户的角色是什么？（研究生、博后、PI、创业者）

3. **初步风险感知：**
   - 这个项目让用户夜不能寐的是什么？
   - 最可怕的假设是什么？

### 第二阶段：全面列举假设

AI助手应与用户合作，从项目启动到结束列出所有假设。假设分为两类：

#### A类：关于生物学现实的假设
这些是关于世界事实的假设，要么是真的要么是假的。在项目期间不会改变。

**示例：**
- 存在目前未知的新细胞类型
- 特定基因调控正在研究的过程
- 两种蛋白质发生物理相互作用
- 信号通路在目标生物体中发挥功能
- 生物效应量是可检测的

#### B类：关于技术能力的假设
这些是关于技术是否能完成所需工作的假设。随着方法的改进，这些假设在项目期间可以改变。

**示例：**
- 特定细胞类型可以被分离
- 测序将产生高质量数据
- 实验检测具有足够的通量
- 计算分析能够区分信号与噪声
- 基因编辑在该系统中有效

**AI助手应询问：**
1. 这个项目成功的生物学前提是什么？
2. 技术需要实现什么功能？
3. 实验设计中有哪些内置假设？
4. 关于分析——它能提供所需结果吗？
5. 如果一切顺利，能验证发现吗？
6. 发现是否可被解读和有意义？

### 第三阶段：风险评分（假设分析表）

对于每个假设，AI助手应帮助用户给出两个分数：

#### 风险等级（1-5分）：
- **1** = 很可能为真/有效（>90%置信度）
- **2** = 可能（70-90%置信度）
- **3** = 不确定（40-70%置信度）
- **4** = 不太可能（10-40%置信度）
- **5** = 非常不可能（<10%置信度）

#### 验证时间（月）：
多久才能知道这个假设是否有效？

**关键规则：**
1. 保持极度诚实——尝试说服自己是错的，而不是对的
2. 区分生物学假设与技术假设
3. 考虑技术假设是否可能随时间改进
4. 注意哪些假设依赖于更早的假设成功

### 第四阶段：风险概况评估

完整的表格准备好后，AI助手应分析风险概况：

#### 识别红色警示：
1. **晚期高风险问题：** 风险等级4-5的假设在>18个月后才能验证
2. **多重奇迹：** 超过2-3个风险等级4-5的假设
3. **依赖链：** 高风险假设依次叠加
4. **鸵鸟模式：** 从低风险工作开始，同时回避高风险测试

#### 绿色信号：
1. **早期通过/不通过：** 最高风险假设在<6个月内可验证
2. **多候选方案：** 项目可以通过多种不同结果取得成功
3. **优雅降级：** 如果假设X失败，假设Y提供替代路径
4. **风险分布：** 高风险假设在时间线上均衡分布

**经验法则：** 如果你有一个三年后才能验证的风险等级5假设，选择另一个项目。

### 第五阶段：风险缓解策略

对于每个高风险假设（等级4-5），AI助手应帮助制定缓解策略：

#### 策略1：将高风险测试提前
**问题：** 能否设计一个更快、更粗略的测试，回答大部分所需内容？

**示例：** 与其等待2年验证新细胞类型是否存在，考虑：
- 使用现有标志物作为代理
- 先在更简单的模型系统中测试
- 使用计算预测提高置信度

#### 策略2：多候选方案方法
**问题：** 能否并行测试多个候选方案以提高成功概率？

**示例：** 与其：
- 测试一种激酶 → 测试10种激酶的面板
- 构建一种工程生物体 → 构建并测试文库
- 追求一个治疗靶点 → 追求3个相关靶点

#### 策略3：重新框架问题
**问题：** 能否调整项目范围以减少关键假设，同时保持影响力？

**示例：**
- **原始：** 识别新的肠内分泌细胞类型（高风险：它们可能不存在）
- **重新框架：** 更好地表征已知但不完全理解的细胞类型（较低风险）

#### 策略4：更换系统
**问题：** 是否有不同的生物学系统，具有相似的科学价值但技术风险较低？

**示例：**
- **原始：** 肠上皮（难以进行遗传操作）
- **替代：** 肝脏（存在更简单的遗传操作选项）

#### 策略5：添加互补方法
**问题：** 能否添加并行方法来降低主要假设的风险？

**示例：**
- 在scRNA-seq基础上添加空间转录组学
- 这提供了生物地理背景，并在更早的时间点验证细胞类型的存在

### 第六阶段：通过/不通过实验设计

对于前3个最高风险假设，AI助手应帮助设计关键的通过/不通过实验：

**对于每个，具体说明：**
1. **问题：** 究竟在测试什么？
2. **实验：** 可能的最直接测试（即使粗略）
3. **成功标准：** 什么结果意味着"通过"？
4. **失败应对：** 什么结果意味着"转向"或"停止"？
5. **时间表：** 多快能运行这个测试？
6. **资源：** 需要什么？

**关键原则：** 直接切入关键的通过/不通过实验。不要只从简单的事情开始——风险点不会消失。

### 第七阶段：文献验证

AI助手应搜索PubMed以帮助校准风险评估：

**搜索：**
1. **先例：** 有没有人做过类似的事？（降低技术风险）
2. **生物学证据：** 系统的已知信息是什么？（为生物学风险提供信息）
3. **技术基准：** 这些方法在实践中效果如何？
4. **相邻成功案例：** 有没有人解决了相关问题？

**向用户询问的问题：**
- 哪些特定搜索有助于校准风险？
- 是否有特别能说明假设的论文？
- 是否有技术基准需要查阅？

### 第八阶段：修订项目计划

基于风险分析，AI助手应帮助制定修订计划：

#### 选项A：降低当前计划的风险
- 重新排列实验，尽早测试高风险假设
- 添加互补方法
- 设计多候选方案策略

#### 选项B：重新框架项目
- 调整范围同时保持影响力
- 改变生物学系统
- 修改技术方法

#### 选项C：选择不同的项目
有时诚实的答案是："这需要太多奇迹。"在投入多年之前知道这一点是有价值的。

## 产出文件

AI助手应产出一份**2页的风险评估文档**：

### 第1页：假设分析表

| 假设 | 类型* | 风险† | 时间‡ | 备注 |
|------|-------|-------|-------|------|
| [假设1] | 生物/技术 | 1-5 | X月 | [评分依据] |
| [假设2] | 生物/技术 | 1-5 | X月 | [评分依据] |
| ... | ... | ... | ... | ... |

*生物=生物学现实，技术=技术能力
†风险：1=很可能，5=非常不可能
‡验证时间（月）

#### 风险概况摘要：
- **假设总数：** X
- **高风险（4-5）：** X个假设
- **晚期高风险（>18个月）：** X个假设
- **关键路径：** [识别依赖假设链]
- **整体评估：** [绿/黄/红灯及解释]

### 第2页：风险缓解计划

#### 前3个高风险假设：
对每个：
1. **假设：** [清晰陈述]
2. **当前风险等级和时间表：** X（风险）在Y个月
3. **风险存在原因：** [解释]
4. **缓解策略：** [来自上述策略1-5]
5. **通过/不通过实验：**
   - 实验设计
   - 成功标准
   - 时间表
   - 失败时的应对措施

#### 修订项目时间表：
```
第0-6个月：   [早期通过/不通过实验]
第6-12个月：  [基于通过/不通过结果]
第12-18个月： [...]
第18个月以后：[...]
```

#### 应急计划：
- **如果假设X失败：** [B计划]
- **如果假设Y失败：** [C计划]
- **多种成功路径：** [项目如何以不同方式取得成功]

#### 决策点：
- **第X个月：** 评估[假设A、B] → 通过/转向/停止决定
- **第Y个月：** 评估[假设C、D] → 通过/转向/停止决定

## 实际案例

### 案例1：肠内分泌细胞的scRNA-Seq

**识别的高风险假设：**
1. "新细胞类型可以通过实验验证"（风险5，24个月）
2. "敲除将产生生物学相关表型"（风险5，30个月）

**问题：** 24个月以上有两个风险5假设 = 红色警示

**应用的缓解措施：**
- 重新框架为研究已知但表征不完整的细胞（将风险5→3）
- 改用肝脏代替肠道（改善验证时间表：30→18个月）
- 添加空间转录组学（在16个月提供更早的验证检查点）

### 案例2：慢性肾病的细菌疗法

**识别的高风险假设：**
"可以确定导致效应的关键尿毒素"（风险4，时间表不明确）

**问题：** 关键假设，解决路径不清晰

**应用的缓解措施：**
- 聚焦于已知的主要毒素（IS和PCS），而非发现新毒素
- 添加平行轨道：测试多种毒素候选
- 设计以学习毒素身份作为结果的研究（多种成功路径）

## 牢记的关键原则

1. **尝试说服自己你是错的：** 目标是批判性评估，而非确认偏见。

2. **忽略除关键风险点外的一切：** 不要被简单任务分散注意力。高风险假设不会消失。

3. **早期且频繁：** 在最早可行的时刻设计通过/不通过实验。

4. **坦诚承认风险：** 在展示想法时，承认风险会使你的论据更有说服力，而非更少。

5. **没有风险，就没有兴趣：** 目标不是零风险——而是被理解、被量化、可管理的风险。

6. **风险可以改变：** 技术假设可能随方法进步而改善。将此纳入规划。

7. **比较风险概况：** 并行评估多个项目以比较风险概况，做出更好的选择。

8. **警惕鸵鸟模式：** 你是否在回避那个可怕的实验？这是人类本性，但也是关键的失败模式。

## 警告信号

**警告信号包括：**
- 风险等级5的假设在>2年后才能验证
- 超过3个风险等级4-5的假设
- 最高风险假设在时间线的末尾
- 为高风险假设"可能会成功"进行合理化
- 计划"从简单的事情开始"同时回避风险测试
- 无法阐明清晰的通过/不通过标准

**状态良好的指标：**
- 最高风险测试在前6个月进行
- 存在多条成功路径
- 有明确的计划来应对关键假设失败
- 风险分布在时间线上
- 测试假设，而非确认希望

## 入门

AI助手应从第一阶段开始，询问：
1. 来自技能1的项目摘要
2. 项目时间表预期
3. 用户最担心这个项目的什么

AI助手与用户将共同构建一份严格的风险评估，通过帮助避免在存在无法克服障碍的项目上浪费数年时间，大幅提高成功的可能性。

---

*请记住：在风险分析上投入时间是科学家能做的最有价值的投资。一个被充分理解的风险概况能够让你充满信心地推进，或者清晰地转向——两者都是有价值的结果。*
