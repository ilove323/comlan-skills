# 技能3：优化函数选择

## 概述
本技能帮助科学家阐明如何评估他们的项目，并定义成功的含义。技能2关注成功的可能性（X轴），而本技能关注成功后的影响力（Y轴）。关键洞见：**价值在于信仰体系的眼中**——价值创造框架必须被明确陈述并加以引领。

## 核心原则

**"选择正确的优化函数。"**

不同类型的项目应该用不同的指标来评估。学员与导师、作者与审稿人之间的冲突一个常见根源，是对项目属于哪个类别的误解。根本原因往往是未能清晰阐明评估标准。

## 基本事实

以下的默认状态：
1. 每个新发现的默认是**无关紧要**
2. 每个新技术的默认是**不被使用**
3. 每家公司的默认是**失败**

科学家必须积极对抗这些默认状态，通过选择正确的指标，并在至少一个轴上取得高分。

## 技能工作流程

### 第一阶段：项目分类（5分钟）

首先，AI助手应确定用户追求的是哪种类型的项目：

**问题1：主要目标是什么？**
A. 理解生物学如何运作（基础知识）
B. 实现新的实验或能力（工具/技术）
C. 解决实际问题（发明/应用）
D. 其他（请描述）

**问题2：3-5年后的"成功"是什么样子的？**
- 1-2句话描述理想结果

**问题3：谁在乎这是否成功？**
- 子领域的学术研究人员？
- 跨领域的更广泛科学界？
- 临床医生或从业者？
- 行业合作伙伴或公司？
- 公众或特定社区？
- 以上所有？

根据答案，AI助手应帮助确定正确的优化函数。

### 第二阶段：理解三个主要框架

#### 框架1：基础科学
**轴：** 我们学到了多少？× 研究对象有多普遍/基础？

**理念：** 在任一轴上取得高分即可产生重大影响。不需要两者兼顾。

**示例：**
- **高普遍性，中等学习：** 核糖体停顿复合物
  - 更新了翻译（基本过程）的理解
  - 因为翻译是普遍的，所以得分高

- **中等普遍性，高学习：** Oxytricha生殖系核
  - 基因组特技可能在其他生物体中不常见
  - 但优雅的图谱在学到多少上得分高
  - 可能产生基因组编辑工具（额外收获）

- **两轴都高（里程碑式）：** RNA干扰、生物分子凝聚物
  - 这些很罕见——不要期望每个项目都能到达这里
  - 但目标是在至少一个轴上得分高

**关键问题：**
- 这适用于多少系统/生物体？
- 它是否更新了对基本过程的理解？
- 教科书需要改写吗？
- 这开启了什么新问题？

#### 框架2：技术开发
**轴：** 将被多大范围使用？× 对于应用有多关键？

**理念：** 同样，任一轴上的高分即已足够。

**示例：**
- **广泛使用，非关键：** BLAST
  - 用于无数项目
  - 很少是关键工具，但巨大的累积影响

- **不广泛使用，高度关键：** 冷冻电子断层扫描
  - 太复杂，难以广泛采用
  - 但生成无法通过其他方式获得的惊人数据
  - 当需要它时，没有其他方法可行

- **两轴都高（改变游戏规则）：**
  - GFP、CRISPR、AlphaFold（著名的）
  - 但也包括：慢病毒递送、细胞分选、大规模并行测序
  - 我们无法想象没有这些技术的生活

**关键问题：**
- 有多少实验室会采用这个？
- 对于多大比例的实验，这是关键的使能技术？
- 以前不可能的事情现在变得可能了？
- 实施有多难？

**关键规则：** 一个不会被广泛使用，且对应用也不关键的工具，可能不值得构建。

#### 框架3：典型发明/应用
**轴：** 有多少好处？× 对多少人？

**理念：** 适用于转化工作、俭朴科学、全球健康。

**示例：**
- Foldscope：纸质显微镜，全球数百万学生可以使用
- 被忽视的热带病干预：每100美元调整质量的生命年
- 医疗器械：能够获得治疗的患者数量

**关键问题：**
- 这解决了什么问题？
- 有多少人有这个问题？
- 如果你解决了它，他们的生活会好多少？
- 每人帮助的成本是多少？

### 第三阶段：选择和阐明你的框架

基于第一阶段的回答，让我帮助你选择：

**如果你选择了A（基础知识）：** → 基础科学框架
**如果你选择了B（使实验成为可能）：** → 技术开发框架
**如果你选择了C（解决实际问题）：** → 发明框架

**现在，让我们明确：**

1. **陈述你的框架：** "这个项目应该作为[基础科学/技术开发/发明]来评估。"

2. **定义你的轴：**
   - X轴测量：[具体指标]
   - Y轴测量：[具体指标]

3. **阐述你的理由：**
   - X轴得分（低/中/高）：[你的评估+理由]
   - Y轴得分（低/中/高）：[你的评估+理由]

4. **阈值检查：**
   - 你在一个轴上得分至少是中高吗？
   - 如果两者都是低中，你有问题了

### 第四阶段：替代或自定义指标

有时标准框架不适用。使用自定义指标的示例：

**替代指标示例：**
- **俭朴科学：** 低/中等收入国家有多少儿童获得显微镜检查机会？
- **被忽视的疾病：** 每100美元投资节省的调整质量生命年
- **可持续性：** 防止的等效CO₂吨数 × 成本效益
- **公平性：** 差异指标的减少 × 受影响人数

**何时提出替代指标：**
- 你的工作针对特定的未被满足需求
- 标准指标忽略了你的核心价值主张
- 你在一个没有既定规范的新兴领域工作
- 你的工作跨越传统边界

**如何提出替代指标：**
1. 解释为何标准指标不足
2. 清晰定义你提出的指标
3. 提供价值创造指数（两个轴）
4. 展示你的项目在这些轴上的得分

### 第五阶段：比较评估

即使绝对影响难以估计，比较评估也是有价值的：

**练习：比较3个相关项目**

对你的项目和两个替代方案（来自文献或假设）：

| 项目 | 框架 | X轴得分 | Y轴得分 | 整体 |
|------|------|---------|---------|------|
| 你的 | [类型] | [低/中/高]+理由 | [低/中/高]+理由 | [评估] |
| 替代1 | [类型] | [低/中/高]+理由 | [低/中/高]+理由 | [评估] |
| 替代2 | [类型] | [低/中/高]+理由 | [低/中/高]+理由 | [评估] |

**比较问题：**
- 如果它们都成功，哪个最有影响？
- 哪个具有最佳的风险调整影响？
- 你是否在追求最佳选项？
- 如果不是，为什么？（有时有充分的理由：资源、专长、时机）

### 第六阶段：避免指标不匹配

**常见不匹配：**

#### 不匹配1：基础科学与技术评估
**场景：** 你在做基础生物学，但审稿人问"这将被多大范围使用？"

**问题：** 他们在用技术指标评估基础科学

**解决方案：** 明确框架为基础科学。引导："这更新了我们对[基本过程]的理解，这在[许多系统]中是保守的。"

#### 不匹配2：技术与基础科学评估
**场景：** 你在构建一个工具，但审稿人问"我们对生物学学到了多少？"

**问题：** 他们在用基础科学指标评估技术

**解决方案：** 明确框架为技术开发。引导："这使目前不可能的实验成为可能，[X]个实验室需要它来进行[Y]应用。"

#### 不匹配3：类别内混淆
**场景：** 你的基础科学是特定的但深入的，但审稿人想要广泛的普遍性

**问题：** 他们认为两个轴都是必需的，而不是任一即可

**解决方案：** 明确承认："虽然这可能不是普遍的，但机制洞察的深度在学习轴上得分高。"

#### 不匹配4：时间跨度不匹配
**场景：** 你在做长期基础研究，但审稿人想要即时影响

**问题：** 对影响何时实现的不同价值观

**解决方案：** 明确阐明你的时间跨度，并提供类似时间线的历史例子

### 第七阶段：价值体系讨论

这是AI助手明确讨论用户信仰体系的地方：

**反思问题：**

1. **是什么驱动用户？**
   - 发现和理解？
   - 使他人成为可能？
   - 解决问题？
   - 构建事物？

2. **什么会让用户感到自豪？**
   - 在Cell/Nature/Science上发论文？
   - 被数百个实验室使用的工具？
   - 到达患者的治疗？
   - 开创新领域？

3. **用户希望如何被记住？**
   - "发现了X"
   - "构建了Y，使Z成为可能"
   - "解决了问题W"
   - "培养了后来取得成就的学生..."

4. **谁的认可最重要？**
   - 领域内特定的资深科学家？
   - 跨领域的更广泛社区？
   - 使用工具的从业者？
   - 生活得到改善的人？

**没有错误答案——但一致性很重要：**
- 项目应该符合用户的价值体系
- 评估框架应该符合项目类型
- 沟通应该以框架为引导

### 第八阶段：文献基准测试

AI助手应使用PubMed对用户所在领域的影响进行基准测试：

**搜索应包括：**

1. **影响示例：** 用户认为高影响的论文
   - 他们使用了什么框架（明确的或隐含的）？
   - 他们在轴上得分如何？
   - 是什么让他们成功？

2. **类似项目：** 相似的方法或系统
   - 它们是如何被评估的？
   - 它们达到了什么影响？
   - 从它们的框架中能学到什么？

3. **领域期望：** 该领域的典型情况
   - 基础科学论文常见吗？
   - 技术开发受重视吗？
   - 什么水平的影响是"足够好的"？

**向用户询问的问题：**
- 应该分析哪些论文作为基准？
- 什么搜索词能捕捉该领域的影响示例？
- 有没有特定期刊或作者的框架值得效仿？

### 第九阶段：沟通策略

一旦框架选定，以下是如何以此为引导：

#### 在报告中：
**开场框架（在前2张幻灯片内）：**
- "这项工作的目标是理解[通用系统Y]中的[基本过程X]" → 基础科学
- "我们在开发一项技术，将使[社区Y]能够进行[关键实验X]" → 技术
- "这项发明解决了影响[N]人的[问题X]" → 应用

#### 在论文中：
**摘要结构：**
- 通过用词隐含地表明你的框架
- 基础科学："揭示"、"证明"、"表明"
- 技术："使...成为可能"、"提供"、"使...得以"
- 应用："解决"、"针对"、"改善"

#### 在基金申请中：
**更广泛影响部分：**
- 明确命名你的评估框架
- 提供双轴评估
- 用证据对每个轴打分

#### 与导师/委员会：
**一致性对话：**
- "我想确保我们在如何评估这项工作上保持一致"
- "我把这看作是[框架]，在[轴1]上得分[X]，在[轴2]上得分[Y]"
- "你同意吗，还是你看法不同？"
- "这很重要，因为..." [解释下游含义]

## 实际案例

### 案例1：核糖体停顿（基础科学）
- **框架：** 基础科学
- **X轴（普遍性）：** 高——翻译是普遍的
- **Y轴（学习）：** 中——一个质量控制系统的机制
- **评估：** 单凭普遍性高分 = 重大影响
- **沟通：** "更新了我们对翻译质量控制的理解"

### 案例2：BLAST（技术）
- **框架：** 技术开发
- **X轴（广泛使用）：** 非常高——几乎所有分子生物学家都使用
- **Y轴（关键）：** 低中——有用但很少是必不可少的
- **评估：** 极广的使用范围 = 巨大的累积影响
- **沟通：** "使跨所有生物数据库的快速序列比较成为可能"

### 案例3：冷冻电子断层扫描（技术）
- **框架：** 技术开发
- **X轴（广泛使用）：** 低——复杂、昂贵、专业化
- **Y轴（关键）：** 非常高——生成其他方式无法获得的数据
- **评估：** 对小众的极端关键性 = 高影响
- **沟通：** "使在天然细胞环境中3D可视化分子机器成为可能"

### 案例4：Foldscope（发明）
- **框架：** 发明（自定义指标）
- **X轴（好处）：** 中——功能性显微镜
- **Y轴（人数）：** 非常高——全球数百万学生
- **评估：** 大范围 × 适度效用 = 对教育具有变革性
- **沟通：** "使全球教育的显微镜大众化"

## 牢记的关键原则

1. **价值在于信仰体系的眼中：** 让你的体系变得明确。

2. **以你的指标为引导：** 不要假设其他人与你有相同的框架。

3. **任一轴即已足够：** 你不需要两者兼顾——只需在其中一个轴上得分高。

4. **尽早阐明：** 在投入2年之前与导师讨论。

5. **避免默认状态：** 积极对抗无关紧要/不被使用的状态。

6. **比较，而非绝对：** 即使粗略的比较也比忽略影响要好。

7. **一致的沟通：** 你的用词应该表明你的框架。

8. **项目与价值观匹配：** 生命太短暂，不能做不一致的工作。

## 警告信号

**警告信号包括：**
- 无法阐明适用哪个框架
- 两个轴都得分低
- 项目类型和评估框架不匹配
- 用户和导师有不同的框架但没有讨论过
- 对工具使用基础科学指标，反之亦然
- 从未明确讨论过影响评估

**状态良好的指标：**
- 清晰陈述的优化函数
- 至少在一个轴上得分中高
- 框架符合项目类型
- 与关键利益相关者一致
- 沟通清晰地表明框架
- 与可比较工作进行基准测试

## 入门

AI助手应从第一阶段开始询问：
1. 主要目标是什么？（A/B/C/D）
2. 3-5年后的成功是什么样子的？
3. 谁在乎这是否成功？
