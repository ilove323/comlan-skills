# /vendor-review

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

对供应商进行结构化分析，涵盖成本、风险、绩效和适配度。

## 用法

```
/vendor-review $ARGUMENTS
```

## 需要您提供的信息

- **供应商名称**：您在评估谁？
- **背景**：新供应商评估、续约决策还是比较？
- **详情**：合同条款、定价、提案文件或当前绩效数据

## 评估框架

### 成本分析（总拥有成本）
- 总拥有成本（不只是许可费）
- 实施和迁移成本
- 培训和入职成本
- 持续支持和维护
- 退出成本（数据迁移、合同终止）

### 风险评估
- 供应商财务稳定性
- 安全和合规状况
- 集中风险（单一供应商依赖）
- 合同锁定和退出条款
- 业务连续性和灾难恢复

### 绩效指标
- SLA 合规性
- 支持响应时间
- 正常运行时间和可靠性
- 功能交付节奏
- 客户满意度

### 比较矩阵
在比较供应商时，生成涵盖以下内容的并排矩阵：定价、功能、集成、安全、支持、合同条款和参考案例。

## 连接器可用时

- 搜索现有供应商评估、合同和绩效审查
- 获取采购政策和审批门槛

- 获取当前合同条款、支出历史和续约日期
- 与现有供应商协议对比定价

## 使用技巧

1. **上传提案** — AI助手可以从供应商文件中提取定价、条款和 SLA。
2. **比较供应商** — "比较供应商 A 与供应商 B"可获得并排分析。
3. **包含当前支出** — 对于续约，了解您现在的支出有助于评估价格变化。
