# 风险评估

系统性地识别、评估运营风险并制定缓解计划。

## 风险评估矩阵

| | 低影响 | 中影响 | 高影响 |
|---|-----------|---------------|-------------|
| **高可能性** | 中 | 高 | 严重 |
| **中可能性** | 低 | 中 | 高 |
| **低可能性** | 低 | 低 | 中 |

## 风险类别

- **运营**：流程故障、人员缺口、系统中断
- **财务**：预算超支、供应商成本增加、收入影响
- **合规**：违反监管规定、审计发现、政策违规
- **战略**：市场变化、竞争威胁、技术转变
- **声誉**：客户影响、公众认知、合作伙伴关系
- **安全**：数据泄露、访问控制失败、第三方漏洞

## 风险登记格式

对于每项风险，记录：
- **描述**：可能发生什么
- **可能性**：高 / 中 / 低
- **影响**：高 / 中 / 低
- **风险级别**：严重 / 高 / 中 / 低
- **缓解措施**：我们正在采取什么措施降低可能性或影响
- **负责人**：谁负责管理此风险
- **状态**：开放 / 已缓解 / 已接受 / 已关闭

## 输出

生成带有具体、可操作缓解措施的优先级风险登记表。重点关注可控且重大的风险。
