# /capacity-plan

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

分析团队容量并规划资源分配。

## 用法

```
/capacity-plan $ARGUMENTS
```

## 需要您提供的信息

- **团队规模和角色**：您有哪些人员？
- **当前工作量**：他们在做什么？（从项目追踪器上传或描述）
- **即将到来的工作**：下季度有什么计划？
- **约束条件**：预算、招聘时间表、技能要求

## 规划维度

### 人员
- 可用人数和技能
- 当前分配和利用率
- 计划招聘及时间表
- 外包商和供应商容量

### 预算
- 按类别的运营预算
- 项目专项预算
- 差异追踪
- 预测与实际对比

### 时间
- 项目时间表和依赖关系
- 关键路径分析
- 缓冲和应急规划
- 截止日期管理

## 利用率目标

| 角色类型 | 目标利用率 | 说明 |
|-----------|-------------------|-------|
| IC / 专业人员 | 75-80% | 为响应性工作和成长留出空间 |
| 管理者 | 60-70% | 管理开销、会议、一对一沟通 |
| 值班 / 支持 | 50-60% | 中断驱动型工作难以预测 |

## 常见陷阱

- 按 100% 利用率规划（无应对意外的缓冲空间）
- 忽视会议负担和上下文切换成本
- 未考虑假期、节假日和病假时间
- 将所有工时等同对待（创造性工作 ≠ 行政工作）

## 连接器可用时

如果已连接 **~~project tracker**：
- 自动获取当前工作量和工单分配情况
- 显示每人的即将到来的冲刺或季度承诺

如果已连接 **~~calendar**：
- 将休假、节假日和定期会议负担纳入考虑
- 计算每人的实际可用工时

## 使用技巧

1. **包含所有工作** — 日常运营、项目、支持、会议。人员不是 100% 用于项目工作的。
2. **规划缓冲空间** — 目标利用率 80%。100% 意味着没有应对意外的余地。
3. **定期更新** — 容量计划很快就会过时。每月审查一次。
