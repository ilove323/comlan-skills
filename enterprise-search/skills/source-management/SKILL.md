# 来源管理

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

了解哪些来源可用，帮助连接新来源，并管理来源的查询方式。

## 检查可用来源

通过检查可用工具确定哪些 MCP 来源已连接。每个来源对应一组 MCP 工具：

| 来源 | 关键能力 |
|--------|-----------------|
| **~~chat** | 搜索消息，读取频道和讨论串 |
| **~~email** | 搜索消息，读取单封邮件 |
| **~~cloud storage** | 搜索文件，获取文档内容 |
| **~~project tracker** | 搜索任务，预输入搜索 |
| **~~CRM** | 查询记录（账户、联系人、商机） |
| **~~knowledge base** | 语义搜索，关键词搜索 |

如果工具前缀可用，该来源已连接且可搜索。

## 指导用户连接来源

当用户搜索但几乎没有或没有来源连接时：

```
您当前已连接 [N] 个来源：[列表]。

要扩展搜索范围，您可以在 MCP 设置中连接额外来源：
- ~~chat — 消息、讨论串、频道
- ~~email — 邮件、对话、附件
- ~~cloud storage — 文档、表格、幻灯片
- ~~project tracker — 任务、项目、里程碑
- ~~CRM — 账户、联系人、商机
- ~~knowledge base — Wiki 页面、知识库文章

连接的来源越多，搜索结果越完整。
```

当用户询问未连接的特定工具时：

```
[工具名称] 当前未连接。添加方法：
1. 打开您的 MCP 设置
2. 添加 [工具] MCP 服务器配置
3. 根据提示进行身份验证

连接后，它将自动包含在后续搜索中。
```

## 来源优先级排序

不同的查询类型受益于先搜索特定来源。使用这些优先级来权重结果，而非跳过来源：

### 按查询类型

**决定查询**（"我们决定了什么..."）：
```
1. ~~chat（发生决定的对话）
2. ~~email（决定确认、公告）
3. ~~cloud storage（会议纪要、决定日志）
4. Wiki（如果决定有文档记录）
5. 任务追踪器（如果决定记录在任务中）
```

**状态查询**（"...的状态是什么"）：
```
1. 任务追踪器（~~project tracker — 权威状态）
2. ~~chat（实时讨论）
3. ~~cloud storage（状态文档、报告）
4. ~~email（状态更新邮件）
5. Wiki（项目页面）
```

**文档查询**（"...的文档在哪里"）：
```
1. ~~cloud storage（主要文档存储）
2. Wiki / ~~knowledge base（知识库）
3. ~~email（通过邮件共享的文档）
4. ~~chat（在频道中共享的文档）
5. 任务追踪器（链接到任务的文档）
```

**人员查询**（"谁在做..."/谁了解..."）：
```
1. ~~chat（消息作者、频道成员）
2. 任务追踪器（任务负责人）
3. ~~cloud storage（文档作者、协作者）
4. ~~CRM（账户负责人、联系人）
5. ~~email（邮件参与者）
```

**事实/政策查询**（"我们关于...的政策是什么"）：
```
1. Wiki / ~~knowledge base（官方文档）
2. ~~cloud storage（政策文档、手册）
3. ~~email（政策公告）
4. ~~chat（政策讨论）
```

### 默认优先级（一般查询）

查询类型不明确时：
```
1. ~~chat（最高容量，最实时）
2. ~~email（正式沟通）
3. ~~cloud storage（文档和文件）
4. Wiki / ~~knowledge base（结构化知识）
5. 任务追踪器（工作事项）
6. CRM（客户数据）
```

## 速率限制感知

MCP 来源可能有速率限制。优雅地处理它们：

### 检测

速率限制响应通常表现为：
- HTTP 429 响应
- 提及"rate limit"、"too many requests"或"quota exceeded"的错误消息
- 受限或延迟的响应

### 处理

当来源受到速率限制时：

1. **不要立即重试** — 尊重限制
2. **继续其他来源** — 不要阻塞整个搜索
3. **告知用户**：
```
注意：[来源] 暂时受到速率限制。以下结果来自
[其他来源]。您可以几分钟后重试以包含 [来源]。
```
4. **对于摘要** — 如果在扫描中途受到速率限制，说明限制前覆盖的时间范围

### 预防

- 避免不必要的 API 调用——在查询前检查来源是否可能有相关结果
- 尽可能使用针对性查询而非广泛扫描
- 对于摘要，API 支持时批量请求
- 缓存感知：如果刚刚运行了搜索，避免立即重新运行相同查询

## 来源健康状态

在会话期间跟踪来源可用性：

```
来源状态：
  ~~chat:          ✓ 可用
  ~~email:          ✓ 可用
  ~~cloud storage:  ✓ 可用
  ~~project tracker:        ✗ 未连接
  ~~CRM:     ✗ 未连接
  ~~knowledge base:       ⚠ 速率受限（2 分钟后重试）
```

报告搜索结果时，包含哪些来源被搜索，使用户了解答案的范围。

## 添加自定义来源

企业搜索插件适用于任何 MCP 连接的来源。随着新的 MCP 服务器推出，可以将其添加到 `.mcp.json` 配置中。搜索和摘要命令将根据可用工具自动检测并包含新来源。

添加新来源：
1. 将 MCP 服务器配置添加到 `.mcp.json`
2. 如需要，进行身份验证
3. 该来源将自动包含在后续搜索中
