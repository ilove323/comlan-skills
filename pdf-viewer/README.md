# PDF 工具插件

在实时交互式查看器中查看、注释和签署PDF。标注合同、带可视反馈地填写表单、盖批准印章、放置签名——然后下载已注释的副本。

## 功能介绍

- **打开PDF** — 来自本地文件或学术来源（arXiv、bioRxiv等）
- **协作注释** — AI助手逐节提议高亮、注释和印章；你在查看器中审阅每批操作
- **填写表单** — 逐字段引导填写，带实时预览
- **签署文档** — 在页面上放置签名/首字母缩写图像
- **盖批准印章** — APPROVED、DRAFT、CONFIDENTIAL或任意自定义标签
- **下载** — 从查看器工具栏导出已注释的PDF

## 命令列表

| 命令 | 功能说明 |
|---------|-------------|
| `/pdf-viewer:open` | 在交互式查看器中打开PDF |
| `/pdf-viewer:annotate` | 逐步浏览文档，提议并应用标记，共同审阅 |
| `/pdf-viewer:fill-form` | 交互式填写PDF表单字段 |
| `/pdf-viewer:sign` | 在页面上放置签名或首字母缩写图像 |

## 何时使用此插件而非直接读取PDF

此插件适用于**交互式可视化工作流**——当你想查看文档、进行标注并下载带注释副本时。

如果你只需要AI助手**摘要或提取PDF文本**，无需使用此插件。AI助手可以原生读取PDF文件，对于纯内容提取速度更快。

## 工作原理

此插件使用**本地MCP服务器**（`@modelcontextprotocol/server-pdf`），通过`npx`在你的机器上运行。无需API密钥或远程服务——PDF服务器在插件加载时自动启动。

## 系统要求

- Node.js >= 18
- 访问远程PDF（arXiv等）需要网络连接

## 支持的PDF来源

- 本地文件（工作目录中的文件路径）
- [arXiv](https://arxiv.org) — `/abs/` URL自动转换为PDF
- 任何直接的HTTPS PDF链接（bioRxiv、Zenodo、OSF等——使用PDF直链，而非落地页）

## 安装方法

将插件目录复制到 `~/.openclaw/skills/`，或使用命令行：

```bash
openclaw skills install pdf-viewer
```

## 签名免责声明

`/pdf-viewer:sign` 在页面上放置**可视**签名图像。这不是经认证的加密数字签名。如需具有法律效力的电子签名，请使用专用签署服务。
