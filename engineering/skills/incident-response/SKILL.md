# /incident-response

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

从发现到复盘，全流程管理线上事故。

## 使用方式

```
/incident-response $ARGUMENTS
```

## 模式

```
/incident-response new [description]     # 新建事故
/incident-response update [status]       # 发布状态更新
/incident-response postmortem            # 根据事故数据生成复盘报告
```

如果未指定模式，询问当前事故处于哪个阶段。

## 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                       故障响应                                    │
├─────────────────────────────────────────────────────────────────┤
│  第一阶段：分级                                                   │
│  ✓ 评估严重等级（SEV1-4）                                        │
│  ✓ 确定受影响的系统和用户                                         │
│  ✓ 分配角色（事故指挥官、沟通负责人、响应者）                     │
│                                                                   │
│  第二阶段：沟通                                                   │
│  ✓ 起草内部状态更新                                               │
│  ✓ 起草客户沟通通知（如需要）                                     │
│  ✓ 建立作战室和更新节奏                                           │
│                                                                   │
│  第三阶段：缓解                                                   │
│  ✓ 记录已采取的缓解措施                                           │
│  ✓ 追踪事件时间线                                                 │
│  ✓ 确认问题已解决                                                 │
│                                                                   │
│  第四阶段：复盘                                                   │
│  ✓ 无责复盘文档                                                   │
│  ✓ 时间线重建                                                     │
│  ✓ 根本原因分析（5个为什么）                                      │
│  ✓ 含责任人的行动项                                               │
└─────────────────────────────────────────────────────────────────┘
```

## 严重等级分类

| 等级 | 判定标准 | 响应时间 |
|-------|----------|---------------|
| SEV1 | 服务完全中断，所有用户受影响 | 立即响应，全员投入 |
| SEV2 | 主要功能降级，大量用户受影响 | 15 分钟内响应 |
| SEV3 | 次要功能问题，部分用户受影响 | 1 小时内响应 |
| SEV4 | 外观或低影响问题 | 下个工作日处理 |

## 沟通指引

按固定节奏提供清晰、基于事实的更新。内容包括：正在发生什么、影响范围、正在采取什么措施、下次更新时间。

## 如果已接入连接器

如果 **~~monitoring** 已接入：
- 拉取告警详情和指标
- 展示受影响指标的图表

如果 **~~incident management** 已接入：
- 在 PagerDuty/Opsgenie 中创建或更新事故
- 通知值班响应人员

如果 **~~chat** 已接入：
- 向事故频道推送状态更新
- 创建作战室频道

## 提示

1. **立即开始记录** — 不要等信息齐全再写。随着了解深入持续更新。
2. **保持更新内容基于事实** — 我们知道什么、做了什么、接下来做什么。不要推测。
3. **复盘不追责** — 聚焦系统和流程，而不是个人。
