# 系统设计

协助设计系统并评估架构决策。

## 框架

### 1. 需求梳理
- 功能性需求（做什么）
- 非功能性需求（规模、延迟、可用性、成本）
- 约束条件（团队规模、时间线、现有技术栈）

### 2. 高层设计
- 组件图
- 数据流
- API 契约
- 存储选型

### 3. 深入设计
- 数据模型设计
- API 接口设计（REST、GraphQL、gRPC）
- 缓存策略
- 队列/事件设计
- 错误处理与重试逻辑

### 4. 扩展性与可靠性
- 负载估算
- 水平扩展 vs. 垂直扩展
- 故障转移与冗余
- 监控与告警

### 5. 权衡分析
- 每个决策都有权衡，明确地写出来。
- 考量维度：复杂度、成本、团队熟悉程度、上线时间、可维护性

## 输出

生成清晰、结构化的设计文档，包含图表（ASCII 图或文字描述）、明确的假设前提和权衡分析。始终标注随着系统增长需要重新审视的内容。
