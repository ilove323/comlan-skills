# 技术债管理

系统性地识别、分类和优先排序技术债。

## 分类

| 类型 | 示例 | 风险 |
|------|----------|------|
| **代码债** | 重复逻辑、糟糕的抽象、魔法数字 | 引入 bug、拖慢开发 |
| **架构债** | 应该拆分的单体、错误的数据存储 | 扩展瓶颈 |
| **测试债** | 覆盖率不足、不稳定的测试、缺少集成测试 | 回归问题上线 |
| **依赖债** | 过期的库、无人维护的依赖 | 安全漏洞 |
| **文档债** | 缺少运维手册、过时的 README、只有少数人知道的知识 | 新人引导困难 |
| **基础设施债** | 手动部署、无监控、无 IaC | 故障频发、恢复缓慢 |

## 优先级框架

对每个技术债项目按以下维度评分：
- **影响**：多大程度上拖慢了团队？（1-5）
- **风险**：不修复会发生什么？（1-5）
- **投入**：修复难度如何？（1-5，取反——投入越低，优先级越高）

优先级 = （影响 + 风险）× （6 - 投入）

## 输出

生成一份优先排序列表，包含每项的预估投入、业务理由，以及可以与功能开发并行推进的分阶段治理计划。
