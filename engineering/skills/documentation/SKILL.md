# 技术文档

为不同受众和用途撰写清晰、易于维护的技术文档。

## 文档类型

### README
- 这是什么，为何存在
- 快速上手（5 分钟内跑通第一个成功案例）
- 配置与使用说明
- 贡献指南

### API 文档
- 请求/响应示例的接口参考
- 认证方式与错误码
- 频率限制与分页
- SDK 示例

### 运维手册（Runbook）
- 何时使用本手册
- 前置条件与所需权限
- 分步操作流程
- 回滚步骤
- 升级路径

### 架构文档
- 背景与目标
- 含图表的高层设计
- 关键决策与权衡
- 数据流与集成点

### 新人引导指南
- 环境搭建
- 核心系统及其关联关系
- 常见任务及操作演示
- 遇到问题找谁

## 原则

1. **为读者而写** — 谁在读这份文档，他们需要什么？
2. **最重要的信息放最前面** — 不要把关键内容藏在后面
3. **展示而非叙述** — 代码示例、命令、截图
4. **保持更新** — 过时的文档比没有文档更糟糕
5. **引用而非复制** — 引用其他文档而非直接复制内容
