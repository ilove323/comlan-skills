# /deploy-checklist

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

生成部署前检查清单，在上线前验证准备情况。

## 使用方式

```
/deploy-checklist $ARGUMENTS
```

## 输出格式

```markdown
## 部署检查清单：[服务/版本]
**日期：** [日期] | **部署人：** [姓名]

### 部署前
- [ ] CI 所有测试通过
- [ ] 代码已审查并批准
- [ ] 本次发布中无已知严重 bug
- [ ] 数据库迁移已测试（如适用）
- [ ] 功能开关已配置（如适用）
- [ ] 回滚方案已记录
- [ ] 值班团队已通知

### 部署中
- [ ] 部署到预生产环境并验证
- [ ] 运行冒烟测试
- [ ] 部署到生产环境（如有，使用金丝雀发布）
- [ ] 监控错误率和延迟 15 分钟
- [ ] 验证核心用户流程

### 部署后
- [ ] 确认各项指标正常
- [ ] 更新 Release Notes / Changelog
- [ ] 通知干系人
- [ ] 关闭相关工单

### 回滚触发条件
- 错误率超过 [X]%
- P50 延迟超过 [X]ms
- [核心用户流程] 失败
```

## 自定义

告诉我你的部署情况，我会定制检查清单：
- "我们使用功能开关" → 添加开关验证步骤
- "这包含数据库迁移" → 添加迁移专项检查
- "这是破坏性的 API 变更" → 添加消费方通知步骤

## 如果已接入连接器

如果 **~~source control** 已接入：
- 拉取发布差异和变更列表
- 验证所有 PR 已批准并合并

如果 **~~CI/CD** 已接入：
- 自动检查构建和测试状态
- 在部署前确认 Pipeline 已全部通过

如果 **~~monitoring** 已接入：
- 从当前基线预填回滚触发阈值
- 设置部署后的指标监控

## 提示

1. **每次部署前都执行** — 即使是例行部署。检查清单能防止"我忘了……"
2. **一次配置，持续复用** — 告诉我你的技术栈，我会记住你的部署流程。
3. **包含回滚标准** — 在部署之前就决定何时回滚，而不是部署过程中。
