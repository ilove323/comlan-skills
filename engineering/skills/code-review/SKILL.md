# /code-review

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

从安全性、性能、正确性和可维护性角度对代码变更进行结构化审查。

## 使用方式

```
/code-review <PR 链接或文件路径>
```

审查提供的代码变更：@$1

如果没有指定文件或链接，询问审查什么。

## 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                      代码审查                                     │
├─────────────────────────────────────────────────────────────────┤
│  单独使用（始终有效）                                             │
│  ✓ 粘贴差异、PR 链接或指定文件                                    │
│  ✓ 安全审计（OWASP Top 10、注入、认证）                           │
│  ✓ 性能审查（N+1、内存泄漏、复杂度）                              │
│  ✓ 正确性（边界情况、错误处理、竞态条件）                          │
│  ✓ 代码风格（命名、结构、可读性）                                  │
│  ✓ 带代码示例的可行动建议                                         │
├─────────────────────────────────────────────────────────────────┤
│  工具增强（接入工具后）                                           │
│  + 代码托管：自动拉取 PR 差异                                     │
│  + 项目追踪：将发现关联到工单                                     │
│  + 知识库：对照团队编码规范检查                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 审查维度

### 安全性
- SQL 注入、XSS、CSRF
- 认证和授权缺陷
- 代码中的密钥或凭据
- 不安全的反序列化
- 路径遍历
- 服务端请求伪造（SSRF）

### 性能
- N+1 查询
- 不必要的内存分配
- 算法复杂度（热路径中的 O(n²)）
- 缺失数据库索引
- 无界查询或循环
- 资源泄漏

### 正确性
- 边界情况（空输入、null、溢出）
- 竞态条件和并发问题
- 错误处理和传播
- 差一错误（off-by-one）
- 类型安全

### 可维护性
- 命名清晰度
- 单一职责
- 重复代码
- 测试覆盖率
- 非明显逻辑的文档说明

```markdown
## 代码审查：[PR 标题或文件]

### 摘要
[1-2 句话，概述变更内容和整体质量]

### 关键问题
| # | 文件 | 行号 | 问题 | 严重程度 |
|---|------|------|-------|----------|
| 1 | [文件] | [行] | [描述] | 🔴 严重 |

### 优化建议
| # | 文件 | 行号 | 建议 | 类别 |
|---|------|------|------------|----------|
| 1 | [文件] | [行] | [描述] | 性能 |

### 做得好的地方
- [正向观察]

### 审查结论
[批准 / 请求修改 / 需要讨论]
```

## 提示

1. **提供背景** — "这是一条热路径"或"这处理 PII"有助于聚焦审查重点。
2. **说明关注点** — "聚焦安全性"可以收窄审查范围。
3. **包含测试** — AI助手也会检查测试覆盖率和质量。
