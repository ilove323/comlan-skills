# /standup

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

整合各工具的近期活动，生成日会更新。

## 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                          日会                                     │
├─────────────────────────────────────────────────────────────────┤
│  单独使用（始终有效）                                             │
│  ✓ 告诉我你做了什么，我来帮你整理结构                             │
│  ✓ 格式化为日会格式（昨天 / 今天 / 阻塞）                        │
│  ✓ 保持简洁、以行动为导向                                        │
├─────────────────────────────────────────────────────────────────┤
│  工具增强（接入工具后）                                           │
│  + 代码托管：近期提交和 PR                                        │
│  + 项目追踪：工单状态变更                                         │
│  + 沟通工具：相关讨论和决策                                       │
│  + CI/CD：构建和部署状态                                          │
└─────────────────────────────────────────────────────────────────┘
```

## 我需要的信息

**方式 A：让我来拉取**
如果你的工具已接入，直接说 `/standup`，我会自动收集所有信息。

**方式 B：告诉我你做了什么**
"处理了认证迁移，审查了 3 个 PR，被 API 频率限制问题阻塞了。"

## 输出

```markdown
## 日会 — [日期]

### 昨天
- [已完成事项（如有则附工单引用）]
- [已完成事项]

### 今天
- [计划事项（如有则附工单引用）]
- [计划事项]

### 阻塞
- [阻塞事项，含上下文及可求助的人]
```

## 如果已接入连接器

如果 **~~source control** 已接入：
- 拉取近期提交和 PR（已创建、已审查、已合并）
- 对代码变更进行高层次总结

如果 **~~project tracker** 已接入：
- 拉取已移入"进行中"或"完成"状态的工单
- 展示即将到来的迭代任务

如果 **~~chat** 已接入：
- 扫描相关讨论和决策
- 标记需要你回复的对话线程

## 提示

1. **每天早上运行** — 养成习惯，不再为日会笔记抓耳挠腮。
2. **补充上下文** — 生成后，为阻塞事项或优先级补充必要的细节说明。
3. **指定分享格式** — 可以让我按 Slack、邮件或团队站会工具的格式输出。
