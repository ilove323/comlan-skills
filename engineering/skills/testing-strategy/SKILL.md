# 测试策略

设计在覆盖率、执行速度和维护成本之间取得平衡的高效测试策略。

## 测试金字塔

```
        /  E2E  \         数量少、速度慢、置信度高
       / Integration \     数量适中、速度中等
      /    Unit Tests  \   数量多、速度快、聚焦单元
```

## 按组件类型制定策略

- **API 接口**：业务逻辑单元测试、HTTP 层集成测试、消费方契约测试
- **数据管道**：输入校验、转换正确性、幂等性测试
- **前端**：组件测试、交互测试、视觉回归、无障碍访问
- **基础设施**：冒烟测试、混沌工程、负载测试

## 测试重点

重点覆盖：核心业务路径、错误处理、边界情况、安全边界、数据完整性。

可跳过：琐碎的 getter/setter、框架代码、一次性脚本。

## 输出

生成测试计划，包含：测什么、每个领域的测试类型、覆盖率目标和示例测试用例。识别现有覆盖的空白点。
