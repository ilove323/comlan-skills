# 工程插件

专为 [OpenClaw](https://openclaw.ai) 设计的软件工程插件，同时支持 Claude Code。覆盖日会、代码审查、架构决策、故障响应、调试和技术文档等工程工作流。适用于任何工程团队——单独使用时提供即时输入，接入代码托管、项目追踪和监控工具后功能全面增强。

## 安装

将插件目录复制到 ~/.openclaw/skills/engineering/
或使用 ClawHub: openclaw skills install engineering

## 命令

通过斜杠命令触发的明确工作流：

| 命令 | 说明 |
|---|---|
| `/standup` | 从最近活动（提交、PR、工单和聊天记录）生成日会更新 |
| `/review` | 审查代码变更——安全性、性能、代码风格和正确性 |
| `/debug` | 结构化调试会话——复现、定位、诊断和修复 |
| `/architecture` | 创建或评估架构决策——ADR 格式，含权衡分析 |
| `/incident` | 运行故障响应工作流——分级、沟通、缓解，并撰写事后复盘 |
| `/deploy-checklist` | 部署前检查清单——验证测试、审查变更、检查依赖，确认回滚方案 |

所有命令均支持**单独使用**（粘贴代码、描述系统、上传文件），接入 MCP 连接器后**功能全面增强**。

## 技能

AI助手在相关场景下自动运用的领域知识：

| 技能 | 说明 |
|---|---|
| `code-review` | 审查代码中的 bug、安全问题、性能和可维护性 |
| `incident-response` | 分级和管理生产故障——状态更新、Runbook、事后复盘 |
| `system-design` | 设计系统和服务——架构图、API 设计、数据建模 |
| `tech-debt` | 识别、分类和优先级排序技术债——制定整治计划 |
| `testing-strategy` | 设计测试策略——单元、集成、端到端覆盖，测试计划创建 |
| `documentation` | 撰写和维护技术文档——README、API 文档、Runbook、入职指南 |

## 工作流示例

### 早间日会

```
/standup
```

如果工具已接入，系统将自动拉取最近的提交、PR 活动和工单更新。否则，请描述你的工作内容，AI助手将帮你整理格式。

### 代码审查

```
/review https://github.com/org/repo/pull/123
```

提供 PR 链接、粘贴差异，或指向文件。获取涵盖安全性、性能、正确性和代码风格的结构化审查报告。

### 调试问题

```
/debug 结账页面用户收到 500 错误
```

通过结构化调试流程逐步排查：复现、定位、诊断、修复。AI助手将帮你系统性地分析。

### 架构决策

```
/architecture 我们的服务间通信应该用消息队列还是直接 API 调用？
```

获得包含选项分析、权衡对比和建议的结构化 ADR。

### 故障响应

```
/incident 支付服务持续返回 503
```

启动故障工作流：评估严重级别、起草沟通内容、追踪时间线，并在问题解决后生成事后复盘文档。

### 部署前检查

```
/deploy-checklist auth-service v2.3.0
```

根据服务和变更内容生成定制化的部署检查清单。

## 单独使用与工具增强对比

每个命令和技能均可在不依赖任何集成的情况下使用：

| 功能 | 单独使用 | 接入工具后增强 |
|-----------------|------------|-------------------|
| 日会更新 | 描述你的工作 | 代码托管、项目追踪、沟通工具 |
| 代码审查 | 粘贴差异或代码 | 代码托管（自动拉取 PR） |
| 调试会话 | 描述问题 | 监控（拉取日志和指标） |
| 架构决策 | 描述系统 | 知识库（查找历史 ADR） |
| 故障响应 | 描述故障 | 监控、故障管理、沟通工具 |
| 部署检查清单 | 描述部署内容 | CI/CD、代码托管 |

## MCP 集成

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](CONNECTORS.md)。MCP 服务器的详细配置说明见 [MCP_SETUP.md](MCP_SETUP.md)。

接入工具获得更丰富的体验：

| 类别 | 示例 | 支持功能 |
|---|---|---|
| **代码托管** | GitHub、GitLab | PR 差异、提交历史、分支状态 |
| **项目追踪** | Linear、Jira、Asana | 工单状态、迭代数据、任务分配 |
| **监控** | Datadog、New Relic | 日志、指标、告警、仪表板 |
| **故障管理** | PagerDuty、Opsgenie | 值班安排、故障追踪、告警通知 |
| **沟通** | Slack、Teams | 团队讨论、站会频道 |
| **知识库** | Notion、Confluence | ADR、Runbook、入职文档 |

详见 [CONNECTORS.md](CONNECTORS.md) 了解完整的集成工具列表。

## 个性化设置

在 `engineering/.openclaw/settings.local.json` 创建本地配置文件进行个性化设置：

```json
{
  "name": "你的姓名",
  "title": "软件工程师",
  "team": "你的团队",
  "company": "你的公司",
  "techStack": ["Python", "TypeScript", "PostgreSQL", "AWS"],
  "defaultBranch": "main",
  "deployProcess": "canary"
}
```

若未配置，插件将在使用时交互式地询问这些信息。
