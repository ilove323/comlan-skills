# /ux-copy

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

为任何界面场景撰写或审查 UX 文案。

## 用法

```
/ux-copy $ARGUMENTS
```

## 需要您提供的信息

- **上下文**：什么页面、流程或功能？
- **用户状态**：用户正在尝试做什么？他们的感受如何？
- **语气**：正式、友好、活泼、令人放心？
- **约束**：字符限制、平台指南？

## 原则

1. **清晰**：确切表达您的意思。不用行话，不含歧义。
2. **简洁**：使用最少的词语传达完整含义。
3. **一致**：相同事物使用相同术语。
4. **有用**：每个词都应帮助用户实现目标。
5. **人性化**：像一个有帮助的人那样写，而不是机器人。

## 文案模式

### CTA
- 以动词开头："开始免费试用"、"保存更改"、"下载报告"
- 具体说明："创建账户" 而非 "提交"
- 标签与结果匹配

### 错误信息
结构：发生了什么 + 为什么 + 如何修复
- "付款被拒。您的银行拒绝了此卡。请尝试其他卡或联系您的银行。"

### 空状态
结构：这是什么 + 为什么为空 + 如何开始
- "暂无项目。创建您的第一个项目以开始与团队协作。"

### 确认对话框
- 明确操作："删除 3 个文件？" 而非 "您确定吗？"
- 描述后果："此操作无法撤销"
- 按钮标注操作："删除文件" / "保留文件" 而非 "确定" / "取消"

### 工具提示
- 简洁、有帮助、永远不说显而易见的话

### 加载状态
- 设定预期，减少焦虑

### 引导
- 逐步展示，每次一个概念

## 声音和语气

根据上下文调整语气：
- **成功**：庆祝但不过分
- **错误**：有同理心且提供帮助
- **警告**：清晰且可操作
- **中性**：信息性且简洁

## 输出格式

```markdown
## UX 文案：[上下文]

### 推荐文案
**[元素]**：[文案]

### 备选方案
| 选项 | 文案 | 语气 | 最适合 |
|--------|------|------|----------|
| A | [文案] | [语气] | [使用场景] |
| B | [文案] | [语气] | [使用场景] |
| C | [文案] | [语气] | [使用场景] |

### 理由
[为什么这个文案有效——用户上下文、清晰度、行动导向]

### 本地化说明
[翻译人员需要了解的内容——需避免的习惯用语、字符扩展、文化背景]
```

## 连接器可用时

如果已连接 **~~knowledge base**：
- 获取您的品牌声音指南和内容风格指南
- 检查现有文案模式和术语标准

如果已连接 **~~design tool**：
- 在 Figma 中查看页面上下文以了解完整用户流程
- 从设计中检查字符限制和布局约束

## 使用技巧

1. **具体说明上下文** — "付款失败时的错误信息" 比 "错误信息" 更好。
2. **分享您的品牌声音** — "我们专业但温暖" 帮助 AI助手匹配您的语气。
3. **考虑用户的情绪状态** — 错误信息需要同理心。成功信息可以庆祝。
