# /design-handoff

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

从设计稿生成全面的开发交接文档。

## 用法

```
/design-handoff $ARGUMENTS
```

生成交接规格：@$1

如果提供了 Figma URL，从 Figma 获取设计稿。否则，根据提供的描述或截图进行处理。

## 需要包含的内容

### 视觉规格
- 精确尺寸（内边距、外边距、宽度）
- 设计令牌引用（颜色、排版、间距）
- 响应式断点和行为
- 组件变体和状态

### 交互规格
- 点击/轻触行为
- 悬停状态
- 过渡和动画（时长、缓动）
- 手势支持（滑动、捏合、长按）

### 内容规格
- 字符限制
- 截断行为
- 空状态
- 加载状态
- 错误状态

### 边缘情况
- 最小/最大内容
- 国际化文本（较长字符串）
- 慢速连接
- 数据缺失

### 无障碍性
- 焦点顺序
- ARIA 标签和角色
- 键盘交互
- 屏幕阅读器播报

## 原则

1. **不要假设** — 如果未指定，开发者会猜测。请指定所有内容。
2. **使用令牌，而非数值** — 引用 `spacing-md` 而非 `16px`。
3. **显示所有状态** — 默认、悬停、激活、禁用、加载、错误、空。
4. **描述原因** — "这在移动端折叠是因为用户主要单手使用" 帮助开发者做出好的判断。

## 输出格式

```markdown
## 交接规格：[功能/页面名称]

### 概述
[此页面/功能的功能，用户背景]

### 布局
[网格系统、断点、响应式行为]

### 使用的设计令牌
| 令牌 | 值 | 用途 |
|-------|-------|-------|
| `color-primary` | #[hex] | CTA 按钮、链接 |
| `spacing-md` | [X]px | 区域间距 |
| `font-heading-lg` | [尺寸/字重/字体] | 页面标题 |

### 组件
| 组件 | 变体 | 属性 | 备注 |
|-----------|---------|-------|-------|
| [组件] | [变体] | [属性] | [特殊行为] |

### 状态和交互
| 元素 | 状态 | 行为 |
|---------|-------|----------|
| [CTA 按钮] | 悬停 | [背景加深 10%] |
| [CTA 按钮] | 加载 | [加载动画，禁用] |
| [表单] | 错误 | [红色边框，下方显示错误信息] |

### 响应式行为
| 断点 | 变化 |
|------------|---------|
| 桌面端 (>1024px) | [默认布局] |
| 平板端 (768-1024px) | [变化内容] |
| 移动端 (<768px) | [变化内容] |

### 边缘情况
- **空状态**：[无数据时显示什么]
- **长文本**：[截断规则]
- **加载中**：[骨架屏或加载动画]
- **错误**：[错误状态外观]

### 动画/动效
| 元素 | 触发 | 动画 | 时长 | 缓动 |
|---------|---------|-----------|----------|--------|
| [元素] | [触发] | [描述] | [ms] | [缓动函数] |

### 无障碍说明
- [焦点顺序]
- [所需 ARIA 标签]
- [键盘交互]
```

## 连接器可用时

如果已连接 **~~design tool**：
- 从 Figma 获取精确尺寸、令牌和组件规格
- 导出资产并生成完整规格文档

如果已连接 **~~project tracker**：
- 将交接文档链接到实施工单
- 为规格的每个部分创建子任务

## 使用技巧

1. **分享 Figma 链接** — AI助手可以获取精确尺寸、令牌和组件信息。
2. **提及边缘情况** — "100 条数据时会怎样？" 帮助 AI助手规格化边界条件。
3. **说明技术栈** — "我们使用 React + Tailwind" 帮助 AI助手给出相关的实施说明。
