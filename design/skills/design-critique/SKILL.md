# /design-critique

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

从多个维度获取结构化设计反馈。

## 用法

```
/design-critique $ARGUMENTS
```

评审设计：@$1

如果提供了 Figma URL，从 Figma 获取设计稿。如果引用了文件，读取它。否则，请用户描述或分享他们的设计。

## 需要您提供的信息

- **设计稿**：Figma URL、截图或详细描述
- **背景**：这是什么？面向谁？处于哪个阶段（探索、完善、最终）？
- **关注点**（可选）："专注于移动端" 或 "专注于引导流程"

## 评审框架

### 1. 第一印象（2 秒）
- 什么最先吸引眼球？这是正确的吗？
- 情绪反应是什么？
- 目的是否立即清晰？

### 2. 可用性
- 用户能完成他们的目标吗？
- 导航是否直觉？
- 交互元素是否明显？
- 是否有不必要的步骤？

### 3. 视觉层次
- 是否有清晰的阅读顺序？
- 是否强调了正确的元素？
- 留白是否有效运用？
- 排版是否创造了正确的层次？

### 4. 一致性
- 是否遵循了设计系统？
- 间距、颜色和排版是否一致？
- 相似元素的行为是否一致？

### 5. 无障碍性
- 色彩对比度
- 触控目标尺寸
- 文字可读性
- 图片的替代文本

## 如何给出反馈

- **具体说明**："CTA 与导航竞争" 而非 "布局令人困惑"
- **解释原因**：将反馈与设计原则或用户需求联系起来
- **提出替代方案**：不只是发现问题，还要提出解决方案
- **承认做得好的**：好的反馈包含正面观察
- **匹配阶段**：早期探索获得不同于最终打磨的反馈

## 输出格式

```markdown
## 设计评审：[设计名称]

### 总体印象
[1-2 句第一反应——什么有效、最大的改进机会是什么]

### 可用性
| 发现 | 严重程度 | 建议 |
|---------|----------|----------------|
| [问题] | 🔴 严重 / 🟡 中等 / 🟢 次要 | [修复方法] |

### 视觉层次
- **最先吸引眼球的**：[元素] — [这是否正确？]
- **阅读流向**：[眼睛如何在布局中移动？]
- **强调**：[是否强调了正确的内容？]

### 一致性
| 元素 | 问题 | 建议 |
|---------|-------|----------------|
| [排版/间距/颜色] | [不一致之处] | [修复方法] |

### 无障碍性
- **色彩对比度**：[关键文本通过/未通过]
- **触控目标**：[尺寸是否足够？]
- **文字可读性**：[字号、行高]

### 做得好的
- [正面观察 1]
- [正面观察 2]

### 优先建议
1. **[影响最大的改动]** — [原因和方式]
2. **[第二优先级]** — [原因和方式]
3. **[第三优先级]** — [原因和方式]
```

## 连接器可用时

如果已连接 **~~design tool**：
- 直接从 Figma 获取设计稿，检查组件、令牌和图层
- 与现有设计系统对比一致性

如果已连接 **~~user feedback**：
- 将设计决策与最近的用户反馈和支持工单交叉对比

## 使用技巧

1. **分享背景** — "这是 B2B SaaS 的结账流程" 帮助 AI助手给出相关反馈。
2. **说明您的阶段** — 早期探索获得与最终打磨不同的反馈。
3. **让 AI助手聚焦** — "只看导航" 能在一个区域获得更深入的分析。
