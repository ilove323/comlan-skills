# 更新命令

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

保持任务列表和记忆的时效性。两种模式：

- **默认：** 从外部工具同步任务，处理过时事项，检查记忆缺口
- **`--comprehensive`：** 深度扫描聊天、邮件、日历、文档——标记遗漏的待办事项并建议新记忆

## 用法

```bash
/productivity:update
/productivity:update --comprehensive
```

## 默认模式

### 1. 加载当前状态

读取 `TASKS.md` 和 `memory/` 目录。如果不存在，建议先运行 `/productivity:start`。

### 2. 从外部来源同步任务

检查可用的任务来源：
- **项目追踪器**（如 Asana、Linear、Jira）（如果 MCP 可用）
- **GitHub Issues**（如果在仓库中）：`gh issue list --assignee=@me`

如果没有可用来源，跳到步骤 3。

**获取分配给用户的任务**（开放/进行中）。与 TASKS.md 对比：

| 外部任务 | TASKS.md 匹配？ | 操作 |
|---------------|-----------------|--------|
| 发现，不在 TASKS.md | 无匹配 | 提议添加 |
| 发现，已在 TASKS.md | 标题匹配（模糊） | 跳过 |
| 在 TASKS.md，不在外部 | 无匹配 | 标记为可能过时 |
| 外部已完成 | 在"进行中"部分 | 提议标记为完成 |

呈现差异，让用户决定添加/完成哪些。

### 3. 处理过时事项

审查 TASKS.md 中的"进行中"任务并标记：
- 截止日期已过的任务
- 在"进行中"超过 30 天的任务
- 没有背景的任务（没有人员、没有项目）

对每个任务进行处理：标记为完成？重新安排？移到"以后再说"？

### 4. 解码任务以填补记忆缺口

对每个任务，尝试解码所有实体（人员、项目、缩写、工具、链接）：

```
任务："将 PSR 发给 Todd，关于 Phoenix 阻塞项"

解码：
- PSR → ✓ 流水线状态报告（在词汇表中）
- Todd → ✓ Todd Martinez（在 people/ 中）
- Phoenix → ? 不在记忆中
```

跟踪完全解码的内容与有缺口的内容。

### 5. 填补缺口

呈现分组的未知术语：
```
AI助手在您的任务中发现了一些没有背景信息的术语：

1. "Phoenix"（来自："将 PSR 发给 Todd，关于 Phoenix 阻塞项"）
   → Phoenix 是什么？

2. "Maya"（来自："与 Maya 同步 API 设计"）
   → Maya 是谁？
```

将答案添加到相应的记忆文件（people/、projects/、glossary.md）。

### 6. 捕获丰富信息

任务通常包含比记忆更丰富的背景。提取并更新：
- 任务中的**链接** → 添加到项目/人员文件
- **状态变化**（"上线完成"）→ 更新项目状态，从 CLAUDE.md 降级
- **关系**（"Todd 对 Maya 提案的签字"）→ 交叉引用人员
- **截止日期** → 添加到项目文件

### 7. 报告

```
更新完成：
- 任务：从项目追踪器新增 +3 个（如 Asana），1 个已完成，2 个已处理
- 记忆：填补 2 个缺口，1 个项目已丰富
- 所有任务已解码 ✓
```

## 综合模式（`--comprehensive`）

默认模式的所有内容，加上最近活动的深度扫描。

### 额外步骤：扫描活动来源

从可用的 MCP 来源收集数据：
- **聊天：** 搜索最近消息，读取活跃频道
- **邮件：** 搜索已发送消息
- **文档：** 列出最近接触的文档
- **日历：** 列出最近和即将到来的事件

### 额外步骤：标记遗漏的待办事项

将活动与 TASKS.md 对比。找出未跟踪的行动项：

```
## 可能遗漏的任务

根据您的活动，以下内容看起来像您尚未记录的待办事项：

1. 来自聊天（1 月 18 日）：
   "我将在周五前发送更新后的原型"
   → 添加到 TASKS.md？

2. 来自会议"Phoenix Standup"（1 月 17 日）：
   您有定期会议但没有活跃的 Phoenix 任务
   → 这里有什么需要处理的吗？

3. 来自邮件（1 月 16 日）：
   "本周我会审查 API 规格"
   → 添加到 TASKS.md？
```

让用户选择要添加哪些。

### 额外步骤：建议新记忆

找出记忆中没有的新实体：

```
## 新人员（不在记忆中）
| 姓名 | 出现频率 | 背景 |
|------|-----------|---------|
| Maya Rodriguez | 12 次提及 | 设计、UI 评审 |
| Alex K | 8 次提及 | 关于 API 的私信 |

## 新项目/主题
| 名称 | 出现频率 | 背景 |
|------|-----------|---------|
| Starlight | 15 次提及 | 规划文档、产品 |

## 建议清理
- **Horizon 项目** — 30 天内无提及。标记为已完成？
```

按置信度分组呈现。高置信度条目直接提供添加；低置信度条目需询问。

## 说明

- 永远不要在未经用户确认的情况下自动添加任务或记忆
- 可用时保留外部来源链接
- 任务标题的模糊匹配处理措辞上的细微差异
- 可频繁运行——只在有新信息时才更新
- `--comprehensive` 始终以交互方式运行
