# 开始命令

> 如果您看到不熟悉的占位符或需要检查已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

初始化任务和记忆系统，然后打开统一仪表板。

## 操作说明

### 1. 检查现有内容

检查工作目录是否有：
- `TASKS.md` — 任务列表
- `CLAUDE.md` — 工作记忆
- `memory/` — 深度记忆目录
- `dashboard.html` — 可视化界面

### 2. 创建缺失内容

**如果 `TASKS.md` 不存在：** 使用标准模板创建（参见 task-management 技能）。放置在当前工作目录中。

**如果 `dashboard.html` 不存在：** 从 `${CLAUDE_PLUGIN_ROOT}/skills/dashboard.html` 复制到当前工作目录。

**如果 `CLAUDE.md` 和 `memory/` 不存在：** 这是全新设置——打开仪表板后，开始记忆引导工作流（见下文）。将这些文件放置在当前工作目录中。

### 3. 打开仪表板

不要使用 `open` 或 `xdg-open` — 在 OpenClaw 中，代理运行在虚拟机中，Shell 打开命令无法到达用户的浏览器。请告知用户："仪表板已准备好，位于 `dashboard.html`。从文件浏览器中打开它即可开始。"

### 4. 引导用户

如果一切已初始化：
```
仪表板已打开。您的任务和记忆均已加载。
- /productivity:update 以同步任务并检查记忆
- /productivity:update --comprehensive 进行所有活动的深度扫描
```

如果记忆尚未引导，继续执行步骤 5。

### 5. 引导记忆（仅首次运行）

仅当 `CLAUDE.md` 和 `memory/` 不存在时执行此步骤。

工作场所语言的最佳来源是用户实际的任务列表。真实任务 = 真实缩写。

**询问用户：**
```
您把待办事项或任务列表放在哪里？可以是：
- 本地文件（如 TASKS.md、todo.txt）
- 应用（如 Asana、Linear、Jira、Notion、Todoist）
- 笔记文件

AI助手将使用您的任务来学习您的工作场所缩写。
```

**获得任务列表访问权限后：**

对每个任务项进行分析，寻找潜在缩写：
- 可能是昵称的名字
- 缩写或简写
- 项目引用或代号
- 内部术语或行话

**对每个条目进行交互式解码：**

```
任务："将 PSR 发给 Todd，关于 Phoenix 阻塞项"

有些术语我想确保理解正确：

1. **PSR** - 这代表什么？
2. **Todd** - Todd 是谁？（全名、角色）
3. **Phoenix** - 这是项目代号吗？是关于什么的？
```

继续遍历每个任务，只询问尚未解码的术语。

### 6. 可选的综合扫描

任务列表解码后，提供：
```
您希望我对您的消息、邮件和文档进行综合扫描吗？
这需要更长时间，但能构建关于您工作中的人员、项目和术语的更丰富背景。

或者我们可以保持现有内容，之后再添加背景。
```

**如果他们选择综合扫描：**

从可用的 MCP 来源收集数据：
- **聊天：** 最近的消息、频道、私信
- **邮件：** 已发送消息、收件人
- **文档：** 最近的文档、协作者
- **日历：** 会议、与会者

构建发现的人员、项目和术语的概要。按置信度分组呈现发现：
- **可直接添加**（高置信度）——提供直接添加
- **需要澄清**——询问用户
- **低频率/不明确**——记录供以后参考

### 7. 写入记忆文件

根据收集的所有内容，创建：

**CLAUDE.md**（工作记忆，约 50-80 行）：
```markdown
# 记忆

## 我
[姓名]，[团队] 的 [角色]。

## 人员
| 谁 | 角色 |
|-----|------|
| **[昵称]** | [全名]，[角色] |

## 术语
| 术语 | 含义 |
|------|---------|
| [缩写] | [展开] |

## 项目
| 名称 | 内容 |
|------|------|
| **[代号]** | [描述] |

## 偏好
- [发现的偏好]
```

**memory/** 目录：
- `memory/glossary.md` — 完整解码表（缩写、术语、昵称、代号）
- `memory/people/{name}.md` — 个人档案
- `memory/projects/{name}.md` — 项目详情
- `memory/context/company.md` — 团队、工具、流程

### 8. 报告结果

```
效率系统已准备好：
- 任务：TASKS.md（X 个事项）
- 记忆：X 人，X 个术语，X 个项目
- 仪表板：在浏览器中打开

使用 /productivity:update 保持内容更新（添加 --comprehensive 进行深度扫描）。
```

## 说明

- 如果记忆已初始化，此命令只打开仪表板
- 昵称至关重要——始终记录人们实际上是如何被称呼的
- 如果某个来源不可用，跳过并记录缺口
- 记忆在引导后通过自然对话有机增长
