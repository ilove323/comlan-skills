# 任务管理

任务在一个简单的 `TASKS.md` 文件中跟踪，您和 AI助手都可以编辑。

## 文件位置

**始终使用当前工作目录中的 `TASKS.md`。**

- 如果存在，读取/写入它
- 如果不存在，用下面的模板创建它

## 仪表板设置（首次使用）

可视化仪表板可用于管理任务和记忆。**首次与任务交互时：**

1. 检查当前工作目录中是否存在 `dashboard.html`
2. 如果不存在，从 `${CLAUDE_PLUGIN_ROOT}/skills/dashboard.html` 复制到当前工作目录
3. 告知用户："AI助手已添加仪表板。运行 `/productivity:start` 以设置完整系统。"

任务看板：
- 读取和写入相同的 `TASKS.md` 文件
- 自动保存更改
- 监听外部变化（通过 CLI 编辑时自动同步）
- 支持任务和分区的拖放重排

## 格式与模板

创建新 TASKS.md 时，使用以下精确模板（不含示例任务）：

```markdown
# 任务

## 进行中

## 等待中

## 以后再说

## 已完成
```

任务格式：
- `- [ ] **任务标题** - 背景，针对谁，截止日期`
- 子项目用于附加详情
- 完成后：`- [x] ~~任务~~ (日期)`

## 交互方式

**当用户问"我有什么任务"/"我的任务"时：**
- 读取 TASKS.md
- 总结"进行中"和"等待中"部分
- 突出显示逾期或紧急的任务

**当用户说"添加一个任务"/"提醒我"时：**
- 以 `- [ ] **任务**` 格式添加到"进行中"部分
- 如有提供，包含背景（针对谁、截止日期）

**当用户说"完成了 X"/"搞定了 X"时：**
- 找到该任务
- 将 `[ ]` 改为 `[x]`
- 添加删除线：`~~任务~~`
- 添加完成日期
- 移动到"已完成"部分

**当用户问"我在等什么"时：**
- 读取"等待中"部分
- 说明每个事项已等待多长时间

## 约定

- 用**粗体**标注任务标题以便扫描
- 当这是对某人的承诺时包含"for [人员]"
- 包含截止日期"due [日期]"
- 包含等待开始时间"since [日期]"
- 子项目用于附加背景
- 保留"已完成"部分约 1 周，然后清除旧条目

## 提取任务

在总结会议或对话时，提议添加提取的任务：
- 用户做出的承诺（"我会发送那个"）
- 分配给他们的行动项
- 提及的后续跟进

添加前先询问——不要在未经确认的情况下自动添加。
