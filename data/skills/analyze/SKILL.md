# /analyze - 回答数据问题

> 如遇到不熟悉的占位符或需要查看哪些工具已接入，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

回答数据问题，从快速查询到完整分析，再到正式报告，全面覆盖。

## 使用方式

```
/analyze <自然语言问题>
```

## 工作流

### 1. 理解问题

解析用户的问题并判断：

- **复杂程度**：
  - **快速回答**：单一指标、简单过滤、事实查询（例如："上周新增了多少用户？"）
  - **完整分析**：多维度探索、趋势分析、对比研究（例如："是什么导致了转化率下降？"）
  - **正式报告**：包含方法论、注意事项和建议的综合调查（例如："准备一份我们订阅指标的季度业务回顾"）
- **数据需求**：需要哪些表、指标、维度和时间范围
- **输出格式**：数字、表格、图表、叙述或组合

### 2. 收集数据

**如果已接入数据仓库 MCP 服务器：**

1. 探索 Schema 以找到相关表和字段
2. 编写 SQL 查询提取所需数据
3. 执行查询并获取结果
4. 如果查询失败，调试并重试（检查字段名、表引用、特定方言的语法）
5. 如果结果看起来异常，先做合理性检查再继续

**如果未接入数据仓库：**

1. 请用户通过以下方式提供数据：
   - 直接粘贴查询结果
   - 上传 CSV 或 Excel 文件
   - 描述 Schema，以便写出可供他们运行的查询
2. 如果为手动执行编写查询，使用 `sql-queries` 技能获取特定方言的最佳实践
3. 数据提供后，继续分析

### 3. 分析

- 计算相关指标、聚合值和对比数据
- 识别规律、趋势、异常值和离群点
- 跨维度对比（时间段、细分、类别）
- 对于复杂分析，将问题分解为子问题逐一解答

### 4. 呈现前验证

分享结果前，执行验证检查：

- **行数合理性**：记录数量是否合理？
- **空值检查**：是否存在可能扭曲结果的意外空值？
- **量级检查**：数字是否在合理范围内？
- **趋势连续性**：时间序列是否存在意外断层？
- **聚合逻辑**：子计总是否正确累加为总计？

如有疑问，调查并注明注意事项。

### 5. 呈现发现

**快速回答：**
- 直接陈述答案并附上相关背景
- 包含所用查询（折叠或代码块形式），方便复现

**完整分析：**
- 以关键发现或洞察开头
- 用数据表格和/或可视化图表支撑
- 注明方法论和注意事项
- 建议后续跟进问题

**正式报告：**
- 含关键要点的摘要
- 说明分析方法和数据来源的方法论章节
- 附佐证的详细发现
- 注意事项、局限性和数据质量说明
- 建议和后续步骤

### 6. 在有帮助时可视化

当图表比表格更能有效传达结果时：

- 使用 `data-visualization` 技能选择合适的图表类型
- 生成 Python 可视化或构建 HTML 仪表板
- 遵循可视化最佳实践，确保清晰准确

## 示例

**快速回答：**
```
/analyze 12月有多少新用户注册？
```

**完整分析：**
```
/analyze 是什么导致过去3个月支持工单量增加？请按类别和优先级细分。
```

**正式报告：**
```
/analyze 对我们的客户表进行数据质量评估——完整性、一致性，以及需要解决的问题。
```

## 提示

- 尽可能明确时间范围、细分维度或指标
- 如果知道表名，提前告知可以加快处理速度
- 复杂问题可能会被拆分为多个查询
- 结果始终在呈现前经过验证——如有异常，AI助手会进行标记
