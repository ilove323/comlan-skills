# /performance-review

> 如遇不熟悉的占位符或需要查看已连接工具，请参见 [CONNECTORS.md](../../CONNECTORS.md)。

生成绩效评估模板并协助构建反馈内容。

## 用法

```
/performance-review $ARGUMENTS
```

## 使用模式

```
/performance-review self-assessment       # 生成自评模板
/performance-review manager [员工姓名]    # 为特定员工生成经理评估模板
/performance-review calibration           # 生成校准准备文档
```

如未指定模式，会询问需要哪种类型的评估。

## 输出——自评模板

```markdown

### 主要成就
[列出本周期3-5项主要成就。每项说明情境、您的贡献和影响。]

1. **[成就]**
   - 情境：[背景]
   - 贡献：[您做了什么]
   - 影响：[可量化的结果]

### 目标完成情况
| 目标 | 状态 | 佐证 |
|------|------|------|
| [上期目标] | 达成/超额/未达成 | [如何判断] |

### 成长领域
[您在哪些方面有所成长？新技能、扩展的职责范围、领导力时刻。]

### 挑战与反思
[哪些方面比较困难？如果重来您会怎么做？]

### 下期目标
1. [目标——具体且可衡量]
2. [目标]
3. [目标]

### 对直属上级的反馈
[您的上级可以如何更好地支持您？]
```

## 已连接的服务

若已连接 **~~HRIS**：
- 拉取历史评估记录和目标跟踪数据
- 预填员工基本信息和当前岗位

若已连接 **~~project tracker**：
- 提取评估周期内的工作成果和贡献
- 引用具体工单和项目里程碑作为佐证

## 使用建议

1. **要具体** ——"做得很好"不是反馈。"你通过实施新CI流程将部署时间缩短了40%"才是。
2. **正向与建设性并重** ——两者都不可或缺，都不应让对方感到意外。
3. **聚焦行为而非性格** ——"你的文档经常不完整"而非"你很粗心"。
4. **让发展计划可落地** ——"改善沟通能力"太笼统；"在下次全员会议上做一次分享"才是可行动的。
