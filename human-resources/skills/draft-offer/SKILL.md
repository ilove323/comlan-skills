# /draft-offer

> 如遇不熟悉的占位符或需要查看已连接工具，请参见 [CONNECTORS.md](../../CONNECTORS.md)。

为新员工起草完整的录用通知书。

## 用法

```
/draft-offer $ARGUMENTS
```

## 所需信息

- **岗位和职称**：具体什么职位？
- **级别**：初级、中级、高级、资深等
- **工作地点**：在哪里工作？（影响薪酬和福利）
- **薪酬**：基本工资、股权、签约奖金（如有）
- **入职日期**：何时开始工作？
- **直属上级**：向谁汇报？

如果信息不完整，AI助手会协助您逐一确认。

## 输出

```markdown
## 录用通知书草稿：[岗位] — [级别]

### 薪酬方案
| 薪酬项目 | 详情 |
|---------|------|
| **基本工资** | $[X]/年 |
| **股权** | [X股/单位]，[归属计划] |
| **签约奖金** | $[X]（如有） |
| **目标奖金** | [X]%基本工资（如有） |
| **第一年总薪酬** | $[X] |

### 条款
- **入职日期**：[日期]
- **直属上级**：[经理]
- **工作地点**：[办公室/远程/混合]
- **雇佣类型**：[全职，豁免制]

### 福利摘要
[与候选人相关的主要福利亮点]

### 录用通知书正文

尊敬的[候选人姓名]，

我们非常高兴地向您提供[公司][职位]一职……

[完整录用通知书正文]

### 给招聘经理的备注
- [如需，提供谈判指导]
- [薪资带背景]
- [任何需要关注的事项]
```

## 已连接的服务

若已连接 **~~HRIS**：
- 拉取对应级别/岗位的薪资带数据
- 核实人员编制审批情况
- 自动填入福利详情

若已连接 **~~ATS**：
- 从候选人申请中拉取基本信息
- 在招聘管道中更新录用状态

## 使用建议

1. **展示总薪酬** ——候选人比较的是总薪酬，而非只看基本工资。
2. **股权说明要具体** ——注明股数、当前估值方式、归属计划。
3. **加入个人元素** ——引用面试过程中的某个细节，让通知书更有温度。
