# /onboarding

> 如遇不熟悉的占位符或需要查看已连接工具，请参见 [CONNECTORS.md](../../CONNECTORS.md)。

为新团队成员生成全面的入职方案。

## 用法

```
/onboarding $ARGUMENTS
```

## 所需信息

- **新员工姓名**：谁要入职？
- **岗位**：什么职位？
- **团队**：加入哪个团队？
- **入职日期**：何时开始？
- **直属上级**：谁是他们的经理？

## 已连接的服务

若已连接 **~~HRIS**：
- 拉取新员工信息和团队组织结构
- 根据岗位自动生成所需工具权限清单

若已连接 **~~knowledge base**：
- 链接相关入职文档、团队Wiki和操作手册
- 提取团队现有入职清单进行定制化

若已连接 **~~calendar**：
- 自动创建第1天的日历事项和第1周的会议邀请

## 使用建议

1. **按岗位定制** ——工程师的入职流程与设计师不同。
2. **第1天不要信息过载** ——重点放在设备和人际关系。深入工作从第2周开始。
3. **安排入职引导人** ——有一个不是直属上级的"问题导航人"非常重要。
