# /comp-analysis

> 如遇不熟悉的占位符或需要查看已连接工具，请参见 [CONNECTORS.md](../../CONNECTORS.md)。

分析薪酬数据，用于对标、薪资带定位和规划。协助将薪酬与市场数据进行对比，支持招聘、留才和股权规划决策。

## 用法

```
/comp-analysis $ARGUMENTS
```

## 所需信息

**方式A：单一岗位分析**
"一个旧金山高级软件工程师应该给多少薪酬？"

**方式B：上传薪酬数据**
上传CSV或粘贴您的薪资带数据，AI助手将分析定位情况、识别异常值并与市场进行对比。

**方式C：股权建模**
"对一个$50股价下4年归属的10,000股期权刷新授予进行建模。"

## 薪酬框架

### 总薪酬构成
- **基本工资**：现金薪酬
- **股权**：限制性股票单位（RSU）、期权或其他股权形式
- **奖金**：年度目标奖金、签约奖金
- **福利**：医疗、养老、员工福利（较难量化）

### 关键变量
- **岗位**：职能和专业方向
- **级别**：个人贡献者层级、管理层级
- **地点**：地区薪酬调整
- **公司阶段**：初创期、成长期还是上市公司
- **行业**：科技、金融、医疗等

### 数据来源
- **已连接 ~~compensation data**：拉取经核实的市场数据
- **未连接**：使用网络调研、公开薪酬数据和用户提供的背景信息
- 始终注明数据的时效性和来源局限性

## 已连接的服务

若已连接 **~~compensation data**：
- 按岗位、级别和地点拉取经核实的市场基准
- 将您的薪资带与实时市场数据进行对比

若已连接 **~~HRIS**：
- 拉取当前员工薪酬数据进行薪资带分析
- 自动识别异常值和留才风险

## 使用建议

1. **地点很重要** ——对标时务必注明地点。旧金山、奥斯汀和伦敦的薪酬水平差异很大。
2. **关注总薪酬，而非仅基本工资** ——纳入股权、奖金和福利才能呈现完整图景。
3. **保护数据隐私** ——薪酬数据属于敏感信息，分析结果仅保留在您的对话中。
