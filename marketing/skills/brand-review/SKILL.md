# 品牌审查

> 如遇不熟悉的占位符或需要查看已连接工具，请参见 [CONNECTORS.md](../../CONNECTORS.md)。

对照品牌声音、风格规范和信息标准审查营销内容，标记偏差并提供具体改进建议。

## 触发方式

用户运行 `/brand-review` 或要求审查、检查内容是否符合品牌规范。

## 输入

1. **待审内容** ——接受以下任何形式：
   - 直接粘贴到对话中
   - 文件路径或 ~~knowledge base 引用（如Notion页面、共享文档）
   - 已发布页面的URL
   - 多篇内容批量审查

2. **品牌规范来源**（自动判断）：
   - 如本地设置中已配置品牌风格指南，自动使用
   - 如未配置，询问："您有希望我参考审查的品牌风格指南或声音规范吗？可以粘贴、分享文件或描述您的品牌风格。否则我将进行通用的清晰度、一致性和专业性审查。"

## 审查流程

### 已配置品牌规范时

从以下维度评估内容：

#### 声音与语气
- 内容是否符合定义的品牌声音属性？
- 语气是否与内容类型和受众相匹配？
- 是否存在感觉不一致的声音转变？
- 标记偏差的具体句子或短语，并说明原因

#### 术语与用语
- 是否正确使用了品牌偏好术语？
- 是否出现了应避免的术语或短语？
- 专业术语使用水平是否适合目标受众？
- 产品名称、功能名称和品牌术语是否正确使用（大小写、格式）？

#### 信息支柱
- 内容是否与定义的信息支柱或价值主张一致？
- 声明是否与已审批的信息保持一致？
- 内容是否强化还是与品牌定位相矛盾？

#### 风格规范合规性
- 按风格指南的语法和标点（如牛津逗号、标题大小写 vs. 句子大小写）
- 格式惯例（标题、列表、强调）
- 数字格式、日期格式
- 缩写用法（首次使用时是否展开？）

### 无品牌规范时（通用审查）

从以下方面评估内容：

#### 清晰度
- 主要信息是否在第一段内清晰表达？
- 句子是否简洁易懂？
- 结构是否清晰易于跟随？
- 是否有歧义或不清楚的引用？

#### 一致性
- 语气是否全文保持一致？
- 术语使用是否一致（同一概念不随意切换同义词）？
- 格式是否一致（标题、列表、大小写）？

#### 专业性
- 内容是否无错别字、语法错误和表达别扭的情况？
- 语气是否适合目标受众？
- 声明是否有依据或佐证？

### 法律合规标记（始终检查）

无论是否配置品牌规范，均需标记：
- **无依据的声明** ——未经证明或限定的绝对语（"最好的"、"最快的"、"唯一的"）
- **缺少免责声明** ——可能需要法律免责声明的金融声明、健康声明或保证
- **比较性声明** ——可能受到挑战的竞争对手比较
- **监管性用语** ——可能需要合规审查的内容（金融服务、医疗保健等）
- **推荐信问题** ——无归因或无披露的引用或背书
- **版权问题** ——疑似与其他来源高度相似的内容

## 品牌声音参考

使用这些框架对照品牌标准评估内容，或帮助用户记录其品牌声音。

### 品牌声音文档框架

完整的品牌声音文档应涵盖以下内容：

1. **品牌个性** ——将品牌视为一个人来定义。示例："如果我们的品牌是一个人，他/她会是一位知识渊博的同事，用简单的语言解释复杂的事物，真诚地为你的成功喝彩，从不俯视你。"
2. **声音属性** ——3-5个定义品牌沟通方式的属性，每个属性包含实际含义、不包含的含义（防止误解）和示例。
3. **受众认知** ——品牌的沟通对象（主要和次要）、他们关心什么、他们的专业水平，以及他们期望被如何称呼。
4. **核心信息支柱** ——品牌持续传递的3-5个核心主题、这些信息的层级，以及每个支柱与受众需求的关联。
5. **语气谱系** ——声音如何在不同情境中调整，同时保持品牌识别度。
6. **风格规则** ——具体的语法、格式和用语规则。
7. **术语表** ——偏好和应避免的术语。

### 声音属性谱系

在定义或评估品牌声音时，将属性定位在谱系上：

| 谱系 | 一端 | 另一端 |
|------|------|-------|
| 正式度 | 正式、机构化 | 随意、对话式 |
| 权威性 | 专家、权威 | 平等、协作 |
| 情感性 | 温暖、有共情 | 直接、就事论事 |
| 复杂度 | 技术性、精确 | 简单、易懂 |
| 能量感 | 大胆、充满活力 | 平静、沉稳 |
| 幽默感 | 轻松、机智 | 严肃、真诚 |
| 创新性 | 前沿、前瞻性 | 成熟、久经验证 |

对于每个选定的属性，按以下格式记录：

**[属性名称]**
- **我们是**：[实际含义]
- **我们不是**：[常见误解，需要避免]
- **听起来像**：[体现该属性的示例句子]
- **听起来不像**：[违背该属性的示例句子]

示例：

**亲和力**
- **我们是**：友善、清晰、无术语、对初学者和专家都友好
- **我们不是**：过于简化、过于随意或缺乏实质内容
- **听起来像**："以下是入门步骤——大约需要五分钟。"
- **听起来不像**："嘿！这超级简单，新手也能搞定哈哈。"

### 渠道和情境中的语气调整

品牌声音保持一致，但语气随情境调整。语气是应用于声音的情感色彩。

#### 按渠道的语气

| 渠道 | 语气调整 | 示例 |
|------|---------|------|
| 博客 | 信息性、对话式、教育性 | "让我们一起了解这是如何运作的，以及为什么对您的团队重要。" |
| 社交媒体（LinkedIn） | 专业、发人深省、简洁 | "我们本季度运营50个营销活动后的三点发现。" |
| 社交媒体（微博/Twitter/X） | 有力、直接、有时机智 | "您的落地页只有3秒。好好利用它们。" |
| 邮件营销 | 个人化、有帮助、行动导向 | "我们整理了一些您可能会觉得有用的内容。" |
| 销售材料 | 自信、突出利益、具体 | "使用我们平台的团队将报告时间缩短了40%。" |
| 帮助文档 | 清晰、耐心、分步骤 | "如果出现此错误，以下是解决方法。" |
| 新闻稿 | 正式、事实性、具新闻价值 | "公司今天宣布推出……" |
| 错误提示 | 有共情、有帮助、无指责 | "我们这边出了点问题，正在排查中。" |

#### 按情境的语气

| 情境 | 语气调整 |
|------|---------|
| 产品发布 | 兴奋、自信、前瞻性 |
| 故障/宕机 | 透明、有共情、负责任 |
| 客户成功故事 | 喜悦、具体、将功劳归给客户 |
| 思想领导力 | 权威、细致入微、有数据支撑 |
| 用户引导 | 热情欢迎、鼓励、清晰 |
| 坏消息（涨价、下线功能） | 诚实、尊重、提供解决方案 |
| 竞争对比 | 自信但公平、有事实依据、不贬低对方 |

#### 语气调整原则
声音属性保持不变，语气根据情境调整其比重。例如，如果一个品牌是"大胆而温暖的"：
- 在产品发布时，调高大胆程度
- 在故障响应时，调高温暖程度
- 两个属性都不会消失，只是比重发生变化

### 风格规范执行

#### 语法与机制
记录并始终如一地执行以下选择：

| 规则 | 可选 | 示例 |
|------|------|------|
| 牛津逗号 | 使用/不使用 | "快速、可靠、安全" vs. "快速、可靠和安全" |
| 标题大小写 vs. 句子大小写 | 标题式/句子式 | "如何入门" vs. "如何入门" |
| 缩写 | 使用/避免 | "we're" vs. "we are" |
| 破折号间距 | 无空格/有空格 | "this—and more" vs. "this — and more" |
| 数字 | 1-9拼写，10+使用数字/始终使用数字 | "五项功能" vs. "5项功能" |
| 百分号 | % / 百分之 | "50%" vs. "50百分之" |
| 日期格式 | 2025年1月15日 / 15/01/2025等 | — |
| 时间格式 | 12小时制/24小时制 | "下午3:00" vs. "15:00" |
| 列表 | 加句号/不加句号 | "设置您的账户。" vs. "设置您的账户" |

#### 格式惯例
- 标题层级（何时使用H1、H2、H3）
- 加粗和斜体用法（加粗用于强调，斜体用于标题/术语）
- 链接文字（描述性 vs. "点击此处"——始终使用描述性）
- 图片alt文本要求
- 代码格式（适用于技术类品牌）
- 标注框或高亮框用法

#### 标点与强调
- 感叹号使用政策（限制使用，每次最多一个）
- 省略号用法（大多数专业情境下应避免）
- 全大写政策（避免；用加粗代替强调）
- 各渠道表情符号用法（专业渠道：极少或不用；社交媒体：适当使用）

### 术语管理

#### 偏好术语

维护偏好术语及其错误替代的对照表：

| 使用 | 不用 | 备注 |
|------|------|------|
| 注册（动词） | 报名注册（动词） | 区分动词和名词形式 |
| 登录（动词） | 登陆（动词） | 区分动词和名词形式 |
| 设置（动词） | 设定（动词） | 区分动词和名词形式 |
| 电子邮件 | e-mail | 无连字符 |
| 网站 | 网页 | 指整体时用"网站" |

#### 产品和功能名称
- 产品名称的官方大小写规则
- 何时使用全称 vs. 简称
- 产品名称前是否加"the"
- 在文案中如何处理版本信息
- 商标和注册标志（何时必须标注，何时可省略）

#### 包容性语言
- 使用性别中性语言（未知性别时使用他们/他们的）
- 避免歧视性用语
- 适当时使用以人为本的语言
- 避免可能无法翻译的文化特定俚语
- 用"简单"或"直接"代替"容易"（因为容易与否因人而异）

## 输出格式

将审查结果以如下形式呈现：

### 总结
- 整体评估：内容与品牌标准的符合程度（或整体质量）
- 1-2句话总结主要优势
- 1-2句话总结最重要的改进方向

### 详细发现

对发现的每个问题提供：

| 问题 | 位置 | 严重程度 | 建议 |
|------|------|---------|------|
| | | | |

严重程度说明：
- **高** ——与品牌声音相悖、存在合规风险或严重削弱信息传递
- **中** ——与规范不一致但无实质伤害
- **低** ——轻微的风格或偏好问题

### 修改建议

针对3-5个最严重的问题，提供修改前后对比，展示原文和建议修改版本。

### 法律/合规标记

单独列出法律或合规方面的关切，并附建议行动。

## 审查后续

询问："您是否希望AI助手：
- 按照这些建议修改全文？
- 仅修复高严重程度的问题？
- 按照相同规范审查其他内容？
- 协助记录您的品牌声音以备未来审查使用？"
