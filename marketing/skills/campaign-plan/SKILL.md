# 活动策划

> 如遇不熟悉的占位符或需要查看已连接工具，请参见 [CONNECTORS.md](../../CONNECTORS.md)。

生成含目标、受众、信息、渠道策略、内容日历和成功指标的完整营销活动简报。

## 触发方式

用户运行 `/campaign-plan` 或要求规划、设计或构建营销活动。

## 输入

从用户处收集以下信息，如未提供则先询问：

1. **活动目标** ——主要目标（如推动注册、提升品牌知名度、产品发布、潜在客户开发、重新激活流失用户）

2. **目标受众** ——活动针对的群体（人口特征、职位、行业、痛点、购买阶段）

3. **时间安排** ——活动周期和关键日期（发布日期、活动日期、季节性截止日期）

4. **预算范围** ——大致预算或预算区间（可选；如不提供，生成渠道中立方案并注明预算分配的重要性）

5. **附加背景**（可选）：
   - 推广的产品或服务
   - 核心差异化因素或价值主张
   - 以往活动表现或经验教训
   - 品牌规范或限制条件
   - 地理范围

## 活动简报结构

生成含以下章节的活动简报：

### 1. 活动概述
- 活动名称建议
- 活动摘要（一句话）
- 含具体可衡量目标的主要目标
- 次要目标（如适用）

### 2. 目标受众
- 主要受众细分及描述
- 次要受众细分（如适用）
- 受众痛点和动机
- 受众活跃的渠道、社区和媒体
- 购买阶段对应关系（认知、考虑、决策）

### 3. 核心信息
- 活动核心信息（一句话）
- 针对受众痛点的3-4条支撑信息
- 按渠道的信息变体（如需要不同语气）
- 支撑每条信息的证据点

### 4. 渠道策略
根据受众和目标推荐渠道，每个渠道包含：
- 为何该渠道适合受众和目标
- 内容形式建议
- 预计工作量（低/中/高）
- 预算分配建议（如已提供预算）

考虑的渠道包括：
- 自有媒体：博客、邮件、网站、社交媒体账号
- 赢取媒体：PR、KOL合作、投稿、社区运营
- 付费媒体：搜索广告、社交广告、展示广告、赞助内容、活动

### 5. 内容日历
创建按周（或短期活动按天）的内容日历：
- 每周产出什么内容
- 每篇内容对应的渠道
- 关键里程碑和截止日期
- 内容之间的依赖关系（如"落地页上线后才能启动付费广告"）

格式为表格：

| 周次 | 内容项目 | 渠道 | 负责人/备注 | 状态 |
|------|---------|------|-----------|------|

### 6. 所需内容清单
列出活动所需的每项内容资产：
- 资产名称和类型（博客文章、邮件、社交媒体帖子、广告创意、落地页等）
- 内容应包含的简要描述
- 优先级（必须有 vs. 加分项）
- 建议制作时间

### 7. 成功指标
定义与活动目标一致的KPI：
- 含目标数字的主要KPI
- 次要KPI（3-5项）
- 每项指标的追踪方式
- 建议的报告频率

若已连接 ~~product analytics，参考可用的历史绩效基准来设定目标。

### 8. 预算分配（如已提供预算）
- 按渠道或活动类型分解
- 内容制作成本 vs. 媒介投放成本
- 应急预算建议（通常10-15%）

### 9. 风险与应对
- 2-3个潜在风险（时间、受众不匹配、渠道表现不佳）
- 每项风险的应对策略

### 10. 下一步行动
- 启动活动的即时行动项
- 所需的利益相关方审批
- 关键决策节点

## 策划参考

### 活动框架：目标、受众、信息、渠道、衡量

每个活动都应建立在这五要素框架之上：

#### 目标
在策划任何内容之前，先定义成功的样子。

- **认知**：提升品牌或产品知名度（用覆盖率、曝光量、声量占比衡量）
- **考虑**：推动互动和教育（用内容互动、邮件注册、网络研讨会参与衡量）
- **转化**：产生潜在客户或销售（用注册、演示、购买、商机衡量）
- **留存**：重新激活现有客户（用降低流失率、增购、NPS衡量）
- **推荐**：将客户转化为传播者（用推荐、评论、用户生成内容衡量）

好的目标是SMART的：具体（Specific）、可衡量（Measurable）、可实现（Achievable）、相关（Relevant）、有时限（Time-bound）。

示例："在活动上线后6周内，从北美中型SaaS公司获得200个市场合格线索（MQL）。"

#### 受众
定义目标受众时要有足够的具体性，以指导信息策略和渠道选择。

- **人口特征**：职位/头衔、资历、公司规模、行业
- **心理特征**：动机、痛点、目标、疑虑
- **行为特征**：他们在哪里消费内容、如何购买、以往的互动记录
- **购买阶段**：对问题不了解、正在研究解决方案，还是准备购买？

创建简短的受众画像（不需要完整的用户画像）用于活动策划：
> "[职位]在[公司类型]，正在为[痛点]困扰，寻找[期望结果]。他们通常通过[渠道]发现解决方案，最关注[优先事项]。"

#### 信息
打造能与受众产生共鸣的核心信息和支撑要点。

- **核心信息**：一句话概括希望受众产生的想法、感受或行动
- **支撑信息**：3-4个提供证据、回应疑虑或阐述利益的要点
- **证据点**：支撑每条信息的数据、案例、用户证言或第三方验证
- **差异化**：与其他选择（包括不行动）相比，您的方案有何不同

信息层级：
1. 我为什么要关心？（针对痛点或机会）
2. 解决方案是什么？（定位您的产品）
3. 为什么选您？（与竞争对手的差异）
4. 我该怎么做？（行动号召）

#### 渠道
根据目标受众的活跃阵地选择渠道，而非仅凭自己最熟悉的。参见下方渠道选择指南。

#### 衡量
定义如何判断活动是否成功。参见下方按活动类型划分的成功指标。

### 渠道选择指南

#### 自有媒体

| 渠道 | 最适用于 | 典型指标 | 工作量 |
|------|---------|---------|-------|
| 博客/网站 | SEO、思想领导力、教育 | 流量、页面停留时间、转化 | 中 |
| 邮件 | 培育、留存、公告 | 打开率、点击率、转化 | 低-中 |
| 社交媒体（自然流量） | 认知、社区、品牌建设 | 互动、覆盖率、粉丝增长 | 中 |
| 网络研讨会 | 教育、潜在客户开发、产品演示 | 注册数、出席率、商机 | 高 |
| 播客 | 思想领导力、品牌认知 | 下载量、订阅增长 | 高 |

#### 赢取媒体

| 渠道 | 最适用于 | 典型指标 | 工作量 |
|------|---------|---------|-------|
| PR/媒体报道 | 认知、公信力、产品发布 | 报道量、声量占比、转介绍流量 | 高 |
| 投稿内容 | 受众拓展、SEO、公信力 | 转介绍流量、反向链接 | 中 |
| KOL/合作伙伴 | 受众拓展、信任度 | 覆盖率、互动、转介绍转化 | 中-高 |
| 社区运营 | 认知、信任、用户反馈 | 提及数、互动、转介绍流量 | 中 |
| 评价/评级 | 公信力、SEO、考虑阶段 | 评价数量、评分、转化提升 | 低-中 |

#### 付费媒体

| 渠道 | 最适用于 | 典型指标 | 工作量 |
|------|---------|---------|-------|
| 搜索广告（SEM） | 高意向潜在客户获取 | CPC、点击率、转化率、CPA | 中 |
| 社交广告 | 认知、再营销、潜在客户 | CPM、CPC、点击率、CPA、ROAS | 中 |
| 展示/程序化广告 | 认知、再营销 | 曝光量、CPM、浏览转化 | 低-中 |
| 赞助内容 | 思想领导力、潜在客户 | 互动、潜在客户数、每线索成本 | 中 |
| 活动/赞助 | 关系建立、品牌 | 线索数、会面数、商机影响 | 高 |

### 内容日历创建

#### 日历规划流程
1. **从里程碑开始**：活动上线日期、活动日期、产品发布日、季节性节点
2. **倒推计划**：哪些内容需要在何时上线？制作周期是多少？
3. **按漏斗阶段规划内容**：确保覆盖认知、考虑和转化各阶段
4. **按主题分批次**：将相关内容按周或双周主题分组
5. **平衡渠道**：不要过度集中于一个渠道；确保受众在不同触点看到活动
6. **预留灵活空间**：保留20%的日历空档用于响应性或机会性内容

#### 内容发布节奏参考
- **博客**：根据团队规模和目标，每周1-4篇
- **邮件简报**：大多数受众每周或每两周一次
- **社交媒体**：每平台每周3-7帖（因平台而异）
- **付费活动**：活动期间持续投放，每2-4周刷新创意
- **网络研讨会**：根据资源每月或每季度一次

#### 制作周期基准
- 博客文章：3-5个工作日（调研、起草、审查、发布）
- 邮件活动：2-3个工作日（文案、设计、测试、发送）
- 社交媒体帖子：1-2个工作日（起草、设计、排期）
- 落地页：5-7个工作日（文案、设计、开发、QA）
- 视频内容：2-4周（剧本、制作、剪辑）
- 电子书/白皮书：2-4周（提纲、起草、设计、审查）

### 预算分配方法

#### 收入百分比法
- 行业基准：营销占收入的5-15%，B2B通常5-10%，B2C通常10-15%
- 初创期和成长期公司通常将收入的15-25%投入营销
- 在营销预算内，在品牌（长期）和效果（短期）之间分配

#### 渠道分配框架
一个通用起始框架（根据目标和历史数据调整）：

| 类别 | 预算占比 | 示例 |
|------|---------|------|
| 付费获客 | 30-40% | 搜索广告、社交广告、展示广告 |
| 内容制作 | 20-30% | 博客、视频、设计、电子书 |
| 活动和赞助 | 10-20% | 会议、网络研讨会、见面会 |
| 工具和技术 | 10-15% | 分析、自动化、CRM |
| 测试和实验 | 5-10% | 新渠道、A/B测试、试点 |

### 按活动类型划分的成功指标

#### 认知活动
| 指标 | 衡量内容 |
|------|---------|
| 覆盖率/曝光量 | 看到活动的人数 |
| 品牌提及量 | 品牌讨论量的增加 |
| 声量占比 | 您的提及量 vs. 竞争对手 |
| 直接流量 | 主动访问您网站的人数 |
| 社交粉丝增长 | 受众积累 |

#### 潜在客户开发活动
| 指标 | 衡量内容 |
|------|---------|
| 总线索数 | 新联系人数量 |
| 市场合格线索（MQL） | 达到质量标准的线索 |
| 每线索成本（CPL） | 支出效率 |
| 线索到MQL转化率 | 线索质量 |
| 影响商机 | 创造的收入机会 |

#### 产品发布活动
| 指标 | 衡量内容 |
|------|---------|
| 注册数或试用数 | 新产品的采用率 |
| 激活率 | 完成关键首次操作的用户比例 |
| 媒体报道量 | 获得的媒体报道 |
| 社交热度 | 提及数、分享数、互动峰值 |
| 功能采用率 | 已发布功能的使用情况 |

#### 留存/互动活动
| 指标 | 衡量内容 |
|------|---------|
| 流失率变化 | 客户留存改善 |
| 互动率 | 与活动内容的互动 |
| NPS或CSAT变化 | 满意度改善 |
| 增购/交叉销售收入 | 扩展收入 |
| 功能采用率 | 推广功能的使用情况 |

## 输出

以清晰的格式和标题呈现完整活动简报。简报结束后询问：

"您是否希望AI助手：
- 深入展开某个章节？
- 根据日历起草具体内容？
- 创建竞争分析以指导信息策略？
- 调整预算或时间安排后重新规划？"
