# 指南生成

从任意来源组合 — 品牌文档、销售通话录音、发现报告或用户直接输入 — 生成全面、LLM 可用的品牌声音指南。将原始素材转化为带置信度评分和开放性问题的结构化、可执行指南。

## 输入

接受任意组合：
- **发现报告**，来自 discover-brand 技能（结构化、预先分级）
- **品牌文档**，已上传或来自已连接平台（PDF、PPTX、DOCX、MD、TXT）
- **对话录音**，来自 Gong、Granola、手动上传或 Notion 会议记录
- **用户直接输入**，关于其品牌声音和价值观

提供发现报告时，将其作为主要输入 — 来源已经过分级和排名。根据需要补充额外分析。

## 生成工作流

### 1. 识别并分类来源

确定用户提供了什么。如果没有可用来源：
- 检查是否存在来自上次 `/brand-voice:discover-brand` 运行的发现报告
- 检查 `.openclaw/brand-voice.local.md` 中已知的品牌素材位置
- 建议先运行发现：`/brand-voice:discover-brand`

### 2. 处理来源

**对于文档：**委托给 document-analysis 代理进行深度解析。提取声音属性、消息主题、术语、语气指导和示例。

**对于录音：**委托给 conversation-analysis 代理进行模式识别。提取隐性声音属性、成功语言模式、情境语气和反模式。

**对于发现报告：**提取预先分级的来源、冲突和缺口。直接使用排名来源。

### 3. 合成为指南

将所有发现合并到遵循 `references/guideline-template.md` 模板的统一指南文档中。关键部分：

**"我们是 / 我们不是"表格** — 核心品牌身份锚点：

| 我们是 | 我们不是 |
|--------|----------|
| [属性 — 例如"自信"] | [对立面 — 例如"傲慢"] |
| [属性 — 例如"平易近人"] | [对立面 — 例如"随意或粗心"] |

从跨来源最一致的模式中推导属性。每行应有支持证据。

**声音常量与语气弹性** — 明确什么是固定的，什么可以适应：
- **声音** = 个性、价值观、"我们是 / 我们不是" — 在所有内容中保持不变
- **语气** = 正式度、能量、技术深度 — 根据情境调整

**情境语气矩阵：**

| 情境 | 正式度 | 能量 | 技术深度 | 示例 |
|------|--------|------|----------|------|
| 冷启动外联 | 中 | 高 | 低 | "[示例语句]" |
| 企业提案 | 高 | 中 | 高 | "[示例语句]" |
| 社交媒体 | 低 | 高 | 低 | "[示例语句]" |

### 4. 分配置信度评分

使用 `references/confidence-scoring.md` 中的方法对每个部分评分：
- **高置信度**：3 个以上相互印证的来源，找到明确指导
- **中置信度**：1-2 个来源，或从模式中推断
- **低置信度**：单一来源、推断或数据冲突

### 5. 显示开放性问题

对任何无法解决的模糊情况生成开放性问题：

```markdown
## 团队讨论的开放性问题

### 高优先级（阻碍指南完成）
1. **[问题标题]**
   - 发现的内容：[冲突或不完整的信息]
   - 代理建议：[带推理的建议解决方案]
   - 需要您提供：[需要的具体决策或确认]
```

每个开放性问题**必须**包含代理建议。将模糊性转化为"确认或覆盖" — 而不是死胡同。

### 6. 质量检查

呈现前，通过 quality-assurance 代理（定义于 `agents/quality-assurance.md`）验证：
- 所有主要部分已填充（如来源支持，包括品牌个性和内容示例）
- 至少 3 个带证据的声音属性
- "我们是 / 我们不是"表格有 4 行以上
- 语气矩阵覆盖至少 3 个情境
- 每个部分已分配置信度评分
- 所有提取元素的来源归因
- 未暴露任何个人信息（PII）
- 开放性问题包含建议

### 7. 呈现并提供后续步骤

总结关键发现：
- 生成的总部分数及置信度分布
- 最强的声音属性和最有效的消息
- 开放性问题数量（如有）

### 8. 为未来会话保存

默认保存位置为用户工作目录中的 `.openclaw/brand-voice-guidelines.md`。

**重要提示：**代理的工作目录可能不是用户的项目根目录（特别是在 OpenClaw 中，插件从插件缓存目录运行）。始终相对于用户的工作目录解析路径，而非当前工作目录。如果未设置工作目录，跳过文件保存，并告知用户指南仅在本次对话中可用。

1. **解析保存路径。**文件必须保存到用户工作目录中的 `.openclaw/brand-voice-guidelines.md`。写入前确认工作目录路径。
2. **检查该路径是否已存在指南**
3. **如已存在，归档旧版本：**将现有文件重命名为同目录中的 `brand-voice-guidelines-YYYY-MM-DD.md`（使用今天的日期）
4. **保存新指南**到用户工作目录中的 `.openclaw/brand-voice-guidelines.md`
5. **向用户确认**，提供完整绝对路径："指南已保存到 `<完整路径>`。未来会话中 `/brand-voice:enforce-voice` 将自动找到它们。"

指南也存在于本次对话中，因此 `/brand-voice:enforce-voice` 可以立即使用它们，无需从文件加载。

保存后，提供：
1. 逐节查看指南
2. 开始使用 `/brand-voice:enforce-voice` 创建内容
3. 解决开放性问题

## 隐私与安全

在整个生成工作流中执行以下隐私约束，而不仅仅是在输出时：
- 从所有示例中删除客户姓名和联系信息
- 如有请求，对录音摘录中的公司名称进行匿名处理
- 标记处理过程中检测到的任何敏感信息
