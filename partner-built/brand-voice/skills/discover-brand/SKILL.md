# 品牌发现

跨企业平台自主发现品牌素材。此技能协调 discover-brand 代理搜索已连接平台（Notion、Confluence、Google Drive、Box、Microsoft 365、Figma、Gong、Granola、Slack），对来源进行分级，并生成附有开放性问题的结构化发现报告。

## 发现工作流

### 0. 引导用户

开始前，简要说明即将发生的事情，让用户了解预期：

"以下是品牌发现的工作方式：

1. **搜索** — 我将在您连接的平台（Notion、Google Drive、Slack 等）中搜索品牌相关素材：风格指南、PPT、模板、录音等。
2. **分析** — 我将对找到的内容进行分类和排名，提取最佳来源，并生成发现报告，包含找到的内容、任何冲突及开放性问题。
3. **生成指南** — 审阅报告后，我可以从结果中生成结构化的品牌声音指南文档。
4. **保存** — 获得您的批准后，指南将保存到您工作目录中的 `.openclaw/brand-voice-guidelines.md`。在此之前不会写入任何内容。

搜索时间通常取决于连接了多少个平台，一般需要几分钟。准备好开始了吗？"

等待用户确认后再继续。如果用户有关于流程的问题，先回答问题。

### 1. 检查设置

如果存在，读取 `.openclaw/brand-voice.local.md`。提取：
- 公司名称
- 哪些平台已启用（notion、confluence、google-drive、box、microsoft-365、figma、gong、granola、slack）
- 搜索深度偏好（standard 或 deep）
- 最大来源限制
- "已知品牌素材"下列出的任何已知品牌素材位置

如果没有设置文件，继续使用所有已连接的平台和标准搜索深度。

### 2. 验证平台覆盖范围

在确认范围之前，检查哪些平台实际已连接并分类：

**文档平台**（品牌指南、风格指南、模板和幻灯片的存放地）：
- Notion、Confluence、Google Drive、Box、Microsoft 365（SharePoint/OneDrive）

**辅助平台**（对模式发现有价值，但不是品牌文档的存放地）：
- Slack、Gong、Granola、Figma

应用以下规则：

1. **如果没有文档平台连接**：**停止。**告知用户："您没有连接任何文档存储平台（Google Drive、SharePoint、Notion、Confluence 或 Box）。品牌指南和风格指南几乎都存放在其中一个平台上。请在运行发现之前至少连接一个。Gong/Granola/Slack 录音是有价值的补充，但不太可能包含正式的品牌文档。"

2. **如果没有 Google Drive 且没有 Microsoft 365 且没有 Box**：**警告**（但继续）："您的主要文件存储平台（Google Drive、SharePoint、Box）均未连接。品牌文档经常存放在这些平台上。发现将继续使用 [已连接平台]，但结果可能存在明显缺口。建议连接 Google Drive 或 SharePoint。"

3. **如果只连接了一个平台**：**警告**（但继续）："只有 [平台] 已连接。发现最好有 2 个以上平台进行跨来源验证。单一平台的结果置信度较低。"

### 3. 与用户确认范围

启动发现之前，确认：
- 搜索哪些平台（默认：所有已连接平台）
- 是否包含对话录音（Gong、Granola）或只搜索文档
- 任何已知的优先位置

保持简洁 — 提一个问题，而非一长串问卷。

### 4. 委托给 Discover-Brand 代理

通过 Task 工具启动 discover-brand 代理。提供：
- 公司名称（来自设置或用户输入）
- 启用的平台
- 搜索深度
- 任何已知的 URL 或优先检查的位置

代理自主执行 4 阶段发现算法：
1. **广泛发现** — 跨平台并行搜索
2. **来源分级** — 分类和排名来源
3. **深度提取** — 从顶级来源检索和提取内容
4. **发现报告** — 带开放性问题的结构化输出

### 5. 呈现发现报告

当代理返回时，向用户呈现报告及摘要：
- 找到和分析的来源总数
- 发现的关键品牌元素
- 来源间的任何冲突
- 需要团队讨论的开放性问题

### 6. 提供后续步骤

呈现报告后，提供：
1. **立即生成指南** — 使用发现报告作为输入，链接到 `/brand-voice:generate-guidelines`
2. **先解决开放性问题** — 在生成之前处理高优先级问题
3. **保存报告** — 将发现报告存储到 Notion 或本地文件
4. **扩展搜索** — 如果覆盖不足，搜索更多平台或更深入

## 开放性问题

当发现代理遇到无法解决的模糊情况时，开放性问题就会出现：
- 冲突文档（例如 2023 年风格指南与 2024 年品牌更新）
- 缺少关键部分（例如未找到社交媒体指南）
- 跨平台的术语不一致

每个开放性问题都包含代理建议。将问题呈现为"确认或覆盖" — 而不是死胡同。

## 与其他技能的集成

- **指南生成**：发现报告由 discover-brand 代理通过 Task 工具返回。将其直接作为结构化输入传递给 guideline-generation 技能，无需用户手动收集来源。
- **品牌声音执行**：一旦从发现中生成指南，执行将自动使用它们。

## 错误处理

- 如果没有连接任何平台，告知用户此插件支持哪些平台以及如何连接它们。
- 如果所有搜索都返回空结果，将发现标记为"低覆盖"，并建议用户手动提供文档或检查平台连接。
- 如果平台已连接但返回权限错误，记录缺口并继续使用其他平台。
