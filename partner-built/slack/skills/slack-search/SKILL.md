# Slack 搜索

此技能提供在 Slack 中有效搜索消息、文件和信息的指导。

## 适用场景

每当需要在 Slack 中查找信息时应用此技能 — 包括用户要求您查找消息、对话、文件或人员，或者在回答关于 Slack 中发生了什么的问题之前需要收集上下文时。

## 搜索工具概览

| 工具 | 适用场景 |
|------|----------|
| `slack_search_public` | 仅搜索公开频道。不需要用户同意。 |
| `slack_search_public_and_private` | 搜索所有频道，包括私密频道、私信和群私信。需要用户同意。 |
| `slack_search_channels` | 按名称或描述查找频道。 |
| `slack_search_users` | 按姓名、邮箱或角色查找人员。 |

## 搜索策略

### 从广到窄

1. 从简单的关键词或自然语言问题开始。
2. 如果结果太多，添加过滤器（`in:`、`from:`、日期范围）。
3. 如果结果太少，移除过滤器，尝试同义词或相关词。

### 选择正确的搜索模式

- **自然语言问题**（例如，"X 项目的截止日期是什么？"）— 最适合不知道确切关键词的模糊、概念性搜索。
- **关键词搜索**（例如，`项目X 截止日期`）— 最适合查找特定、精确的内容。

### 使用多次搜索

不要依赖单一搜索。将复杂问题分解为更小的搜索：
- 先搜索话题
- 然后搜索特定人员的贡献
- 然后在特定频道中搜索

## 搜索修饰符参考

### 位置过滤器
- `in:频道名称` — 在特定频道内搜索
- `in:<#C123456>` — 按 ID 在频道中搜索
- `-in:频道名称` — 排除某个频道
- `in:<@U123456>` — 在与用户的私信中搜索

### 用户过滤器
- `from:<@U123456>` — 来自特定用户的消息（按 ID）
- `from:用户名` — 来自用户的消息（按 Slack 用户名）
- `to:me` — 直接发给您的消息

### 内容过滤器
- `is:thread` — 只搜索线程消息
- `has:pin` — 固定的消息
- `has:link` — 包含链接的消息
- `has:file` — 有文件附件的消息
- `has::emoji:` — 有特定反应的消息

### 日期过滤器
- `before:YYYY-MM-DD` — 某日期之前的消息
- `after:YYYY-MM-DD` — 某日期之后的消息
- `on:YYYY-MM-DD` — 特定日期的消息
- `during:月份` — 特定月份期间的消息（例如，`during:january`）

### 文本匹配
- `"精确短语"` — 匹配精确短语
- `-词语` — 排除包含某词语的消息
- `通配*` — 通配符匹配（`*` 之前至少 3 个字符）

## 文件搜索

要搜索文件，在 `content_types="files"` 参数中使用类型过滤器：
- `type:images` — 图片文件
- `type:documents` — 文档文件
- `type:pdfs` — PDF 文件
- `type:spreadsheets` — 电子表格文件
- `type:canvases` — Slack 画布

示例：`content_types="files" type:pdfs budget after:2025-01-01`

## 对搜索结果的后续操作

找到相关消息后：
- 使用 `slack_read_thread` 获取任何线程消息的完整线程上下文。
- 使用带 `oldest`/`latest` 时间戳的 `slack_read_channel` 读取周围的消息以获取背景。
- 使用 `slack_read_user_profile` 识别结果中出现其 ID 的用户是谁。

## 常见陷阱

- **布尔运算符不起作用。**`AND`、`OR`、`NOT` 不受支持。使用空格（隐式 AND）和 `-` 进行排除。
- **括号不起作用。**不要尝试用 `()` 对搜索词分组。
- **搜索不是实时的。**非常近期的消息（最后几秒）可能不会出现在搜索结果中。对于最近的消息，使用 `slack_read_channel`。
- **私密频道访问。**需要包含私密频道时使用 `slack_search_public_and_private`，但注意这需要用户同意。
