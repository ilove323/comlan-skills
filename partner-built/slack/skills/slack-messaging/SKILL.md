# Slack 消息最佳实践

此技能提供撰写格式规范、有效的 Slack 消息的指导。

## 适用场景

每当撰写、起草或帮助用户写 Slack 消息时应用此技能 — 包括使用 `slack_send_message`、`slack_send_message_draft` 或 `slack_create_canvas` 时。

## Slack 格式（mrkdwn）

Slack 使用自己的标记语法，称为 **mrkdwn**，与标准 Markdown 不同。撰写 Slack 消息时始终使用 mrkdwn：

| 格式 | 语法 | 注意事项 |
|------|------|---------|
| 粗体 | `*文本*` | 单个星号，不要用双星号 |
| 斜体 | `_文本_` | 下划线 |
| 删除线 | `~文本~` | 波浪号 |
| 代码（行内） | `` `代码` `` | 反引号 |
| 代码块 | `` ```代码``` `` | 三反引号 |
| 引用 | `> 文本` | 尖括号 |
| 链接 | `<url\|显示文本>` | 在尖括号中用竖线分隔 |
| @用户 | `<@U123456>` | 尖括号中的用户 ID |
| #频道 | `<#C123456>` | 尖括号中的频道 ID |
| 项目列表 | `- 项目` 或 `• 项目` | 破折号或项目符号字符 |
| 编号列表 | `1. 项目` | 数字后跟句点 |

### 常见错误避免

- **不要**使用 `**粗体**`（双星号）— Slack 使用 `*粗体*`（单星号）
- **不要**使用 `## 标题` — Slack 不支持 Markdown 标题。改用单独一行上的 `*粗体文本*`。
- **不要**使用 `[文本](url)` 作为链接 — Slack 使用 `<url|文本>` 格式
- **不要**使用 `---` 作为水平分割线 — Slack 不渲染这些

## 消息结构指南

- **以重点开头。**将最重要的信息放在第一行。很多人在手机上或通过通知阅读 Slack，只显示第一行。
- **保持简洁。**目标是 1-3 个短段落。如果消息很长，考虑使用画布。
- **多用换行。**大段文字很难阅读。用空行分隔不同的想法。
- **对列表使用项目符号。**任何包含 3 个以上项目的内容应该是列表，而非流水句。
- **加粗关键信息。**对姓名、日期、截止日期和行动项使用 `*粗体*`，扫描时一眼看出。

## 线程与频道礼仪

- **在线程中回复**特定消息，以保持主频道整洁。
- **使用 `reply_broadcast`**（同时发布到频道）只有在回复包含所有人都需要看到的信息时使用。
- **在频道中发帖**（不是线程）用于开始新话题、发布公告或向整个群体提问。
- **不要发起新线程**来继续现有对话 — 找到原始消息并回复它。

## 语气与受众

- 将语气与频道匹配 — `#general` 通常比 `#random` 更正式。
- 对简单的确认使用 emoji 反应，而非回复消息（注意：MCP 工具无法添加反应，如果适当，建议用户手动操作）。
- 撰写公告时，使用清晰的结构：背景、关键信息、行动号召。
