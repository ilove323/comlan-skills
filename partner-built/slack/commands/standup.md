---
name: standup
description: 根据您最近的 Slack 活动生成站会更新。触发词：/standup、站会、日会更新
user-invocable: true
---

1. 使用 `slack_read_user_profile`（不传 user_id）获取当前用户的档案信息，包括他们的用户 ID 和显示名称。

2. 使用 `slack_search_public` 搜索用户的最近消息，过滤条件为 `from:<@USER_ID>`，`after:` 设为昨天的日期。这将捕获上一个工作日的消息。

3. 查看找到的消息，将其分类为站会主题：
   - **我完成的工作** — 用户讨论或贡献的话题、项目或任务
   - **我接下来要做的工作** — 任何对即将进行的工作、计划或跟进的提及
   - **阻碍** — 任何未被回答的提问、提出的问题，或明确提到被卡住的情况

4. 对于讨论串中的消息，使用 `slack_read_thread` 获取完整上下文，以便准确描述用户的贡献内容。

5. 将站会格式化为：
   ```
   *<显示名称>的站会 — <今日日期>*

   *已完成：*
   - 事项 1
   - 事项 2

   *进行中：*
   - 事项 1

   *阻碍：*
   - 无 / 事项 1
   ```

6. 向用户呈现站会以供审查。他们可以编辑、调整，或要求您将其发布到特定频道。
