为 "$ARGUMENTS" 生成全面的账户计划。

## 流程

1. **完整账户研究** — 遵循账户研究技能，生成完整的账户概览。提取所有可用的第一方（产品）、第二方（社区）和第三方（意图）信号、CRM 数据、评分和 RoomieAI 研究。

2. **利益相关者映射** — 按成员评分降序获取此公司的前 5 位联系人。将每位分类为：冠军、经济买家、影响者、终端用户或未知。如果可用，使用 Spark 角色数据；如果 Spark 不可用，从参与模式和活动时效性推断。如果也没有活动数据，分类为未知。

3. **参与度分析** — 提取过去 90 天的所有组织活动（最多 50 个活动）。识别趋势：参与度是增长、稳定还是下降？哪些联系人最活跃？他们通过哪些渠道参与？

4. **网络搜索（补充）** — 如果 CR 数据丰富，跳过。如果数据稀疏或用户有要求，搜索过去 30 天内的公司近期新闻：融资、收购、产品发布、领导层变动、竞争动态。

5. **综合** — 将所有数据整合为结构化账户计划。当用户的公司上下文可用时（参阅 `references/my-company-context.md`），根据用户的产品和 ICP 定制执行摘要、机会和行动项。

## 输出格式

```
## 账户计划：[公司名称]

### 执行摘要
[3-4 句话：关系状态、关键机会、主要风险、推荐优先级]

### 账户概览
| 字段 | 值 |
|------|----|
| 行业 | ... |
| 规模 | ... |
| 域名 | ... |
| CRM 归属 | ... |
| 机会阶段 | ... |
| ARR | ... |
| 评分 | ... |

### 利益相关者地图

**冠军**
- [姓名] — [职位] — [关键信号，最后活动日期]

**经济买家**
- [姓名] — [职位] — [关键信号]

**影响者**
- [姓名] — [职位] — [关键信号]

**终端用户**
- [姓名] — [职位] — [关键信号]

### 参与度分析
[趋势摘要：增长/稳定/下降，最活跃的联系人，顶级渠道，与 90 天前的对比（如果有数据）]

### 近期新闻 [如果运行了网络搜索]
[网络搜索发现，附来源和日期]

### 机会
1. [基于信号的机会，附具体下一步]
2. ...

### 风险
1. [基于信号的风险，附缓解措施]
2. ...

### 优先行动项
1. [具体行动] — [负责人建议] — [时间线]
2. ...
3. ...
```

将每项见解都建立在实际数据上。当数据稀疏或不可用时，明确标注。

**如果 Common Room 对此账户返回的数据稀疏**，生成仅涵盖有真实数据的部分的简化计划。不要从最少输入生成完整的账户计划 — 一个带有明显缺口注明的简短诚实计划，比一个建立在虚构细节上的看起来全面的计划有用得多。如果没有数据支持，完全省略利益相关者地图、参与度分析、机会、风险等部分。
