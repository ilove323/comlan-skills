# 我的公司上下文

## 用途

某些技能在了解用户公司 — 销售的产品、ICP 和竞争定位 — 时能产出更好的结果。此上下文让 LLM 能够根据用户的特定产品和价值主张定制账户研究、通话准备和外联内容。

## 如何收集公司上下文

1. **Me 对象** — 获取 `Me` 对象。它包含用户的公司名称和职位。
2. **CR 查找** — 在 Common Room 中将用户的公司作为 Organization 查找，以获取企业画像、行业和描述。
3. **网络搜索** — 如果 CR 数据稀疏或看起来过时，搜索用户公司的客户案例研究、产品页面和营销素材。这用新鲜的定位上下文填补了仅靠 CR 企业画像无法捕获的空缺。
4. **会话上下文** — 检查用户是否已在当前对话中描述了他们的产品、ICP 或价值主张。
5. **询问一次** — 如果以上方法都无法获取公司上下文，每次会话询问用户一次："为了量身定制我的建议，您能简要描述一下您的公司是做什么的，以及您的理想客户是谁吗？"

## 如何使用

- **账户研究（推理步骤）：**相对于用户的 ICP 定位发现 — "此账户符合您的 ICP，因为..."
- **通话准备（话语点）：**围绕用户的产品价值而非通用建议来构建话语点
- **撰写外联（价值桥梁）：**将推介建立在用户特定的产品和定位上
- **潜客挖掘：**当用户未指定条件时，将 ICP 标准用作默认过滤建议
