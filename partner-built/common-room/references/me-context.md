# `Me` 对象 — 用户上下文指南

## `Me` 是什么

Common Room 中的 `Me` 对象代表当前已认证的用户。在会话开始时获取它可以为 LLM 提供上下文，用于个性化输出并正确限定查询范围。

在运行账户研究、潜客挖掘或任何应限定在用户领域范围内的工作流之前，始终获取 `Me`。

## `Me` 返回的内容

### 用户档案
- 登录标识符（邮箱或用户名）
- 全名和显示名称
- 职位和角色
- CR 中的 Persona（例如，客户经理、SDR、CSM、管理者）
- 所有关联档案（例如，Salesforce 用户 ID、LinkedIn 账号）

### 用户细分市场（"我的细分市场"）
- 属于此用户的所有细分市场的列表（各包含名称和细分市场 ID）
- 对应 Common Room 产品中的**"我的细分市场"标签页**

## 如何使用 `Me` 上下文

### 1. 限定查询范围并尊重领域边界
在运行账户研究、潜客挖掘或生成简报时，将结果过滤到用户自己的细分市场范围内，除非他们要求更广泛的视图。

> 默认："向我显示显示购买信号的账户" -> 限定在我的细分市场
> 覆盖："向我显示工作区中所有显示购买信号的账户" -> 移除细分市场范围

如果用户询问不在其细分市场中的账户，注意："此账户似乎不在您的细分市场中 — 您还想让我研究它吗？"

### 2. 个性化外联和简报
使用用户的姓名、职位和角色来个性化输出 — 例如，在每周简报中引用他们的领域，在起草的邮件中使用他们的姓名。

### 3. 推断上下文进行推理
用户在 CR 中的 Persona 影响输出的框架方式：
- **客户经理 / 客户管理经理 / Account Executive / Account Manager** — 专注于管道、交易、扩展、成交时间线
- **SDR / BDR / 销售发展代表 / 业务发展代表** — 专注于潜客挖掘、热信号、首次外联
- **CSM / 客户成功经理** — 专注于健康度、留存、扩展、冠军参与
- **管理者 / 总监 / VP** — 专注于团队级趋势，而非个人外联
