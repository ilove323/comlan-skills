# 每周准备简报

生成一份单一的全面每周简报，涵盖未来 7 天内每一个外部客户或潜在客户通话，结合 Common Room 的账户和联系人研究。

## 简报流程

### 第一步：获取本周的外部会议

**选项 A — 已连接日历：**
使用 `~~calendar` 连接器获取未来 7 天（或用户指定的范围）内排定的所有会议。筛选只保留外部会议 — 那些有来自您组织之外参与者的会议。排除仅限内部的会议、与同事的一对一，以及反复出现的内部同步。

为每次外部会议识别：
- 公司名称
- 会议日期和时间
- 外部参与者姓名和邮件地址

**选项 B — 未连接日历：**
询问用户："要构建您的每周准备简报，我需要您即将进行的外部通话。请列出：公司名称、日期/时间和参与者姓名。"

接受自由格式输入并在继续之前将其解析为结构化列表。

### 第二步：向用户确认会议列表

在开始研究之前，向用户呈现识别到的会议进行确认：

> "以下是我找到的本周外部通话。如果有遗漏或需要排除的，请告诉我：
> - [公司] — [日期]，[时间] — [参与者]
> - ..."

这防止在取消或错误的会议上浪费研究精力。

### 第三步：研究每次会议

对于每次已确认的外部会议，在可能的情况下并行运行：
1. **账户研究** — 使用账户研究技能的完整账户快照
2. **联系人研究** — 使用联系人研究技能对每位外部参与者进行档案研究

Common Room 数据是主要来源。CR 研究后，对每家公司运行快速**时效性检查** — 这是补充性的，而非主要的：
- 在过去 7 天内限定范围搜索 `"[公司名称]" news`
- 对高管参与者，搜索其姓名以找近期公开帖子或采访
- 只包含真正值得关注的发现（融资、领导层变动、重大媒体报道）。不要用通用新闻填充简报。

研究深度校准：
- 对于高优先级账户（大账户、开放机会、续约风险），进行完整深度的研究
- 对于优先级较低或会议时间较短的，进行简短快照（每个 3-4 个要点）

### 第四步：综合每周简报

将所有每次会议的研究整合到一份单一的结构化文档中，按会议日期/时间排序。

以简短的本周级别概述开头，标注：
- 任何有紧急信号的账户（处于风险、试用即将到期、扩展机会）
- 任何需要特殊准备或高管参与的会议
- 外部通话总数和预计时间投入

## 输出格式

```
# 每周准备简报 — [日期] 当周

## 本周概述
[2-4 个要点：关键主题、标注的优先级、通话数量]

---

## [周一/周二/等]

### [公司名称] — [时间]
**参与者：**[姓名和职位]
**会议类型：**[发现/QBR/续约/扩展/等 — 如可能推断]

**公司快照**
[4-5 个要点：账户状态、顶级信号、近期活动]

**参与者档案**
- **[姓名]**（[职位]）：[2-3 个关于他们的信号、角色、对话角度的要点]
- [对每位参与者重复]

**本周顶级信号**
[与此次特定通话最相关的 2-3 个信号]

**本周新闻** [如果找到值得关注的新闻]
[只有真正值得关注的发现 — 融资、领导层变动、重大媒体报道]

**推荐目标**
[1-2 句话：这次会议要完成什么]

---

[对每次会议重复，按日期/时间排序]
```

## 当某次会议的数据稀疏时

如果 Common Room 对某次会议的账户或参与者返回的数据有限，使用压缩格式代替完整模板：

```
### [公司名称] — [时间] ⚠️ 数据有限
**参与者：**[如已知，姓名和职位]
**可用数据：**[Common Room 实际返回的内容]

**网络搜索结果**
[来自网络搜索的发现 — 公司新闻、参与者 LinkedIn 档案]

**说明：**Common Room 对此账户的数据有限。代表可能需要直接在 CR 中检查，或在此通话前向同事收集背景信息。
```

不要从稀疏数据生成完整的会议准备部分（公司快照、信号亮点、话语点、推荐目标）。简短诚实的部分比虚构的完整内容更有用。

## 质量标准

- 保持每个会议部分可以快速扫描 — 代表在早上阅读这些，通常在手机上
- 始终按日期/时间升序排序
- 突出标注紧急情况（风险、试用到期、开放机会）— 不要埋没它们
- 如果某次会议在 Common Room 中的数据非常少，使用上面的稀疏数据格式 — 绝不用猜测填充完整模板
- 对于 4-6 次会议的一周，整体简报应在 10-15 分钟内可读
- **每项事实都必须来自工具调用** — 不虚构交易背景、活动或信号
