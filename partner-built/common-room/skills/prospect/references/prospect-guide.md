# Common Room Prospector — 使用指南

## 两种对象类型（关键）

绝对不要混淆这两种类型 — 它们有不同的字段和过滤器集合。

| | `ProspectorOrganization`（净新） | `Organization`（已在 CR 中） |
|---|---|---|
| **来源** | Common Room 的外部数据图 | CR 工作区（完整丰富信息 + 信号历史） |
| **可用字段** | 公司名称、域名、规模、行业、融资额、营收、地点 | 一切 — 信号、评分、CRM 数据、RoomieAI、社区活动 |
| **可用过滤器** | 仅企业画像和技术栈 | 所有企业画像过滤器加上基于信号、基于评分、基于细分市场和 CRM 过滤器 |
| **用于** | 领域规划、漏斗顶部列表建立 | 优先处理热门账户、扩展候选者、意图信号 |

## 决定使用哪种对象类型

| 用户说... | 可能的对象类型 |
|-----------|---------------|
| "我的哪些账户显示购买信号？" | `Organization` |
| "找到伦敦我还没接触过的金融科技公司" | `ProspectorOrganization` |
| "找到符合我们 ICP 的新公司" | `ProspectorOrganization` |
| "显示 90 天内没有参与的账户" | `Organization` |
| 模糊 — 两者都可能适用 | 询问："您是在寻找净新公司，还是在筛选工作区中已有的账户？" |

## 迭代细化

- 从用户的初始条件开始
- 对于大量结果（50+），先返回数量："我找到了 500 个结果。想按规模或技术栈进一步筛选吗？"
- 将后续细化接受为过滤器调整，而非新的搜索
- 如果结果少于 5 个，建议放宽一个条件

## 常见陷阱

- **混淆对象类型** — 绝不将 ProspectorOrganization 和 Organization 的结果混在同一列表中。
- **不限定"我的细分市场"范围** — 查询 Organization 记录时，默认限定在用户的细分市场范围内。
- **过度过滤** — 如果少于 5 个结果，建议放宽一个条件。
