# 撰写外联

针对特定公司或联系人，以 Common Room 信号为基础，生成三种个性化外联格式 — 邮件、通话脚本和 LinkedIn 消息。

## 外联流程

### 第一步：查找目标

使用 Common Room MCP 工具查找并检索目标（公司和/或特定联系人）的数据。提取：
- 最近的产品活动和参与信号
- 社区活动（帖子、问题、反应）
- 第三方意图信号（招聘信息、新闻、融资）
- 关系历史（先前联系、会议、邮件打开）

如果用户指定了一个人，运行联系人级别的研究。如果只给出了公司，根据职位、参与度和角色确定最佳联系目标。

### 第二步：外部钩子的网络搜索（如果 CR 信号稀疏）

如果 CR 返回了强信号（近期活动、参与、产品使用），这些应该驱动个性化 — 跳过网络搜索。如果 CR 信号稀疏或潜在客户在 CR 中几乎没有活动，运行网络搜索获取外部钩子：

**搜索内容：**
- `"[公司名称]" funding OR acquisition OR launch OR announcement` — 过去 30 天
- `"[联系人全名]" "[公司名称]"` — 查找近期文章、采访、LinkedIn 帖子或会议演讲

**优先选择具有以下特征的外部钩子：**
- 非常近期（< 2 周）— 潜在客户可能还在想这件事
- 公开可见 — 他们知道您可能已经看到了
- 变化信号 — 增长、新角色、新产品、新市场

如果用户明确要求网络搜索或外部钩子，无论 CR 信号丰富程度如何都运行。

### 第三步：Spark 丰富（如果可用）

如果 Spark 可用，对目标联系人运行丰富以获取角色分类、背景和影响力信号。使用此信息校准语气和消息角度。

### 第四步：确定最佳钩子

从信号数据中确定 1-3 个最强的个性化钩子。按以下排名：
1. **时效性** — 在过去 7-14 天内发生
2. **具体性** — 他们采取的具体行动，而非一般趋势
3. **相关性** — 直接与您产品提供的价值相关

好的钩子：在社区发帖询问关于 X 的问题、刚雇了 5 名工程师、最近开始使用 [功能]、公司刚完成 B 轮融资、试用即将到期、冠军刚换工作。

差的钩子："我注意到您是一位客户"或通用行业趋势。

### 第五步：生成全部三种格式

使用最强的钩子撰写全部三种格式。每种格式有不同的约束和惯例 — 遵循 `references/outreach-formats-guide.md` 中的格式专属指南。

始终生成全部三种，并清晰标注。

当用户的公司上下文可用时（参阅 `references/my-company-context.md`），将价值桥梁和推介内容建立在用户特定产品和定位上。

### 第六步：注释您的选择

三份草稿后，包含简短说明（2-4 句话）解释：
- 使用了哪些信号以及为什么选择它们
- 做出的任何假设（例如，推断的通话目标）
- 如果主要钩子无效，可选的备选角度

## 输出格式

```
## [姓名/公司] 的外联

### 邮件

**主题：**[主题行]

[邮件正文 — 3-5 句话]

---

### 通话脚本

**开场：**
[开场语 — 对话式，1-2 句话]

**价值桥梁：**
[为什么打这个电话，为什么是现在 — 2-3 句话，与信号相关]

**请求：**
[单一、低摩擦的请求 — 例如，15 分钟通话，具体问题]

---

### LinkedIn 消息

[300 字符以内。温暖、个人、不推销。]

---

### 信号说明
[2-4 句话：使用了哪些信号，为什么，以及备选角度]
```

## 信号数据稀疏时

如果 Common Room 对目标的数据很少（例如，只有姓名、职位、标签 — 无活动、无评分、无 Spark）：

1. **不要凭空起草外联。**基于虚构信号的外联比没有外联更差。
2. **先运行网络搜索** — 这成为您的主要个性化来源。查找近期新闻、LinkedIn 帖子、会议演讲、公司公告。
3. **如果网络搜索结果也很少**，诚实地呈现您有的内容并向用户询问上下文：

```
## [姓名/公司] 的外联 — 数据有限

**找到的内容：**
[只有来自 CR 和网络搜索的真实数据]

**我没有足够的信号来起草个性化外联。**要写出有力的内容，我需要：
- 近期活动或参与信号
- 您从之前对话中掌握的背景
- 现在联系的具体原因

您能分享以上任何信息吗？
```

## 质量标准

- 每条消息必须引用具体内容 — 通用外联不是可接受的输出
- 根据情境匹配语气：对于入站/社区信号，温暖友好；对于冷启动/高管外联，更正式
- LinkedIn 消息必须在 300 字符以内 — 不允许例外
- 通话脚本必须能自然朗读 — 在心里大声朗读检查节奏
- **不要虚构信号** — 只引用从 Common Room 或网络搜索检索到的数据
