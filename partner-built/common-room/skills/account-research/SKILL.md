# 账户研究

从 Common Room 检索并综合账户信息。处理四种交互模式：完整概览、目标字段问题、稀疏数据情况，以及 MCP 数据 + LLM 推理的组合。

## 第零步：加载用户上下文（Me）

在研究任何账户之前，从 Common Room 获取 `Me` 对象。这提供：
- 用户的档案、职位、角色和 CR 中的 Persona
- 用户的细分市场（"我的细分市场"）

除非用户明确要求更广泛的视图，否则所有查询默认限定在用户自己的细分市场范围内。这将结果限定在他们的领域范围内。

## 第一步：识别交互模式

在决定获取多少数据之前，先确定用户实际需要什么：

**模式 1 — 完整概览：**"告诉我关于 Datadog 的信息" / "总结 cloudflare.com"
→ 获取完整字段集并生成结构化简报。

**模式 2 — 目标问题：**"谁负责 Snowflake 账户？" / "acme.io 是否显示购买信号？" / "notion.so 的员工人数是多少？"
→ 只获取相关字段。返回直接、简洁的回答 — 不要因为一个简单问题就生成完整简报。

**模式 3 — 稀疏数据：**"告诉我关于 tiny-startup.io 的信息"
→ 如果 Common Room 对某账户的数据有限，如实说明："此账户的可用信息有限。"不要推测或用通用陈述填充空缺。

**模式 4 — 综合推理：**获取结构化 MCP 数据，然后叠加 LLM 分析 — 例如："Stripe 有 8,000 名员工，正在大量招聘 AI 职位。根据您针对 1k-10k 金融科技公司的 ICP，这是一个强匹配。"

## 第二步：查找账户

通过域名或公司名称在 Common Room 中搜索账户。优先精确匹配；如果没有结果，尝试模糊匹配，并在继续之前向用户确认。

## 第三步：获取正确的字段

使用 Common Room 对象目录查看可用的字段组及其内容。对于完整概览，请求所有字段组。对于目标问题，只请求相关内容。

**需要了解的关键字段组：**
- **评分** — 始终以原始值或百分位数形式返回，从不使用标签
- **摘要研究** — RoomieAI 输出；通常是最丰富的定性信号
- **顶级联系人** — 按评分降序排列；使用 communityMemberID 进行完整查找

**选择要获取的内容：**

| 用户查询类型 | 需要请求的字段 |
|-------------|---------------|
| 完整账户概览 | 所有字段组 |
| "谁负责这个账户？" | 公司档案和链接、CRM 字段 |
| "这家公司是否适合？" | 关键字段、评分、关于 |
| "这个账户显示什么信号？" | 评分、摘要研究、CRM 字段 |
| "顶级联系人是谁？" | 顶级联系人 |
| "RoomieAI 对他们有什么说法？" | 摘要研究、所有研究 |
| "找到这个账户的工程师" | 潜客（带职位过滤） |

## 第四步：网络搜索（仅稀疏数据时）

Common Room 是主要数据来源。当 CR 返回丰富数据时，不要运行网络搜索。

当 CR 数据稀疏时（模式 3 — 返回的字段少，无活动，无评分），运行针对性的网络搜索填补空缺：
- `"[公司名称]" news` — 限定在过去 30 天
- 查找：融资轮次、收购、产品发布、高管变动、媒体报道

如果用户明确要求外部背景或最新消息，无论数据丰富程度如何都运行网络搜索。

## 第五步：应用推理（模式 4）

当用户的问题涉及综合而非仅仅数据检索时，叠加分析：
- 将账户数据与会话上下文中的已知 ICP 标准对比
- 识别匹配信号（规模、行业、技术栈、招聘模式）
- 注意时机信号（融资、试用状态、近期活动激增）
- 明确表明洞见来自数据，而非假设

当用户的公司上下文可用时（参阅 `references/my-company-context.md`），相对于用户的价值主张和 ICP 对发现进行定位。

## 第六步：生成输出

只包含 Common Room 实际返回数据的部分。完全省略各部分，而非用猜测填充。

**完整概览（数据丰富时）：**

```
## [公司名称] — 账户概览

**快照**
[2-3 句话：他们做什么，计划/阶段，关系状态]

**关键详情**
[员工人数、行业、地点、域名、融资 — 来自关键字段]

**CRM 和归属** [如果 CRM 字段已返回]
[负责人、机会阶段、ARR]

**评分** [如果已返回评分]
[所有可用评分，以原始值或百分位数形式]

**信号亮点** [如果活动/信号存在]
[3-5 个最重要的信号，附日期]

**顶级联系人** [如果已返回联系人]
[姓名 | 职位 | 评分 — 按评分降序排列的前 5 位]

**RoomieAI 研究** [如果摘要研究非空]
[摘要研究输出；列出所有可用的研究主题名称]

**推荐的后续步骤**
[2-3 个具体的、基于信号的行动]
```

**目标问题：**1-3 句直接回答。不需要完整简报。

**稀疏数据（返回字段少，大多数部分会是空的）：**

```
## [公司名称] — 账户概览（数据有限）

**可用数据：**[列出 Common Room 实际返回的内容]

[只呈现已返回的字段]

**网络搜索**
[网络搜索发现 — 或"未发现近期重大新闻"]

**说明：**Common Room 对此账户的数据有限。该账户可能需要在 Common Room 中进行丰富。
```

## 质量标准

- 评分必须始终是原始值或百分位数 — 不使用分类标签
- 对于目标问题，精确回答，不要过度提供信息
- 数据缺失或过期时明确说明 — 不要推测
- 完整简报保持在 2-3 分钟内可读
- **每项事实都必须追溯到一个工具调用** — 不包含 Common Room 未返回的数据
