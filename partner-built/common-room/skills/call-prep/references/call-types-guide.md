# 通话类型 — 准备指南

## 识别通话类型

如果用户未指定，从上下文推断：试用 + 早期信号 = 发现/演示；活跃的 CRM 机会 = 演示/评估/谈判；现有客户 + 续约日期 = 续约/QBR；扩展信号 = 扩展；支持工单 + 使用量下降 = 挽留通话。

---

## 发现通话

**目标：**了解潜在客户的痛点、背景和匹配程度。

- 识别触发会议的信号，并准备与这些信号相关的开放性问题
- 根据 Spark 角色和活动数据形成关于其主要使用场景的假设
- **话语点结构：**信号 -> 问题 -> 您可能如何提供帮助

## 演示通话

**目标：**在其具体需求的背景下展示产品。

- 记录他们已经探索过的功能以及技术栈中的竞争对手
- 识别受众（技术评估者 vs 经济买家）并相应调整深度
- 根据他们的已知活动突出 1-2 个功能

## 扩展对话

**目标：**识别并完成追加销售/交叉销售或席位扩展。

- 识别近期采用率最高的产品领域和新兴团队/使用场景
- 以已交付价值的使用数据作为开场
- 将扩展呈现为与其业务成果相关的自然下一步

## 续约/留存通话

**目标：**确保续约；解决任何风险。

- 检查参与度趋势、开放的支持工单和冠军状态
- 量化已交付的价值（使用统计、成果）作为续约理由
- 主动直接地解决风险信号 — 如有必要，准备好留存方案

## QBR（季度业务回顾）

**目标：**回顾进展，对齐战略，加深高管关系。

- 分析 90 天信号趋势和各团队的采用广度
- 以业务成果为重点，而非功能使用 — 找出 1-2 个令人惊喜的统计数据
- 准备前瞻性议程和下季度目标

## 挽留/风险通话

**目标：**理解并扭转负面轨迹。

- 识别具体的下降信号（下降了什么，什么时候）并检查未解决的工单
- 不要防御性地承认信号 — 提出解决方案之前先问问题
- 准备具体的行动计划
