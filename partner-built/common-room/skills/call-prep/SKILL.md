# 通话准备

通过综合 Common Room 的账户研究、联系人研究和信号，生成完整、易于扫描的通话准备简报。

## 准备流程

### 第一步：识别账户和参与者

解析用户提供的信息：
- **公司名称** — 必填；在 Common Room 中查找账户
- **参与者姓名** — 可选；如果提供，对每个人进行研究

**日历查找：**如果有 `~~calendar` 连接器，搜索与指定公司的即将进行的会议，以自动提取参与者姓名、会议时间和任何会议记录或议程。用此信息填补用户未提供的空缺。

如果既没有提供参与者，也找不到日历匹配，询问："[公司] 方面谁会参加这次通话？我可以研究每位参与者，让您的准备更有针对性。"

### 第二步：运行账户研究

使用账户研究技能流程构建完整的账户快照。对于通话准备，优先关注：
- 最近的产品信号（他们现在在产品中做什么？）
- 开放的机会或续约时间线
- 任何风险信号（使用量下降、支持工单、已流失席位）
- 关键近期事件（融资、高管变动、新员工）

在查看活动历史时，优先处理 Gong 和通话录音活动 — 这些提供了之前对话的直接背景。不要按活动来源过滤掉通话录音。

### 第三步：对每位参与者运行联系人研究

对于每位外部参与者，使用联系人研究技能流程。对于通话准备，重点关注：
- 在购买流程中的角色和影响力
- 他们的个人活动和参与历史
- 任何表明当前情绪/优先事项的近期信号
- 如果可用，Spark 角色分类

### 第四步：综合话语点和目标

基于综合的账户和联系人研究：
- 识别**通话目标**（例如，发现、演示、扩展对话、续约、QBR）
- 生成 **3-5 个基于具体信号数据的针对性话语点**
- 预判**客户可能提出的 2-3 个可能的异议或话题**
- 建议**通话的推荐结果**

当用户的公司上下文可用时（参阅 `references/my-company-context.md`），根据用户的产品和价值主张定制话语点。

### 第五步：时效性检查（网络搜索）

收集所有 Common Room 数据后，运行快速时效性检查以捕获上次 CR 数据同步后发生的事情。这是补充性的 — CR 数据驱动准备；网络搜索只增加时效性。

**公司新闻：**搜索限定在过去 14 天的 `"[公司名称]" news`。查找融资公告、产品发布、领导层变动、裁员、合作关系或媒体报道。

**参与者动态：**对于每位外部参与者，搜索 `"[全名]" "[公司名称]"` — 查找近期文章、LinkedIn 帖子、会议演讲、播客或发表的观点。

如果公司新闻重大（例如，刚完成融资，宣布了重要招聘），在信号亮点中标注。否则，简短包含发现 — 不要让网络搜索结果盖过 CR 信号。

## 输出格式

输出根据 Common Room 返回多少数据进行适应。只包含有真实数据的部分。不要用虚构细节填充任何部分。

### 数据丰富时（返回了多个字段组、活动历史、评分、信号）：

```
## 通话准备：[公司] — [日期/时间（如已知）]

**会议背景**
[参与者、会议类型和任何已知议程]

---

### 公司快照
[4-6 个要点：账户状态关键点、信号和近期活动]

---

### 参与者档案

**[参与者姓名] — [职位]**
[3-4 个要点：角色、近期活动、Spark 角色（如可用）、个人钩子]

[对每位参与者重复]

---

### 信号亮点
[与此次特定通话最相关的前 3 个信号]

---

### 话语点
1. [与特定信号相关的要点]
2. [与特定信号相关的要点]
3. [与特定信号相关的要点]

### 可能的话题/需要准备的异议
- [话题或异议 + 建议回应]
- [话题或异议 + 建议回应]

### 推荐的通话结果
[1-2 句话：这次会议成功的样子]
```

### 数据稀疏时（返回字段少，无活动，sparkSummary 为空）：

```
## 通话准备：[公司] — [日期/时间（如已知）]

**可用数据：**[列出 Common Room 实际返回的内容 — 例如，"姓名、职位、邮箱、两个标签。无活动历史，无评分，无 Spark 数据。"]

### 找到的信息
[只呈现实际返回的字段，原样展示]

### 网络搜索结果
[来自公司和参与者网络搜索的发现 — 或"无重要结果"]

### 建议的后续步骤
- 如果可用，我可以从 Common Room 提取 [特定字段组]
- 我可以对 [特定话题] 进行更深入的网络搜索
- 您可能需要直接在 Common Room 中检查 [缺少的内容]
```

不要从稀疏数据生成完整的通话准备简报。简短诚实的输出始终比长篇虚构的输出更好。

## 质量标准

- 每个话语点都要以真实信号为依据 — 不要通用填充
- 保持简报紧凑 — 应在 5 分钟内可读
- 明确标注未知信息 — 如果参与者研究数据较少，如实说明
- 时间管理研究 — 不要过度研究而牺牲速度
- **切勿虚构交易背景** — 不虚构提案、竞争对手比较、定价、试用条款或工具调用未返回的异议
