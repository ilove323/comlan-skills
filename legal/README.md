# 法务效率插件

一款为企业内部法务团队设计的 AI 效率插件，适用于 [OpenClaw](https://openclaw.ai)，同时也支持在 Claude Code 中使用。可自动化处理合同审查、NDA 分类、合规工作流、法律简报和模板回复——所有功能均可根据您组织的具体规范手册和风险偏好进行配置。

> **免责声明：** 本插件协助处理法律工作流，但不提供法律建议。请务必由具备资质的法律专业人员对结论进行核实。AI 生成的分析在用于法律决策前，须经执业律师审阅。本插件默认规范手册示例反映的是美国法律立场和司法管辖区（特拉华州、纽约州、加利福尼亚州）。如果您在不同法律体系下运营（欧盟、英国、荷兰、澳大利亚等），在使用本插件的分析结果之前，必须自定义 `.openclaw/legal.local.md` 中的规范手册，以反映您所在司法管辖区的具体法律要求、标准合同条款和合规义务。

## 目标用户

- **商业法律顾问** — 合同谈判、供应商管理、交易支持
- **产品法律顾问** — 产品审查、服务条款、隐私政策、知识产权事务
- **隐私/合规** — 数据保护法规、DPA 审查、数据主体请求、监管动态
- **诉讼支持** — 证据保全、文件审查准备、案件简报

## 安装

```
openclaw skills install legal
```

或将文件夹复制到 `~/.openclaw/skills/legal`。

## 快速开始

### 1. 安装插件

```
openclaw skills install legal
```

### 2. 配置您的规范手册

创建一个本地配置文件，定义您组织的标准立场。在此编码您团队的谈判规范手册、风险容忍度和标准条款。

在 AI助手 可以找到的位置创建 `legal.local.md` 文件：

- **OpenClaw**：保存在您与 OpenClaw 共享的任意文件夹中（通过文件夹选择器）。插件会自动找到它。
- **OpenClaw**：保存在项目的 `.openclaw/` 目录中。

```markdown
# 法律规范手册配置

## 合同审查立场

### 责任限制
- 标准立场：双方责任上限为已支付/应付费用的 12 个月金额
- 可接受范围：费用的 6-24 个月
- 上报触发条件：无上限责任、包含间接损害赔偿

### 赔偿
- 标准立场：双方就知识产权侵权和数据泄露互相赔偿
- 可接受：仅限于第三方索赔的赔偿
- 上报触发条件：单方面赔偿义务、无上限赔偿

### 知识产权归属
- 标准立场：各方保留既有知识产权；客户拥有客户数据
- 上报触发条件：宽泛的知识产权转让条款、既有知识产权的雇佣作品条款

### 数据保护
- 标准立场：任何个人数据处理均需要 DPA
- 要求：分处理方通知、终止时数据删除、72 小时内泄露通知
- 上报触发条件：不提供 DPA、无保护措施的跨境传输

### 期限和终止
- 标准立场：一年期限，提前 30 天终止便利权
- 可接受：初始期限后具有终止便利权的多年期
- 上报触发条件：无通知期的自动续期、无终止便利权

### 适用法律
- 首选：[您的司法管辖区]
- 可接受：主要商业司法管辖区（纽约州、特拉华州、加利福尼亚州、英格兰和威尔士）
- 上报触发条件：非标准司法管辖区、在不利场所的强制仲裁

## NDA 默认值
- 需要双方义务
- 期限：标准 2-3 年，商业秘密 5 年
- 标准豁免：独立开发、公开可用、合法从第三方获取
- 残留条款：如范围有限则可接受

## 回复模板
配置模板文件路径或为常见查询定义内联模板。
```

### 3. 连接您的工具

本插件与通过 MCP 接入的现有工具配合使用效果最佳。预配置的服务包括 Slack、Box、Egnyte、Atlassian 和 Microsoft 365。完整支持的类别和选项列表请参阅 [CONNECTORS.md](CONNECTORS.md)。

## 命令

### `/review-contract` — 对照规范手册进行合同审查

对照您组织的谈判规范手册审查合同。标记偏差、生成修订意见，并提供商业影响分析。

```
/review-contract
```

接受：文件上传、URL 或粘贴的合同文本。将询问背景（您的立场、截止日期、重点领域），并逐条对照您配置的规范手册进行审查。

### `/triage-nda` — NDA 预筛选

快速对照标准标准对传入的 NDA 进行分类。分类为绿色（标准批准）、黄色（法律顾问审查）或红色（重大问题）。

```
/triage-nda
```

### `/vendor-check` — 供应商协议状态

检查您连接的系统中与某供应商现有协议的状态。

```
/vendor-check [供应商名称]
```

报告现有 NDA、MSA、DPA、到期日期和关键条款。

### `/brief` — 法务团队简报

为您的法务工作生成上下文简报。

```
/brief daily          # 早间法务相关事项简报
/brief topic [查询]   # 特定法律问题的研究简报
/brief incident       # 突发情况的快速简报
```

### `/respond` — 生成模板回复

从您配置的模板中生成常见查询类型的回复。

```
/respond [查询类型]
```

支持的查询类型包括：数据主体请求、证据保全、供应商问题、NDA 请求以及您自定义的类别。

## 技能

| 技能 | 描述 |
|------|------|
| `contract-review` | 基于规范手册的合同分析、偏差分类、修订意见生成 |
| `nda-triage` | NDA 筛选标准、分类规则、路由建议 |
| `compliance` | 隐私法规（GDPR、CCPA）、DPA 审查、数据主体请求 |
| `canned-responses` | 模板管理、回复类别、上报触发器 |
| `legal-risk-assessment` | 风险严重程度框架、分类级别、上报标准 |
| `meeting-briefing` | 会议准备方法论、背景收集、行动项跟踪 |

## 示例工作流

### 合同审查

1. 通过邮件收到供应商合同
2. 运行 `/review-contract` 并上传文件
3. 提供背景："我们是客户，需要在本季度末完成，重点关注数据保护和责任"
4. 获得逐条分析，标注绿色/黄色/红色标志
5. 获取黄色和红色条款的具体修订措辞
6. 与您的交易团队分享分析结果

### NDA 分类

1. 销售团队发来新潜客的 NDA
2. 运行 `/triage-nda` 并粘贴或上传 NDA
3. 获得即时分类：绿色（直接签署）、黄色（需审查的具体问题）或红色（需要完整法律顾问审查）
4. 绿色 NDA 直接批准；黄色/红色处理标记的问题

### 每日简报

1. 早上用 `/brief daily` 开始工作
2. 获取隔夜合同请求、合规问题、即将到期的截止日期以及需要法律准备的日历事项摘要
3. 根据紧急程度和截止日期安排当天工作优先级

### 供应商核查

1. 业务团队询问与现有供应商的新合作事项
2. 运行 `/vendor-check Acme Corp`
3. 一览现有协议、到期日期和关键条款
4. 立即了解是否需要新 NDA 或可在现有条款下推进

## MCP 集成

> 如果遇到不熟悉的占位符或需要查看已连接的工具，请参阅 [CONNECTORS.md](CONNECTORS.md)。

本插件通过 MCP（Model Context Protocol）服务器连接您的工具：

| 类别 | 示例 | 用途 |
|------|------|------|
| 即时通讯 | Slack、Teams | 团队请求、通知、分类 |
| 云存储 | Box、Egnyte | 规范手册、模板、先例文件 |
| 办公套件 | Microsoft 365 | 邮件、日历、文档 |
| 项目跟踪器 | Atlassian（Jira/Confluence） | 事项跟踪、任务管理 |

完整支持的集成列表（包括 CLM、CRM、电子签署等更多选项）请参阅 [CONNECTORS.md](CONNECTORS.md)。

在 `~/.openclaw/openclaw.json` 中配置连接。工具不可用时，插件会优雅降级——会标注缺口并建议手动检查。

## 自定义

### 规范手册配置

规范手册是合同审查系统的核心。在 `legal.local.md` 中定义您的立场：

- **标准立场**：您首选的合同条款
- **可接受范围**：无需上报即可接受的条款
- **上报触发器**：需要高层审查或外部法律顾问的条款

### 回复模板

定义常见查询的模板。模板支持变量替换，并内置上报触发器，用于不应使用模板回复的情况。

### 风险框架

自定义风险评估矩阵，以匹配您组织的风险偏好和分类方案。
