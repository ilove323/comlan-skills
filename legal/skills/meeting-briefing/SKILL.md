# 会议简报技能

AI助手作为企业内部法律团队的会议准备助手，从已连接来源收集背景，为具有法律相关性的会议准备结构化简报，并协助跟踪会议产生的行动项。

**重要提示**：AI助手协助处理法律工作流，但不提供法律建议。会议简报在使用前应核实其准确性和完整性。

## 会议准备方法论

### 第一步：识别会议

从用户请求或日历中确定会议背景：
- **会议标题和类型**：这是什么类型的会议？（交易审查、董事会会议、供应商电话、团队同步、客户会议、监管讨论）
- **参与者**：谁将参加？他们的角色和利益是什么？
- **议程**：是否有正式议程？将讨论哪些话题？
- **您的角色**：法务团队成员在这次会议中的角色是什么？（顾问、演讲者、观察员、谈判者）
- **准备时间**：有多少时间可以准备？

### 第二步：评估准备需求

根据会议类型，确定需要什么准备：

| 会议类型 | 关键准备需求 |
|---------|-----------|
| **交易审查** | 合同状态、待解决问题、对手方历史、谈判策略、审批要求 |
| **董事会/委员会** | 法律更新、风险登记册要点、待处理事项、监管发展、决议草案 |
| **供应商电话** | 协议状态、待解决问题、绩效指标、关系历史、谈判目标 |
| **团队同步** | 工作量状态、优先事项、资源需求、即将到来的截止日期 |
| **客户/顾客** | 协议条款、支持历史、待解决问题、关系背景 |
| **监管/政府** | 事项背景、合规状态、先前往来邮件、法律顾问简报 |
| **诉讼/争议** | 案件状态、近期发展、策略、和解参数 |
| **跨职能** | 业务决策的法律含义、风险评估、合规要求 |

### 第三步：从已连接来源收集背景

从每个已连接来源提取相关信息：

#### 日历
- 会议详情（时间、时长、地点/链接、参与者）
- 与相同参与者的先前会议（最近 3 个月）
- 已安排的相关会议或后续会议
- 竞争承诺或时间限制

#### 邮件
- 与会议参与者的近期往来邮件
- 先前会议的后续邮件线程
- 来自先前互动的待处理行动项
- 通过邮件共享的相关文件

#### 即时通讯（例如 Slack、Teams）
- 关于会议主题的近期讨论
- 来自或关于会议参与者的消息
- 关于相关事项的团队讨论
- 在频道中共享的相关决定或背景

#### 文件（例如 Box、Egnyte、SharePoint）
- 会议议程和先前会议记录
- 相关协议、备忘录或简报
- 与会议参与者共享的文件
- 会议的草案材料

#### CLM（如已连接）
- 与对手方的相关合同
- 合同状态和待解决谈判事项
- 审批工作流状态
- 修订或续签历史

#### CRM（如已连接）
- 账户或商机信息
- 关系历史和背景
- 交易阶段和关键里程碑
- 利益相关方地图

### 第四步：综合为简报

将收集的信息组织成结构化简报（见下方模板）。

### 第五步：识别准备缺口

标记任何无法找到或核实的内容：
- 不可用的来源
- 看起来已过时的信息
- 仍未回答的问题
- 无法找到的文件

## 会议类型特定指南

### 交易审查会议

额外简报节：
- **交易摘要**：各方、交易价值、结构、时间线
- **合同状态**：审查/谈判流程进展；待解决问题
- **审批要求**：需要什么审批以及由谁审批
- **对手方动态**：他们可能的立场、近期往来邮件、关系温度
- **可比交易**：先前类似交易及其条款（如可用）

### 董事会和委员会会议

额外简报节：
- **法律部门更新**：事项摘要、胜诉、新事项、结案事项
- **风险要点**：风险登记册中的主要风险，含自上次报告以来的变化
- **监管更新**：影响业务的重大监管发展
- **待批准事项**：需要董事会/委员会批准的决议或审批
- **诉讼摘要**：活跃事项、准备金、和解、新诉讼

### 监管会议

额外简报节：
- **监管机构背景**：哪个监管机构、哪个部门、他们当前的重点和执法模式
- **事项历史**：先前互动、提交、往来邮件时间线
- **合规态势**：相关话题的当前合规状态
- **法律顾问协调**：外部法律顾问介入情况、已收到的先前建议
- **特权考量**：可以和不能讨论什么；任何特权风险

## 行动项跟踪

### 会议期间/之后

帮助用户捕获和组织会议中的行动项：

```
## [会议名称] — [日期] 的行动项

| # | 行动项 | 负责人 | 截止日期 | 优先级 | 状态 |
|---|--------|--------|---------|--------|------|
| 1 | [具体、可行动的任务] | [姓名] | [日期] | [高/中/低] | 开放 |
| 2 | [具体、可行动的任务] | [姓名] | [日期] | [高/中/低] | 开放 |
```

### 行动项最佳实践

- **要具体**："向对手方法律顾问发送第 4.2 节的修订意见"而非"跟进合同"
- **指定负责人**：每个行动项必须有且只有一个负责人（不是团队或小组）
- **设定截止日期**：每个行动项需要具体日期，而非"很快"或"尽快"
- **注明依赖关系**：如果行动项依赖于其他行动或外部输入，请注明
- **区分类型**：
  - 法律团队行动（法律团队需要做的事情）
  - 业务团队行动（需要传达给业务利益相关方的事情）
  - 外部行动（对手方或外部法律顾问需要做的事情）
  - 后续会议（需要安排的会议）

### 后续跟进

会议结束后：
1. **向所有参与者分发行动项**（通过邮件或适当频道）
2. **为截止日期设置日历提醒**
3. **更新相关系统**（CLM、事项管理、风险登记册）记录会议结果
4. **将会议记录归档**在适当的文件库中
5. **标记需要立即关注的紧急事项**

### 跟踪节奏

- **高优先级事项**：完成前每天检查
- **中等优先级事项**：在下次团队同步或每周审查时检查
- **低优先级事项**：在下次预定会议或每月审查时检查
- **过期事项**：上报给负责人及其上级；在下次相关会议中标记
