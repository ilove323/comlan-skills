# /triage-nda — NDA 预筛选

> 如果遇到不熟悉的占位符或需要查看已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

对 NDA 进行分类：@$1

对照标准筛选标准快速分类传入的 NDA。对 NDA 进行路由分类：标准批准、法律顾问审查或完整法律审查。

**重要提示**：AI助手协助处理法律工作流，但不提供法律建议。所有分析在被依赖使用前应由具备资质的法律专业人员审阅。

## 调用方式

```
/triage-nda
```

## 工作流程

### 第一步：接受 NDA

以任何格式接受 NDA：
- **文件上传**：PDF、DOCX 或其他文档格式
- **URL**：文件系统中 NDA 的链接
- **粘贴文本**：直接粘贴的 NDA 文本

如果未提供 NDA，提示用户提供。

### 第二步：加载 NDA 规范手册

在本地设置（例如 `legal.local.md`）中查找 NDA 筛选标准。

NDA 规范手册应定义：
- 双方还是单方要求
- 可接受的期限长度
- 必需的豁免
- 禁止条款
- 组织特定要求

**如果未配置 NDA 规范手册：**
- 使用合理的市场标准默认值继续
- 清楚注明正在使用默认值
- 应用的默认值：
  - 需要双方义务（除非组织仅在披露方）
  - 期限：标准 2-3 年，商业秘密最长 5 年
  - 必需的标准豁免：独立开发、公开可用、合法从第三方获取、法律要求
  - 无非招揽或非竞争条款
  - 无残留条款（或范围有限如存在）
  - 在合理商业司法管辖区的适用法律

### 第三步：快速筛选

对照每个筛选标准系统地评估 NDA。

#### 1. 协议结构
- [ ] **已识别类型**：双方 NDA、单方（披露方）或单方（接收方）
- [ ] **适合背景**：NDA 类型是否适合业务关系？（例如，探索性讨论用双方，单方披露用单方）
- [ ] **独立协议**：确认 NDA 是独立协议，而非嵌入在较大商业协议中的保密节

#### 2. 保密信息的定义
- [ ] **合理范围**：不过于宽泛（避免"任何种类的信息，无论是否标记为保密"）
- [ ] **标记要求**：如果需要标记，是否可行？（口头披露后 30 天内书面标记是标准的）
- [ ] **已有豁免**：定义了标准豁免（见下方标准豁免）
- [ ] **无问题包含项**：不将公开信息或独立开发材料定义为保密

#### 3. 接收方义务
- [ ] **谨慎标准**：合理谨慎或至少与对待自身保密信息相同的谨慎
- [ ] **用途限制**：仅限于规定目的
- [ ] **披露限制**：仅限于受类似义务约束的有知情必要人员
- [ ] **无繁重义务**：无不切实际的要求（例如，加密所有通讯、维护实物记录）

#### 4. 标准豁免
以下所有豁免应存在：
- [ ] **公开知识**：通过接收方无过失方式已或将公开的信息
- [ ] **事先知悉**：接收方在披露前已知晓的信息
- [ ] **独立开发**：在未使用或参考保密信息的情况下独立开发的信息
- [ ] **第三方接收**：从没有限制的第三方合法收到的信息
- [ ] **法律强制**：当法律、法规或法律程序要求时披露的权利（在法律允许情况下通知披露方）

#### 5. 允许的披露
- [ ] **员工**：可与有知情需要的员工共享
- [ ] **承包商/顾问**：可在类似保密义务下与承包商、顾问和专业咨询人员共享
- [ ] **关联公司**：可与关联公司共享（如业务目的需要）
- [ ] **法律/监管**：可按法律或法规要求披露

#### 6. 期限和持续时间
- [ ] **协议期限**：适合业务关系的合理期间（1-3 年是标准的）
- [ ] **保密存续**：义务在终止后存续合理期间（2-5 年是标准的；商业秘密可能更长）
- [ ] **不是永久性的**：避免无限期或永久性保密义务（例外：商业秘密，可能需要更长保护）

#### 7. 归还和销毁
- [ ] **已触发义务**：终止时或应要求
- [ ] **合理范围**：归还或销毁保密信息及所有副本
- [ ] **保留例外**：允许保留法律、法规或内部合规/备份政策要求保留的副本
- [ ] **认证**：销毁认证是合理的；宣誓书是繁重的

#### 8. 救济
- [ ] **禁令救济**：承认违约可能造成无法弥补的损害，公平救济可能适当，这是标准的
- [ ] **无预先确定损害赔偿**：避免 NDA 中的违约金条款
- [ ] **不是单方面的**：救济条款对双方平等适用（在双方 NDA 中）

#### 9. 需要标记的问题条款
- [ ] **无非招揽**：NDA 不应包含员工非招揽条款
- [ ] **无非竞争**：NDA 不应包含非竞争条款
- [ ] **无排他性**：NDA 不应限制任何一方与他人进行类似讨论
- [ ] **无暂停条款**：NDA 不应包含暂停或类似限制性条款（并购背景除外）
- [ ] **无残留条款**（或范围有限）：如存在残留条款，应仅限于有授权访问个人无意识记忆中保留的信息，且不应适用于商业秘密或专利信息
- [ ] **无知识产权转让或许可**：NDA 不应授予任何知识产权权利
- [ ] **无审计权**：标准 NDA 中不常见

#### 10. 适用法律和司法管辖区
- [ ] **合理的司法管辖区**：成熟的商业司法管辖区
- [ ] **一致性**：适用法律和司法管辖区应在相同或相关的司法管辖区
- [ ] **标准 NDA 中无强制仲裁**：NDA 争议通常首选诉讼

### 第四步：分类

根据筛选结果，分配分类：

#### 绿色 — 标准批准

以下**所有**条件必须为真：
- NDA 是双方的（或单方且方向正确）
- 所有标准豁免均存在
- 期限在标准范围内（1-3 年，存续 2-5 年）
- 无非招揽、非竞争或排他性条款
- 无残留条款，或残留条款范围有限
- 合理的适用法律司法管辖区
- 标准救济（无违约金损害赔偿）
- 允许的披露包括员工、承包商和顾问
- 归还/销毁条款包含法律/合规的保留例外
- 保密信息的定义范围合理

**路由**：通过标准授权审批。无需法律顾问审查。
- **行动**：在标准授权下继续签署

#### 黄色 — 需要法律顾问审查

以下**一个或多个**条件存在，但 NDA 没有根本性问题：
- 保密信息的定义比首选的更宽泛，但不是不合理的
- 期限比标准长但在市场范围内（例如，协议期限 5 年，存续 7 年）
- 缺少一个可以不困难地添加的标准豁免
- 残留条款存在但范围有限于无意识记忆
- 在可接受但非首选司法管辖区的适用法律
- 双方 NDA 中轻微不对称（例如，一方有略微更宽泛的允许披露）
- 标记要求存在但可行
- 归还/销毁缺少明确的保留例外（可能隐含但应添加）
- 不寻常但无害的条款（例如，潜在违约通知义务）

**路由**：标记具体问题供法律顾问审查。法律顾问可能只需一次审查就能通过小修订解决。
- **行动**：法律顾问可能只需一次审查就能解决

#### 红色 — 重大问题

以下**一个或多个**条件存在：
- **需要双方时为单方**（或关系方向错误）
- **缺少关键豁免**（尤其是独立开发或法律强制）
- NDA 中**嵌入非招揽或非竞争条款**
- **没有适当业务背景的排他性或暂停条款**
- **不合理的期限**（10 年以上，或没有商业秘密理由的永久性）
- **可能捕获公开信息或独立开发材料的宽泛定义**
- **实际上创建使用保密信息许可的宽泛残留条款**
- NDA 中**隐藏的知识产权转让或许可授予**
- **违约金或罚款条款**
- **没有合理范围或通知要求的审计权**
- **有强制仲裁的高度不利司法管辖区**
- **文件实际上不是 NDA**（包含商业条款、排他性或超出保密范围的其他义务）

**路由**：需要完整法律审查。不要签署。需要谈判、用组织标准 NDA 表格提出反提案或拒绝。
- **行动**：不要签署；需要谈判或反提案

### 第六步：路由建议

根据分类，建议适当的后续步骤：

| 分类 | 建议行动 | 典型时间线 |
|------|---------|---------|
| 绿色 | 根据授权审批并路由签署 | 当天 |
| 黄色 | 附具体标记问题发送给指定审阅者 | 1-2 个工作日 |
| 红色 | 聘请法律顾问进行完整审查；准备反提案或标准表格 | 3-5 个工作日 |

对于黄色和红色分类：
- 识别应该审查的具体人员或角色（如果组织已定义路由规则）
- 包含适合审阅者快速了解关键点的问题简要摘要
- 如果组织有标准 NDA 表格，建议将其作为红色分类 NDA 的反提案发送

## 常见 NDA 问题和标准立场

### 问题：保密信息定义过于宽泛
**标准立场**：保密信息应限于在规定目的相关情况下披露的非公开信息，并有明确豁免。
**修订方法**：将定义缩小到被标记或标识为保密的信息，或在信息性质和披露情况下，理性人会理解为保密的信息。

### 问题：缺少独立开发豁免
**标准立场**：必须包含在未参考或使用披露方保密信息的情况下独立开发的信息的豁免。
**如果缺少的风险**：可能产生关于内部开发的产品或功能是从对手方保密信息衍生的主张。
**修订方法**：添加标准独立开发豁免。

### 问题：员工非招揽
**标准立场**：非招揽条款不属于 NDA。它们适合就业协议、并购协议或特定商业协议。
**修订方法**：完全删除该条款。如果对手方坚持，限制为定向招揽（而非一般招聘）并设置短期（12 个月）。

### 问题：宽泛残留条款
**标准立场**：抵制残留条款。如果必须，限制为：（a）有授权访问的个人无意识记忆中保留的一般想法、概念、专有技术或技术；（b）明确排除商业秘密和可申请专利的信息；（c）不授予任何知识产权许可。
**如果过于宽泛的风险**：实际上为任何目的使用披露方保密信息授予许可。

### 问题：永久性保密义务
**标准立场**：从披露或终止起 2-5 年，以较晚者为准。商业秘密可能需要在保持商业秘密期间的保护。
**修订方法**：将永久性义务替换为规定期限。为商业秘密的更长期保护提供保留条款。

## 说明

- 如果文件实际上不是 NDA（例如，标记为 NDA 但包含实质性商业条款），立即将其标记为红色，并建议进行完整合同审查
- 对于较大协议中的 NDA 部分（例如，MSA 中的保密节），注明更广泛的协议背景可能影响分析
- 始终注明这是筛选工具，法律顾问应审查用户不确定的任何事项
