# /vendor-check — 供应商协议状态

> 如果遇到不熟悉的占位符或需要查看已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

跨所有已连接系统检查与供应商现有协议的状态。提供法律关系的综合视图。

**重要提示**：本命令协助处理法律工作流，但不提供法律建议。协议状态报告在被依赖使用前应由具备资质的法律专业人员对照原始文件核实。

## 调用方式

```
/vendor-check [供应商名称]
```

如未提供供应商名称，提示用户指定要检查的供应商。

## 工作流程

### 第一步：识别供应商

从用户处接受供应商名称。处理常见变体：
- 完整法人名称与商业名称（例如，"Alphabet Inc."与"Google"）
- 缩写（例如，"AWS"与"Amazon Web Services"）
- 母公司/子公司关系

如果供应商名称不明确，请用户澄清。

### 第二步：搜索已连接系统

按优先顺序在所有可用已连接系统中搜索供应商：

#### CLM（合同生命周期管理）— 如已连接
搜索涉及该供应商的所有合同：
- 有效协议
- 已到期协议（最近 3 年）
- 谈判中或待签署的协议
- 修订和附录

#### CRM — 如已连接
搜索供应商/账户记录：
- 账户状态和关系类型
- 相关商机或交易
- 供应商法律/合同团队的联系方式

#### 邮件 — 如已连接
搜索最近相关往来邮件：
- 与合同相关的邮件（最近 6 个月）
- NDA 或协议附件
- 谈判邮件线程

#### 文件（例如 Box、Egnyte、SharePoint）— 如已连接
搜索：
- 已执行的协议
- 修订意见和草案
- 尽职调查材料

#### 即时通讯（例如 Slack、Teams）— 如已连接
搜索近期提及：
- 涉及该供应商的合同请求
- 关于供应商的法律问题
- 相关团队讨论（最近 3 个月）

### 第三步：汇总协议状态

对于找到的每个协议，报告：

| 字段 | 详情 |
|------|------|
| **协议类型** | NDA、MSA、SOW、DPA、SLA、许可协议等 |
| **状态** | 有效、已到期、谈判中、待签署 |
| **生效日期** | 协议开始的时间 |
| **到期日期** | 到期或续期的时间 |
| **自动续期** | 是/否，含续期期限和通知期 |
| **关键条款** | 责任上限、适用法律、终止条款 |
| **修订** | 文件中的任何修订或附录 |

### 第四步：缺口分析

识别存在的协议和可能缺失的协议：

```
## 协议覆盖

[✓] NDA — [状态]
[✓/缺失] MSA — [状态或"未找到"]
[✓/缺失] DPA — [状态或"未找到"]
[✓/缺失] SOW — [状态或"未找到"]
[✓/缺失] SLA — [状态或"未找到"]
[✓/缺失] 保险证明 — [状态或"未找到"]
```

根据关系类型标记任何可能需要的缺口（例如，如果有 MSA 但无 DPA，而供应商处理个人数据）。

### 第五步：生成报告

输出综合报告：

```
## 供应商协议状态：[供应商名称]

**搜索日期**：[今天的日期]
**已检查来源**：[已搜索系统的列表]
**不可用来源**：[如有，未连接系统的列表]

## 关系概述

**供应商**：[完整法人名称]
**关系类型**：[供应商/合作伙伴/客户等]
**CRM 状态**：[如可用]

## 协议摘要

### [协议类型 1] — [状态]
- **生效**：[日期]
- **到期**：[日期]（[自动续期 / 不自动续期]）
- **关键条款**：[重要条款摘要]
- **位置**：[已执行副本的存储位置]

### [协议类型 2] — [状态]
[等等]

## 缺口分析

[已就位的内容与可能需要的内容]

## 即将到来的行动

- [任何即将到期或续期截止日期]
- [尚未就位的必需协议]
- [可能需要的修订或更新]

## 备注

[来自邮件/即时通讯搜索的任何相关背景]
```

### 第六步：处理缺失来源

如果通过 MCP 未连接关键系统：

- **无 CLM**：注明未连接 CLM。建议用户手动检查其 CLM。报告在其他系统中找到的内容。
- **无 CRM**：跳过 CRM 背景。注明缺口。
- **无邮件**：注明未搜索邮件。建议用户在邮件中搜索"[供应商名称] 协议"或"[供应商名称] NDA"。
- **无文件**：注明未搜索文件存储。

始终清楚地说明检查了哪些来源，哪些没有检查，以便用户了解报告的完整性。

## 说明

- 如果在任何已连接系统中均未找到协议，清楚地报告这一点，并询问用户是否在其他地方存储了协议
- 对于供应商集团（例如，有多个子公司的供应商），询问用户是否要检查特定实体或整个集团
- 标记任何已到期但可能仍有存续义务的协议（保密性、赔偿等）
- 如果协议即将到期（90 天内），突出显示这一点
