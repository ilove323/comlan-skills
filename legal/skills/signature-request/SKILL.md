# /signature-request — 电子签署路由

> 如果遇到不熟悉的占位符或需要查看已连接的工具，请参阅 [CONNECTORS.md](../../CONNECTORS.md)。

准备文件进行电子签署——核实完整性、设置签署顺序并路由执行。

**重要提示**：本命令协助处理法律工作流，但不提供法律建议。发送签署前，请核实文件已是最终版本。

## 用法

```
/signature-request $ARGUMENTS
```

准备签署：@$1

## 工作流程

### 第一步：接受文件

以任何格式接受文件：
- **文件上传**：PDF、DOCX
- **URL**：~~cloud storage 或 ~~CLM 中的文件链接
- **引用**："我们昨天最终确定的 Acme Corp MSA"

### 第二步：签署前清单

路由签署前，核实：

```markdown
## 签署前清单

- [ ] 文件已是最终商定版本（无未解决修订意见）
- [ ] 所有附件和附录已附上
- [ ] 签署块上的正确法人主体名称
- [ ] 日期正确或留空以待执行日期填写
- [ ] 签署块与授权签署人匹配
- [ ] 任何必要的内部审批均已获得
- [ ] 文件已由适当法律顾问审阅
```

### 第三步：配置签署

收集签署详细信息：
- **签署人**：谁需要签署？（姓名、邮件、职务）
- **签署顺序**：顺序还是并行？
- **内部审批**：在对手方签署之前是否需要有人审批？
- **抄送收件人**：谁应收到已执行文件的副本？

### 第四步：路由签署

**如果已连接 ~~e-signature：**
- 创建签署信封/请求
- 设置签署字段和顺序
- 添加任何必要的首字母缩写或日期字段
- 发送签署

**如果未连接：**
- 生成签署说明文件
- 提供格式化为湿签或手动电子签署的文件
- 列出所有签署人及联系方式

## 输出

```markdown
## 签署请求：[文件标题]

### 文件详情
- **类型**：[MSA / NDA / SOW / 修订等]
- **各方**：[甲方] 和 [乙方]
- **页数**：[X]

### 签署前检查：[通过 / 发现问题]
[列出发送前需要关注的任何问题]

### 签署配置
| 顺序 | 签署人 | 邮件 | 角色 |
|------|--------|------|------|
| 1 | [姓名] | [邮件] | [甲方授权签署人] |
| 2 | [姓名] | [邮件] | [乙方授权签署人] |

### 抄送收件人
- [姓名] — [邮件]

### 状态
[已发送签署 / 准备好发送 / 需先解决问题]

### 后续步骤
- [发送后的预期]
- [预期周转时间]
- [如果 X 天内未签署的跟进]
```

## 提示

1. **仔细检查主体名称** — 最常见的签署错误是法人主体名称不正确。
2. **核实授权** — 确保每位签署人有权约束其组织。
3. **保留副本** — 执行后应立即将已执行副本归档到 ~~cloud storage 或 ~~CLM 中。
